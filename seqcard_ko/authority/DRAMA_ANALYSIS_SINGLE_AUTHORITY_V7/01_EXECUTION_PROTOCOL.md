# 01. 실행 프로토콜

## A. 작업 개시
1. 작품 ID, 총 회차, 원본 위치, 판본, 원본 해시를 확정한다.
2. `source_lock/current/<work_id>.source_lock.v4.json`을 만든다.
3. 총 회차를 8회 이하의 연속 블록으로 계산한다. 예: 18부는 EP01–08, EP09–16, EP17–18. 블록은 계산 단위이며 이야기 구조를 강제로 끊지 않는다.
4. `work_state.json`에 블록 계획과 상태를 기록한다.

## B. 블록 실행 루프
각 블록은 앞에서 뒤로 순서대로 처리하며 블록 감사 PASS 전에는 다음 블록으로 가지 않는다.

### B1. 원문 직접 독해
블록의 모든 대본을 회차 순서대로 처음부터 끝까지 읽는다. 인물, 장소, 시간, 행동 목표, 갈등, 결정, 정보 공개, 결과 상태와 앞 블록의 열린 질문·약속·위협·관계 변화를 추적한다. 기존 분석은 독해 완료 뒤 비교자료로만 본다.

### B2. Stage01
장면 경계는 시간·장소·참여자·행동 목표·갈등 축·결과 상태 변화로 판정한다. 모든 장면을 원문 순서로 기록하고 관찰 가능한 사건과 변화만 쓴다. `core_primary`는 주 기능, `core_secondary`는 실제 보조 기능이 있을 때만 쓴다. 회차 완료 즉시 EpisodeMeta를 계산한다.

### B3. Stage02
인접 장면을 하나의 목표/문제/행동 진행과 종결 전환으로 묶는다. 시퀀스 수를 미리 정하지 않는다. 회차 전체 장면을 중복·누락 없이 연속 타일링한다. `turn_type`은 마지막 소속 장면 primary core, `turn_class`는 권위 파생표 결과다.

### B4. Stage03
- EpisodeArc: 시작 상태→압력→선택/충돌→종결 상태.
- CharacterArc: 실제 상태 변화가 있는 인물만.
- RelationshipArc: 신뢰·권력·감정·의존이 실제로 변할 때만.
- LocalEdge: 같은 회차 또는 짧은 인접 구간의 명확한 원인→결과.
- PayoffCandidate: 장기 회수 가능성과 원문 근거가 있는 설정·약속·정보·위협만.

### B5. 블록 종료 감사
모든 회차 Stage01–03 존재, 장면/시퀀스 범위, FK, 앞 블록 후보 상태, 반복 고정 수량과 문구 재사용을 감사한다. 실패하면 수정 후 재검사한다.

## C. Stage04
모든 블록이 끝난 후에만 수행한다. 전 후보를 재수집해 정확히 하나의 disposition으로 처분하고, 명시적 회차 간 원인→결과만 causal edge로 쓴다. setup→발전/지연→payoff→상태 변화가 입증된 것만 Payoff로 확정한다. FullSeriesArc는 전 시즌 관점에서 신규 저작하며 회차 산출물 단순 합본으로 만들지 않는다.

## D. 중단·재개
세션 중단 전 `work_state.json`에 마지막 완료 회차·stage, 열린 후보, 미해결 검사, 다음 행동을 기록한다. 새 세션은 이 권위와 work_state만 읽고 이어서 실행한다.
