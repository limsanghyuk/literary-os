## 0. SPE 감독 요지 (먼저 읽을 것)

개발자 지적 3·4·5는 방향은 옳으나 **3건의 잘못된 전제와 1건의 자기모순**을 포함한다. 무조건 수용 시 "고장난 손실을 스케일링"하는 최악 경로(LADDER 전략 A)로 직행한다. SPE 판정: **데이터 스케일 선행을 차단(BLOCK)**, 페어 구조·학습 단위·평가 신뢰를 먼저 고친 뒤 스케일한다. 처방1~3도 무수정 채택이 아니라 3건을 보강해 확정한다.

| 항목 | 개발자 주장 | SPE 판정 | 조치 |
|---|---|---|---|
| B1 | "데이터가 작다" = 1차 문제 | **부분오류** | BLOCK: 1차는 신호 다양성. 스케일은 B 입증 후 |
| B2 | 20k면 복선·회차구조·장기페이오프 학습 | **원리오류** | CORRECT: 씬/회차 단위 DPO로 아크 단위 목표 학습 불가 |
| B3 | 0.357 → "문학성 미달" | **자기모순** | BLOCK: 무효 인정한 지표로 결론. per-token 재측정 전 보류 |
| B4 | train/held 둘 다 작다 | **불완전** | REFINE: held=56이 평가 신뢰의 즉시 병목(우선 확대) |
| B5 | 암기/표절 = 평가 6항목 중 1개 | **과소평가** | ELEVATE: 저작권 원칙 직결 → 하드 안전 게이트 |

## 1. 개발자 지적 비판 검토 (블로킹/수정)

### B1 — "데이터가 작다"는 1차 문제가 아니다 (BLOCK)
224/56쌍이 적은 건 사실이나, **같은 페어 구조(chosen=ref, rejected=AI draft)를 280→20k로 늘리면 표면 신호(길이·포맷·어투)와 ref 암기가 증폭**될 뿐 깊은 목표는 안 늘어난다. 권장 규모 표(500–2k / 5k–20k / 20k+)는 일반 LLM 파인튜닝 민담이지 DPO·본 셋업 캘리브레이션이 아니다. DPO는 수백 쌍으로도 방향이 보인다. → **양보다 페어 구조 선결.**

### B2 — 일부 목표는 양 문제가 아니라 학습 단위 불일치다 (CORRECT)
현재 쌍은 `next_episode_bench`(N화 생성 vs 실제 N+1 원작) 기반 — **생성 단위(씬/회차) 수준**이다(`self_eval_pipeline.to_preference_pairs`, `loop_c.PreferencePair`). 그런데 개발자가 나열한 "회차 구조·장기 페이오프·인물 궤적"은 **여러 회차에 걸친 아크 단위** 속성이다. 씬/회차 쌍 DPO는 이 아크 단위 속성을 **몇 쌍을 줘도 원리적으로 학습 못 한다**(unit-of-learning mismatch). → 이 목표들은 DPO가 아니라 **계층 플래너 + 구조 게이트(아크 단위)**가 담당해야 한다(제품 비전: 화→거시/미시플롯→씬). 데이터 스케일로 풀려는 시도 자체가 오진.

### B3 — 0.357 결론은 자기모순이다 (BLOCK)
개발자는 "이 지표는 단순 log-prob라 완벽한 품질평가 아님"을 인정하고도 같은 문단에서 "0.357은 낮다 → 문학성 강화 성공이라 보기 어렵다"를 결론낸다. **무효라고 선언한 자로 잰 값으로 결론낼 수 없다.** 게다가 0.357<0.5는 단순 "낮음"이 아니라 모델이 ref보다 draft를 64% 더 선호하는 **부호 역전** 신호일 수 있는데, 이는 길이 교란(draft가 더 긴데 sumlogP는 길수록 누적 음수↑)으로 충분히 설명된다(V788 실측: draft 593자 vs ref 424자, "짧은 쪽=승자" 규칙이 라벨 95.5% 재현). → **per-token 재측정(V788 도구) 전에는 개선 여부·방향 모두 판단 보류.**

### B4 — held=56이 더 급한 병목이다 (REFINE)
코드 `MIN_PAIRS_RELIABLE=50`(winrate_gate). held=56은 간신히 신뢰 하한이라 승률 CI가 과대하다. 학습용 train 부족보다 **평가 신뢰가 즉시 병목**이다. → held 우선 확대(≥250 목표, LADDER §3.3 Σn_pairs와 정합).

### B5 — 암기/표절은 하드 안전 게이트로 승격 (ELEVATE)
chosen=저작권 한국 명작 ref인 DPO는 본질적으로 "ref의 logp를 올려라"라서 **ref 암기·재현이 중심 위험**이다. 이는 품질이 아니라 본 프로젝트가 지켜온 "ref 원문 비노출·비커밋" 원칙을 깨는 **법적·안전 사고**다. → n-gram 중복/표절은 평가 6항목 중 1개가 아니라 통과를 막는 하드 게이트.

## 2. 3인 교차 논의

### 2.1 ARCH (수석 아키텍트) — 학습 단위와 책임 분리
"B2가 핵심이다. 현재 루프는 **국소(씬/회차) 선호학습**과 **광역(아크) 구조**를 한 메커니즘에 욱여넣고 있다. 책임을 분리하라:
- 국소 문체·대사·지문 = DPO 선호학습(현 루프, 페어링만 개선).
- 광역 복선·페이오프·궤적 = 계층 플래너 + 아크 단위 구조 게이트(c3의 R_path/R_pair를 회차 경계로 확장).
LADDER로 보면 국소는 LLM-1→2 졸업 대상, 광역은 LLM-2의 '생성 주력' 오케스트레이터 영역. 둘을 섞으면 둘 다 실패한다."

### 2.2 COMP (수석 컴파일러) — 무엇이 하드 게이트로 컴파일되는가
"나는 '측정 가능·결정론·강제'만 신뢰한다.
- B5 암기 게이트: 결정론(최장 공통 n-gram, ROUGE-L)이라 **하드 게이트로 즉시 컴파일 가능**. 채택.
- 처방1의 P1(대조 열화)도 결정론(critic_qualification 축 재사용)이라 컴파일 친화적. **단, 경고**: 열화쌍은 너무 분리 가능(문장 셔플·어휘치환 흔적)이라 모델이 '열화 탐지'만 배우고 pref-acc가 **과대 팽창**할 수 있다. 이건 reward-hacking이다. → 열화 강도를 등급화하고 P2 온폴리시와 혼합해 분리도를 낮춰야 게이트가 정직하다.
- 처방3의 E2 LLM-심사: 비결정·유료. `G_LLM1_COST`(soft $30/hard $50) 한도와 충돌. 20k쌍×심사호출은 예산 초과. → 심사는 표본추출(예: held 전수 + train 층화표본)로만, 위치 스왑 의무."

### 2.3 DATA (수석 데이터 전문가) — 페어 공급과 현실 제약
"처방1 페어 다변화에 동의하나 **공급 현실을 보정**한다.
- chosen 다양화의 한계: 우리 명작 ref는 유한(corpus_ko 2,030작품). chosen을 ref 밖으로 늘리면 합성 chosen이라 목표가 바뀐다. 그래서 **P3(초고간 비교, chosen·rejected 모두 AI)**가 양 확보의 주력이어야 한다 — 암기위험 0, 공급 무제한.
- '122k씬에서 채굴'은 과대낙관: rejected 생성은 씬마다 LLM 호출 비용. 122k 전부는 비현실적. → 장르 층화 표본으로 1차 2k, 그다음 장르별 5k.
- held 확대(B4)는 **train과 작품 누출 없이**(같은 작품 씬이 train/held 양쪽에 들어가면 평가 오염) 작품 단위 분할로."

### 2.4 교차 충돌 → 합의
| 충돌 | 입장 | 합의 |
|---|---|---|
| P1 비중 | COMP: 과대팽창 위험 / DATA: 저렴·고신호 | P1은 **1차 부트스트랩 한정**, 등급화+P2혼합으로 분리도 완화, 게이트 신뢰는 P3/held로 교차확인 |
| 양 확보 주력 | DATA: P3(AI간) / ARCH: 온폴리시 P2 | 1차=P1+P3(저비용), 2차=P2 온폴리시 추가(현 모델 의존) |
| LLM-심사 범위 | COMP: 예산 / 처방3: 광범위 | 표본추출+스왑, 전수는 per-token(E1)·암기(E4) 결정론 게이트가 담당 |
| 광역 목표 | ARCH: DPO 밖 | 회차/아크 구조는 DPO 비대상 → 계층 플래너로 이관(별도 트랙) |

## 4. LADDER 정합 (§3 빈칸 채움)

본 확정안은 DESIGN-LLM-LADDER-v1 §3.3(L1→L2 졸업)의 데이터·평가 빈칸을 채운다:
- "길이 단순규칙 재현율≤0.60" ← E1+길이매칭 페어링으로 충족.
- "Σn_pairs≥250" ← held 250 확대(B4).
- "per-token W CI하한>0.5" ← E1 전수 + E2 표본 교차.
- 추가: **E4 암기 하드게이트를 L1→L2 졸업 전제로 편입**(암기로 올린 승률은 무효).
- ARCH 분리 원칙 → 광역 목표는 LLM-2 오케스트레이터/계층 플래너 트랙으로 명시 이관.

## 5. SPE 최종 점검 (반론·미해결)

1. **P1 과대팽창(COMP 경고)이 가장 큰 잔존 위험.** 1차 pref-acc가 좋아 보여도 열화 탐지 학습일 수 있음 → E1은 P3·held 전수로 교차검증, P1 단독 졸업 판정 금지.
2. **광역 목표 이관은 회피가 아니라 재배치.** 계층 플래너 트랙의 게이트(아크 단위 R)는 아직 미설계 → 별도 후속(DESIGN-HIER-PLANNER) 필요.
3. **E2 표본의 통계력.** 예산 제약상 표본이면 CI가 넓다 → held 전수 결정론 게이트(E1/E4)가 1차 판정, E2는 보조.
4. **모든 임계 미실측.** 실 GPU 1라운드 전이라 2k/250/0.60 등은 제안값 → 실측 후 캘리브레이션.
5. **개발자 지적의 가치 인정.** 4·5번(좁은 목표·약한 지표)의 문제의식은 정확했고 본 설계의 출발점 — 차단한 것은 "스케일 선행"과 "무효지표 결론"뿐.

**SPE 확정:** 위 1~5 보강을 전제로 §3 통합 처방을 채택한다. 다음 액션 1순위는 **E1 per-token으로 0.357 재측정**(개선 여부의 진짜 답) + **E4 암기게이트 결정론 구현**.

## 6. 실행 로드맵
1. E1 재측정 → 0.161/0.357 재해석(결론 보류 해제).
2. E4 암기 하드게이트(`learning/memorization_gate.py`, 결정론) + 테스트.
3. P1 등급화 열화 + P3 생성기로 1차 2k/held 250(작품분리).
4. 1차 DPO 방향성 재실행 → LADDER §3.3 게이트로 판정.
5. 광역 목표 별도 트랙(DESIGN-HIER-PLANNER) 착수.

## 7. 자기 점검
3인 역할은 동일 모델의 분석 페르소나로, 진짜 조직적 독립성은 아니다(확증편향 잔존 가능) → 실 개발자 리뷰로 외부 검증 권고. 수치·비중(30/40/20/10, 2k/250)은 휴리스틱. 암기 게이트 임계(n-gram 길이)는 장르·언어별 보정 필요.

## 8. 멀티에이전트 독립 교차검토 (§7 자기점검 해소)

§7에서 인정한 "동일 모델 페르소나" 한계를 해소하기 위해, **진짜 독립 서브에이전트 3기**(ARCH/DATA/COMP)를 병렬 기동해 §3 처방을 외부 검증했다. 페르소나가 아니라 별도 컨텍스트의 독립 에이전트이므로 확증편향이 제거된 비판이 회수됐고, 그 결과 본안의 실제 결함 3건이 교정됐다.

**ARCH 회수 — B2 과장 교정.** "씬·회차 DPO로는 아크 학습 원리적 불가"는 과장. 진짜 블로커는 DPO 자체가 아니라 **현행 N-vs-N+1 페어링이 회차 단위라 광역 아크 신호를 담지 못한다**는 점. 장문(long-context) 페어는 아직 미실험일 뿐 원리적 불가가 아니다. 누락 리스크 = **크레딧 할당 계약 부재**(어느 씬/축의 기여로 winner가 갈렸는지 인터페이스가 없음). → 본안 B2 표현을 "원리적 불가"에서 "현 페어링 부적합 + 장문 페어 미검증"으로 완화하고, 크레딧 할당 인터페이스 계약을 별도 빈칸으로 등록.

**DATA 회수 — P1 길이교란 재유입 차단.** P1(등급화 열화) 비중 30%는 위험. 열화 4축 중 3축(flatten_emotion·remove_foreshadow·generic_diction)이 텍스트를 **짧게** 만들어 V788에서 차단한 "짧은 쪽이 이긴다" 길이교란을 재유입시킨다. → **P1 30%→15%로 축소**, 혼합비 **15/55/20/10**(P1/P3/P2/P4)로 조정, 열화쌍은 **길이매칭** 필수 + 길이중립축인 break_causality 가중. E4 한국어 임계는 연속일치 ≈15형태소·ROUGE-L(LCS)≥0.35·동시발화.

**COMP 회수 — E4 결정론 구현 명세.** 외부 의존성 0의 char-level 게이트로 구현 가능. LCS비율·최장연속일치·char n-gram Jaccard 3신호 + 페널티 규약(-9.99). → 본 회차에서 `learning/memorization_gate.py`로 **실구현·12 테스트 green** 완료.

**SPE 종합 교정 반영:**
1. B2: "DPO 원리적 불가" → "현 N-vs-N+1 페어링이 광역아크 부적합; 장문 페어 미검증". 크레딧 할당 계약을 신규 빈칸으로 등록.
2. 페어링 혼합비: 30/40/20/10 → **15/55/20/10**(P1 축소, P3 생성기 주력 확대). 열화쌍 길이매칭 강제.
3. E4: 설계→**구현 완료**(char-level, 한국어 안전, 무의존성, 결정론). 임계는 보수적 기본값, 코퍼스 캘리브레이션 후 ADR 재보정.

**잔존 한계(정직 고지):** 독립 에이전트라도 동일 코드베이스·동일 측정치를 본다 → 데이터 자체의 편향은 공유. 실 GPU ΔW 1라운드·인간 블라인드 평가만이 최종 검증. 혼합비 15/55/20/10·임계값은 여전히 미실측 휴리스틱.

## 9. 누적분 (2026-06-20) — 기법축 페어링 업그레이드 + 안티-LLM 목업 즉석 실측

개발자 통찰 회수: "인간 글이 짧은 것은 길이 인공물이 아니라 **기법(show-don't-tell)의 그림자**"(AI=감정 직접서술·장황 / 인간=행동·표정 묘사·압축). 이를 §3에 누적한다.

**확정 누적사항:**
1. per-token 유지(측정 정직성, sum 금지 ADR-LADDER-3). **부정되지 않음.**
2. P0 격상: "기계적 길이매칭" → **"기법을 가르쳐 자연히 짧아지게"**. 측정=per-token, 데이터=안티-LLM+작가문체로 압축·show 학습.
3. 미세신호 per-token 마진 +0.434/token(명작쪽) → **안티-LLM 페어가 1순위 강화 후보**.
4. AI 자기평가 → **보조 판정으로 강등**(judge=writer 순환편향). 확정=per-token held-out + 인간 블라인드.

**안티-LLM 목업 즉석 실측(자작 8쌍, verbatim 비포함, GPU 불요):**
- 길이 showing/telling=0.80(8/8 짧음) — "명작이 짧다"와 동일 방향.
- 명시 감정어 telling 5.12/편 → showing **0.00/편** = show-don't-tell 기전 직접 입증(길이 단축의 원인).
- **길이교란 재현**: sum 기준 showing 선호 **8/8** vs per-char 기준 **3/8**(우연). V788 "짧은쪽=승자 92.5%"의 미니 재현.
- 한계: surprisal 프록시 Δ+0.01(무의미). 명작 토큰당 밀도↑는 8B GPU에서만 분해 → P3 필요 강화.

**P0 갱신:** P1/P3 생성기에 안티-LLM+작가문체 적용; P1 열화="안티-LLM 끄기(telling 재주입)"; ⚠️안티-LLM 페어는 **sum 채점 시 8/8 거짓압승=Round#2 부호반전 재현** → 길이매칭+per-token 강제; 페어 차이를 기법 축(감정어 호명률)으로 계측.
