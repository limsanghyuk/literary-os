## 3. 핵심 발견

**(A) 기능·인물 축 = 강함.** 시목의 정의적 특질(이명 + 감정 차단)이 7씬 전부에 정확히 재현됨. intent_gist/core가 "무엇을 각인시킬지"를 안정적으로 전달. → SeqCard의 의도층은 이미 렌더러를 올바른 방향으로 조종한다.

**(B) 사실·동선 축 = 약함.** blueprint가 구체적 사건/소품/공간동선/폭로장치를 담지 않아 렌더러가 진공을 창작으로 메움:
- S2: 원본=번지수 개그·"빚쟁이" 오해·케이블차 위기·반찬통. 생성=단전 고지서·이름 김정순·차내 취조. **동일 기능, 전혀 다른 내용.**
- S5: 원본=폭탄 맞은 듯 어질러진 집 + 시목이 흉기 챙김. 생성=말끔한 현장 + 소주병. **핵심 사실 누락.**
- S7: 원본=시목이 방 훑고 장미무늬 주방칼 발견, "자상 셋·오른손잡이·원한". 생성=**존재하지 않는 협박 녹음파일 서브플롯을 환각(hallucinate)** — blueprint에도 원본에도 없는 음모를 창작.

**(C) 밀도 축 = 무통제.** 길이 제약 필드 부재 → 원본 대비 2.7–10.1× 팽창. 특히 짧고 절제된 씬(S4·S6)에서 최악(×8–10).

## 4. Ablation — intent_gist는 load-bearing인가?

S5에서 title+intent_gist를 제거하고 재렌더:
- **intent 포함**: 시목의 무감각을 강하게 각인 — "표정은 평평하다", 차갑게 "살인입니다" 선언, 법의학적 자문 "강타. 둔기. 저항 흔적 미약."
- **intent 제거**: 유능한 절차극이지만 **인간적으로 표류** — 어머니를 위로("손수건을 접어 손에 쥐어준다", "기억하세요"), 차가운 심리 각인 약화.

→ intent_gist는 "각인시킬 기능"을 실제로 산출물에 주입한다. 제거 시 그 기능이 소실. **필드 가치 기능적 입증.**

## 5. 결론 → 스키마 정식화 동인

SeqCard 현재 스키마(intent_gist/core/core2/skin)는 **기능·인물층을 강하게 전달하나, 사실·동선·폭로·제약층이 비어 있다.** 이 공백이 (B)(C) 결함의 직접 원인. GPT SceneBlueprint의 dramatic_conflict / causal_input·output / plant·payoff_operations / information_reveal / character_entry·exit_state / visual_directorial_notes / renderer_prompt_constraints 필드가 정확히 이 공백을 메우는 대상 → 다음 문서(스키마 정식화)에서 매핑·갭 확정.
