# 품질 기준선 평가 — 프론티어 생성 vs 실제 명작 블라인드 쌍대 (v1, 2026-06-24)

상태: **방법 실행됨 / ★결과 신뢰불가(평가 편향) — 조기 성공선언 금지 적용**
산출: `examples/quality_baseline_eval.py` (gen/judge 2단계, gpt-4o-mini)
배경: 사용자 제안 — "4070의 8B로 프론티어와 경쟁은 비현실 → GPT로 산문 생성·평가해 품질 기준선을 잡자." (타당. GenerativePort 좌석에 프론티어는 합법)

## 0. 한 줄
프론티어로 생성 + 블라인드 쌍대 평가를 **실행**했다(메커니즘 PASS). 그러나 결과(생성 15/15 전승)는 **진짜 우월이 아니라 평가 편향의 산물**이라, 이대로는 품질 기준선이 될 수 없다.

## 1. 방법
- 생성: gpt-4o-mini에 한 줄 전제만(실제 텍스트 누출 0) → 시그널/대장금/미생 1화 오프닝 생성.
- 평가: 실제 명작 오프닝 vs 생성본을 **A/B 순서 무작위 블라인드**로 gpt-4o-mini 심사 → 5축(구조/인물/대사/감정/장르) + 전체.

## 2. 결과 (표면)
| 작품 | 5축 + 전체 |
|---|---|
| 시그널 | 생성 6/6 승 |
| 대장금 | 생성 6/6 승 |
| 미생 | 생성 6/6 승 |
→ **생성 15/15 축 전승, 전체 3/3.** 

## 3. ★이건 우월이 아니라 편향이다 (자기비판)
너무 깨끗한 전승 = 방법 결함의 신호. 4가지 교란:
1. **AI-judge-AI 편향**: 심사도 gpt-4o-mini, 생성도 gpt-4o-mini → 자기 분포·문체 선호. (메모리 기존 실측: "AI 심사위원이 안티-LLM·인간 문체 구별 못함".) 같은 모델 패밀리 자기심사는 무효에 가깝다.
2. **길이·완결성 비대칭**: 실제 발췌 220~651자(잘린 단편·맥락 없음) vs 생성 745~980자(자기완결 오프닝). 심사가 "완성도"로 읽으면 생성 유리.
3. **맥락 결손**: 실제 오프닝은 느린 빌드(예: 시그널 발췌는 대사 중간 인서트, 미생은 220자 내레이션 도입)인데, 단장면만 떼어 비교 → 명작의 장기 설계가 평가에 안 들어옴.
4. **형식 청결도**: 생성=깔끔한 현대 대본체 / 실제=OCR·사극체·희소 대사태그 노이즈. 표면 가독성이 점수에 혼입.

## 4. 결론
- **메커니즘**: 생성→블라인드 쌍대 평가 파이프라인은 작동한다(좌석·심사 결선 확인).
- **신뢰성**: 현재 구성(동일모델 자기심사 + 잘린 단편)으로는 **품질 기준선으로 무효**. BROAD-MEASUREMENT가 C(인간명작 직접대조)를 "천장 시험"으로 미룬 이유, 3팀의 Goodhart 경고가 그대로 재현됐다.
- 사용자 제안의 **생성** 부분은 유효(프론티어가 좌석에서 실 산문 생성). 막힌 건 **평가**: AI 자기심사는 못 쓴다.

## 5. 유효한 품질 기준선의 요건 (다음)
1. **이종 심사**: 심사자를 생성자와 다른 패밀리로(생성 GPT면 심사 Claude, 또는 인간). 자기심사 금지.
2. **공정 입력**: 실제 명작을 **완결 장면 단위·동일 분량대**로 정규화(잘린 단편 비교 금지). 형식 노이즈 정리.
3. **인간 GT 표본**: 소수라도 인간 심사 라벨(천장 시험 본체).
4. **5축 분해 + 축간 균형**(BROAD-MEASUREMENT A+B) — 단일 전승 수치 신뢰 금지.
5. 그 위에서 비로소 (현재) 프론티어 좌석 기준선 → 이후 LLM1Port(8B) 격차 측정.
