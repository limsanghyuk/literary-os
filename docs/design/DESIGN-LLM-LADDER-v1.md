# DESIGN-LLM-LADDER-v1 — LLM 자율성 사다리 (LLM-0 ~ LLM-3) 상세 설계도

상태: DRAFT v1 · 2026-06-19 · 기준 HEAD `ec1b5e2`(SGATE-v788) · 작성=계획/설계 모드
관련: DESIGN-SGATE-v1(c3 구조게이트), ADR-243(M1 자격검정), ADR-218(G_LLM1_COST), ADR-031(LLM0StaticGate)
전제: 본 문서는 "각 자율 단계에서 공식이 얼마나 강제하고 LLM이 얼마나 자유로운가, 그리고 다음 단계로 올라가려면 어떤 게이트를 **수치로** 통과해야 하는가"를 못 박는 계약서다.

## 0. TL;DR — 한 장 요약

| 레벨 | 비유(자율주행) | 공식 권한 | LLM 권한 | 진급 게이트(요지) | 현 상태 |
|---|---|---|---|---|---|
| **LLM-0** | L1 운전보조 | 전권(하드 거부) | 호출 0 (결정론) | LLM0StaticGate: llm_bridge import 0건 | ✅ 가동(ADR-031) |
| **LLM-1** | L2 부분자율 | 바닥(floor)+거부권 | 쌍대 판정(a/b/tie) only | 자격검정 WIN≥0.80·단조 ∧ G_LOOPC adopt | ✅ Critic 코드+게이트(오늘 c3 실체화) |
| **LLM-1.5** | L2+ | 바닥 | 5축 전체 AI + 생성 초안만 완화 | 5축 자격 동시통과 ∧ 비용 게이트 | ◻ Phase F 미착수 |
| **LLM-2** | L3 조건부자율 | 안전 바닥만(비퇴행 감시) | 생성 주력 | 실 GPU ΔW 신뢰라운드 누적 adopt(per-token) | ◻ **미설계 핵심** |
| **LLM-2.5** | L3+ | 안전 바닥 | 자율 평가루프 | 자가평가 vs 인간 일치 κ 임계 | ◻ Phase G |
| **LLM-3** | L4 고도자율 | 최후 안전망만 | 거의 전권+자기교정 | 블라인드 인간평가 vs 실명작 비열위 | ◻ 천장(개념) |

**핵심 불변식(전 레벨 공통):** 공식 점수 R은 **"넘어야 할 바닥(floor)"이지 "최대화할 보상(reward)"이 아니다** (G_NO_ABSOLUTE_REWARD). 레벨이 올라갈수록 바닥은 낮아지되 절대 사라지지 않는다.

## 1. 설계 원칙

### 1.1 자율주행 사다리 비유
공식(formula)은 정답 생성기가 아니라 **AI 학습의 보조 해법이자 안전 검사기**다. 자율주행이 L1→L5로 운전자 개입을 점진적으로 줄이듯, 본 시스템은 LLM-0→LLM-3로 공식 제약을 점진적으로 완화한다. 단번에 전권을 주지 않는 이유: 자유를 주는 순간 품질이 무너졌는지 검증할 수단이 없으면 되돌릴 수 없기 때문이다.

### 1.2 바닥 vs 보상 (Goodhart 안전)
구조 점수 R을 보상으로 직접 주면 AI가 점수만 노리고 실질 품질을 버린다(Goodhart). 따라서 R은 **통과/탈락의 바닥**으로만 쓴다. Critic은 절대 점수를 내지 않고 **쌍대 승자(a/b/tie)만** 판정한다(`critic/base.py`, G_NO_ABSOLUTE_REWARD).

### 1.3 진급은 자동이 아니라 게이트 통과
각 레벨 전이는 "감(感)"이 아니라 **측정 가능한 수치 계약**을 통과해야 한다. 통과 못 하면 그 레벨에 머문다. 오늘(V788) 실체화한 c3 구조 게이트가 바로 LLM-1→LLM-2 진급 시험의 핵심 부품이다.

## 2. 레벨별 상세 정의

### LLM-0 — 결정론 (외부 LLM 호출 0)
- **정의:** 모든 추론이 구조화 메타데이터 산술. 외부 LLM 미호출.
- **공식 권한:** 전권. 하드 거부권(게이트 FAIL = 작업 차단).
- **LLM 권한:** 없음.
- **시행 장치:** `graph_intelligence/llm0_static_gate.py` — AST 정적 분석으로 `llm_bridge` 직접 import 0건 강제(ADR-031). 위반 시 게이트 FAIL.
- **산출물:** 그래프 인덱싱·서사 영향 분석·패치 임팩트·ASD(Auto Story Doctor) 전부 LLM-0 준수.
- **상태:** ✅ 가동 중(V546~).

### LLM-1 — 쌍대 Critic (부분자율)
- **정의:** LLM이 두 후보를 **비교만** 한다(생성 주도 아님). 절대 점수 금지.
- **공식 권한:** 바닥(floor) + 거부권. 공식이 baseline sanity를 잡고, Critic은 그 위에서 미세 변별.
- **LLM 권한:** 5축(STRUCTURE·CHARACTER·DIALOGUE·EMOTION·GENRE) 쌍대 판정 winner∈{a,b,tie}.
- **필수 가드:**
  - `G_LLM1_RAG` — RAG 컨텍스트(rag_refs) 없는 critic 호출 거부(`critic/base.py CriticContext`).
  - `G_NO_ABSOLUTE_REWARD` — 절대 점수 금지.
  - `G_LLM1_COST` — 월 비용 soft $30 / hard $50(ADR-218, `critic/llm1_metrics.py`).
- **자격검정(진입 조건):** M1 자격검정(ADR-243, `critic/critic_qualification.py`) — 등급화 열화 사다리(미세→심각)에서 명작 우위 승률 WIN_MIN ≥ 0.80 + 변별곡선 단조 증가. 축별 표적 열화(복선 제거·감정 평탄화·인과 단절·상투 어휘) 통과.
- **자기개선 게이트(졸업 시험):** `G_LOOPC_WINRATE`(`learning/winrate_gate.py`) = c1(ΔW>0) ∧ c2(KL≤τ=0.50) ∧ c3(구조 R 비퇴행). c3는 오늘 `critic/structure_conformance.py`로 실체화됨(이전엔 자동통과).
- **상태:** ✅ Critic 코드(V753~782) + 게이트 완성. 실 GPU ΔW 1라운드 미실행(RunPod 키 차단).

### LLM-1.5 — 5축 전체 AI + 생성 초안 완화 (Phase F)
- **정의:** Critic 5축을 전부 AI로 전환하고, **생성은 초안에 한해서만** 공식 제약을 완화. 코퍼스 200편·다언어 확장.
- **공식 권한:** 바닥. 생성 초안 단계에서만 일부 비퇴행 감시로 완화.
- **진입 조건:** 5축 각각이 LLM-1 자격검정을 동시 통과 ∧ 비용 게이트 녹색 유지.
- **상태:** ◻ Phase F(V796~875) 미착수.

### LLM-2 — 생성 주력 (조건부자율) ★미설계 핵심
- **정의:** LLM이 **생성을 주도**하고 공식은 안전 바닥(비퇴행)만 감시. 본 문서가 채워야 할 최우선 빈칸.
- **공식 권한:** 안전 바닥만. ΔW·KL·구조 비퇴행 게이트로 사후 검열, 사전 강제 최소화.
- **LLM 권한:** 장면 생성 + 자기 수정 제안.
- **진입(졸업) 조건:** §3.3 수치 계약 참조 — 실 GPU ΔW 신뢰 라운드 누적 adopt.
- **상태:** ◻ 미설계 → 본 v1의 §3.3에서 1차 계약 제시.

### LLM-2.5 — 자율 평가루프 (Phase G)
- **정의:** AI가 자기 산출물을 스스로 평가·재생성하는 폐루프. B2B SaaS 매출 단계.
- **진입 조건:** 자가평가 판정이 인간 평가와 일치(κ) 임계 이상 ∧ 폐루프 발산 없음.
- **상태:** ◻ Phase G(V876~955).

### LLM-3 — 고도자율 (천장, 개념)
- **정의:** 공식은 최후 안전망만. LLM이 거의 전권으로 생성·자기교정.
- **진입 조건:** 블라인드 인간평가에서 생성 신작이 실명작 대비 **비열위**(통계적으로 지지 않음) + 병리율 상한 유지.
- **주의:** 천장은 정직하게 "모작 수준"으로 명시(ADR-243) — 인간 GT는 최종시험으로 보류. LLM-3 통과 = 인간 작가 대체가 아니라 **인간-in-the-loop 보조의 신뢰 상한**.
- **상태:** ◻ 개념 정의만.

## 3. 진급 게이트 수치 계약 (핵심)

각 전이는 아래 계약을 **전부 AND**로 통과해야 진급한다. 수치는 v1 제안값(보수적). 실측 후 ADR로 캘리브레이션.

### 3.1 L0 → L1 진입
| 항목 | 계약 | 근거/코드 |
|---|---|---|
| 자격 변별력 | 열화 사다리 전 구간 명작 우위 승률 ≥ 0.80 | `critic_qualification.WIN_MIN` |
| 단조성 | 열화 강도↑ → 명작 승률↑ 단조 증가 | ADR-243 |
| RAG 강제 | rag_refs 없는 호출 0건(거부) | G_LLM1_RAG |
| 비용 | 월 ≤ $30(soft) 유지, $50(hard) 미초과 | G_LLM1_COST |

### 3.2 L1 자기개선 루프 단일 라운드 채택(adopt)
| 조건 | 계약 | 코드 |
|---|---|---|
| c1 승률 상승 | ΔW = W₁ − W₀ > 0 | `g_loopc_winrate` |
| c2 붕괴 방지 | KL(학습‖기준) ≤ τ = 0.50 /token | `TAU_KL_DEFAULT` |
| c3 구조 비퇴행 | R_after ≥ R_before − tol(0.0) (R_struct ∧ R_pair ∧ R_path) | `structure_conformance` |
| 신뢰 | n_pairs ≥ 50 (미만 시 '조건부 채택'으로 정직 표기) | `MIN_PAIRS_RELIABLE` |
| **길이 비편향** | per-token 정규화 승률 사용. "짧은 쪽=승자" 단순규칙 재현율과 관측 승률 분리 보고 | `pertoken_winrate`(V788) |

> ⚠️ **V788 발견:** 어제 Round#2의 W는 길이 confound 위험이 컸다(held 56쌍 draft 593자 vs ref 424자, "짧은 쪽 승" 단순규칙이 라벨 95.5% 재현). 따라서 L1 졸업 판정은 반드시 per-token 정규화 승률로 한다 — sumlogP 원점수 금지.

### 3.3 L1 → L2 졸업 (★본 v1의 핵심 신규 계약)
단일 라운드 adopt는 노이즈에 약하다. L2 진입은 **누적 신뢰**를 요구한다.
| 항목 | 계약(v1 제안) | 비고 |
|---|---|---|
| 신뢰 라운드 수 | adopt 라운드 ≥ 5회 연속(롤백 없이) | 단일 라운드 ΔW 노이즈 차단 |
| 누적 표본 | Σ n_pairs ≥ 250 | MIN_PAIRS_RELIABLE×5 |
| 승률 신뢰구간 | per-token W₁의 95% CI 하한 > 0.5 | 우연 초과 |
| 길이 무편향 | 길이 단순규칙 재현율 ≤ 0.60 (실력 신호가 길이를 압도) | V788 진단 도구 |
| 구조 비퇴행 | 전 라운드 c3 PASS(R_path 병리율 비증가) | distribution_guard |
| 비용 | 누적 비용 게이트 녹색 | G_LLM1_COST |

### 3.4 L2 → L2.5 / L3 졸업 (개념 계약, 실측 전 캘리브레이션 대상)
| 전이 | 계약(개념) |
|---|---|
| L2→L2.5 | 자가평가 winner와 인간 라벨 일치 Cohen's κ ≥ 0.6 ∧ 폐루프 N회 재생성 발산(품질 단조 하락) 없음 |
| L2.5→L3 | 블라인드 인간평가: 생성 신작 vs 실명작 쌍대에서 신작 승률 95% CI 하한 ≥ 0.45(비열위) ∧ 병리율(R_path) 상한 유지 |

## 5. 신규 ADR 제안 (본 설계도에서 파생)

- **ADR-LADDER-1:** 자율성 사다리 6단계(0/1/1.5/2/2.5/3) 공식 정의 + 바닥-불변식(G_NO_ABSOLUTE_REWARD 전 레벨 유지).
- **ADR-LADDER-2:** L1→L2 누적 신뢰 게이트(G_LADDER_PROMOTE) — §3.3 수치 계약 코드화. 신규 모듈 `learning/promotion_gate.py`.
- **ADR-LADDER-3:** per-token 승률을 모든 ΔW 판정의 표준 입력으로 의무화(sumlogP 원점수 금지).
- **ADR-LADDER-4:** L2 생성 오케스트레이터의 안전 바닥 사양(사전 강제 최소·사후 비퇴행 검열).
- **ADR-LADDER-5:** L3 블라인드 인간평가 하니스 + 비열위 통계 기준.

## 7. 자기 점검 — 논리적 약점 (정직 표기)

1. **수치 계약은 미실측 제안값이다.** §3.3/§3.4 임계(라운드≥5, κ≥0.6, CI하한≥0.45)는 실 GPU ΔW 라운드와 인간평가 표본 없이 확정 불가 → 1차 실측 후 ADR로 캘리브레이션 필수.
2. **L1조차 졸업 미검증.** 실 GPU ΔW 1라운드도 아직(RunPod 키 차단). 즉 L1→L2 게이트는 "설계는 됐으나 한 번도 돌려보지 못함".
3. **길이 외 confound 잔존 가능.** per-token이 길이는 잡지만 어휘 빈도·장르 편향 등 다른 교란은 별도 진단 필요.
4. **κ·비열위 기준의 인간 표본 비용.** L2.5/L3 게이트는 인간 라벨러 비용·일정이 큼 → 단계적 소표본 파일럿 권고.
