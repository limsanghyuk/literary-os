# DESIGN-BROAD-MEASUREMENT-v1 — 좁은 졸업 → 넓은 실측 방향 설계

- 상태: 제안(PROPOSAL · 누적) · 2026-06-24 · 기준 허브 main `ca2fee6` / v14.0.0 (Phase E Exit)
- 성격: 결정 문서 아님. 실험 설계 초안 — 의견 수렴용.
- 연계: ADR-249(졸업 정직경계), DESIGN-ROADMAP-REANCHOR-v1 §5(졸업계약)·§8(약점), DESIGN-PHASE-F-LLM15-v1, project_pathB_hardsignal_generator(메모리)
- 원칙: 미선언 금지. 졸업은 "진짜지만 좁은 승리"임을 출발점으로 한다.

---

## 0. 문제 (한 줄)

v14.0.0 졸업은 **show/tell 한 축, per-token W 하나**만 실측한 "진짜지만 좁은" 승리다(ADR-249가 스스로 인정). LLM-1.5→2로 가려면 **무엇을 더 넓게 측정할지**를 설계하고 실험해야 한다. 본 문서가 그 측정 축을 못 박는다.

---

## 1. 졸업이 측정한 것 / 못 한 것 (정직한 경계)

| 구분 | 내용 |
|---|---|
| **측정함** | per-token W(길이중립 승률)로 show↔tell craft 한 칸. 5R 단조상승 0.580→0.808, 5/5 adopt |
| **못 함 (1)** | **거시 일관성** — 복선→회수 정합, 4막 균형, 인물 톤 16부 유지. 현재 *공식이 검사*만, 학습/승부로 측정 안 됨 |
| **못 함 (2)** | **다축 craft** — 대사 force·감정선·장르 적합을 동시에. 한 축 쏠림 미방지 |
| **못 함 (3)** | **인간 명작 직접 대조** — 현재 rejected=AI평이체. 실명작 vs 생성작 블라인드는 미실시(천장 시험) |
| **구조 결함** | floor는 "측정 쉬운 축으로 쏠리는 방향"을 못 막음(ADR-249 명시). 한 축만 잘하는 가짜 향상 차단 장치 부재 |

## 2. 넓은 실측 후보 — Tree of Thought (3안)

| 안 | 측정 대상 | 장점 | 단점 | 리스크 | 비용 |
|---|---|---|---|---|---|
| **A. 거시 일관성 실측** | 복선→회수 정합·4막 균형·인물 톤 유지 across 16부 | 가장 큰 빈칸 직격, 기존 기관(PayoffScheduler·CausalPlotGraph) 재사용 | 자동 측정 어려움, 장편 GT 필요 | 측정기 자체가 휴리스틱이면 순환논증 | 중 |
| **B. 다축 craft 블라인드 평가** | 대사 force·감정선·장르적합 동시 + 축 균형 | 축 쏠림 방지, per-token W 확장 | 인간평가 비용·일관성 | 축 정의 모호 | 중~고 |
| **C. 인간 명작 직접 대조** | 생성작 vs 실명작 블라인드(현 rejected 교체) | 최종시험 본체 | LLM-2 미도달이라 조기엔 전패만 | 신호 0(전패=학습불가) | 고 |

**최비합리 제거 → C 단독 선행.** 이유: 현재 LLM-1.5 수준에서 실명작 직접 대조는 *전패만* 나온다 = per-token dW가 항상 음수 = 학습 신호 0. C는 **천장 시험**으로 보존하되(LLM-3 졸업 정의 = 블라인드 인간평가 비열위, REANCHOR §5), 지금 굴리면 비용만 태우고 신호 없음.

**채택 → A + B 병행 + 메타게이트 1.** A·B는 상보적이다: A=거시(구조), B=미시(craft 다축). 둘 위에 **axis-balance 메타게이트**를 얹어 "한 축만 올리고 다른 축 방치"를 차단한다. 이것이 ADR-249가 지적한 구조 결함의 직접 해법.

## 3. 채택안 상세 설계

### 3.1 축 A — 거시 일관성 실측 (3개 하위지표)

| 지표 | 정의 | 측정 방법 | 기존 기관 재사용 |
|---|---|---|---|
| **A1 복선 정합** | 심은 복선이 약속된 화에 회수되는 비율 | PayoffSchedule.residue_ids ∩ 실제 회수 노드 / 전체 | PayoffScheduler(예산 원장 = GT) |
| **A2 막 균형** | 생성작 4막 비율 vs 이상비율 [0.25,0.35,0.25,0.15] L2거리 | CausalPlotGraph.validate_act_structure() 확장 | CausalPlotGraph |
| **A3 인물 톤 유지** | 회차별 인물 발화 문체벡터의 앵커 대비 드리프트 | VoiceManifold.cosine_distance() 16화 누적 | VoiceManifold(단 13D 추출기 선결 필요) |

- **승부쌍 구성**: chosen=일관성 높은 장편 / rejected=복선 미회수·막 붕괴·톤 이탈을 *의도 주입*한 변형본. (Path B 방식의 거시판). 
- **선결물**: A3은 VoiceManifold의 13차원 특징추출기가 부재(2.2 기준선) → **추출기 구현이 A 착수 전제**.

### 3.2 축 B — 다축 craft 블라인드 평가 (per-token W 확장)

- 현 per-token W(show/tell 1축)를 **4축으로 확장**: 대사 force·감정선·장르적합·서술 craft.
- 각 축 독립 win-rate W_i 측정. 기존 critic 5축(`critic/llm_critics.py`, 이미 LLM)을 평가자로 재사용.
- **블라인드 프로토콜**: 평가자에 작품 출처(생성/명작/변형) 비공개. AI-judge-AI 편향(06-20 실측: 심사위원이 안티LLM↔인간 구별 못함) 통제 위해 **인간 GT 샘플 교차검증**(소량) 동반.

### 3.3 메타게이트 — axis-balance (핵심 신규)

```
졸업/적재 조건 확장:
  기존:  per-token W₁ CI하한 > 0.5  (1축)
  신규:  ∀ 축 i ∈ {A1,A2,A3,B1..B4}:  W_i CI하한 > τ_i
         ∧  min_i(W_i) ≥ mean_i(W_i) − δ      ← 축 균형 강제
         ∧  Δ(max_i − min_i) ≤ ε              ← 쏠림 상한
```

- **의미**: 한 축만 0.9이고 나머지 0.5인 "가짜 향상"을 `min ≥ mean−δ`가 차단. ε는 축 간 격차 상한.
- **양성축(positive axis) 추가**: 현재 floor는 "하지 마라(감정어 금지 등)"의 음성 제약 위주 → 양성축(있어야 좋은 것: 복선 회수율·톤 유지)을 게이트에 넣어 측정 쉬운 축 쏠림을 구조적으로 상쇄.

## 4. 실험 순서 (무GPU 가능 범위 우선)

1. **(무GPU) A1·A2 측정기 구현** — PayoffScheduler·CausalPlotGraph 출력에서 정합/균형 산출. 기존 결정론 기관 위 얇은 집계층. → 회사세션 가능.
2. **(무GPU) 변형본 생성기** — Path B 방식으로 거시 결함 주입 변형(복선 끊기·막 비율 붕괴). chosen/rejected 쌍 빌드.
3. **(무GPU) 메타게이트 단위검증** — 합성 W 벡터로 `min≥mean−δ`·`ε` 불변식 단위테스트.
4. **(선결) VoiceManifold 13D 추출기** → A3 활성화.
5. **(집 GPU) B축 4-craft 블라인드 1라운드** — critic 5축 평가자, 인간 GT 소량 교차.
6. **(집 GPU) A+B 통합 메타게이트 1라운드** — axis-balance 적용 adopt/rollback 실측.

## 5. 결과·다음

- **산출**: 넓은 실측 3축 설계(A 거시·B 다축·메타게이트) + 무GPU 우선 실험순서 6단계.
- **즉시 착수 가능**: 1~3단계는 GPU·과금 0(기존 결정론 기관 재사용).
- **선결 의존**: A3 = VoiceManifold 13D 추출기(현재 부재).
- **천장 보존**: C(인간 명작 대조)는 LLM-3 졸업 시험으로 유보.

### 자기점검 (논리적 약점)
1. **순환논증 위험**: A의 측정기가 휴리스틱이면 "공식으로 공식을 채점"하는 꼴. → A1·A2는 PayoffSchedule.residue_ids 같은 *원장(ground truth)* 기반이라 비교적 안전하나, A3 드리프트 임계(0.15/0.30)는 동결 계수 = 검증 필요.
2. **B축 정의 모호**: "대사 force·감정선" 축의 조작적 정의가 critic 5축에 위임됨 → critic 자체 신뢰도가 상한.
3. **메타게이트 δ·ε·τ 미실측**: 잠정 파라미터. 1라운드 캘리브레이션 전까지 임계 미확정(미선언).
4. **인간 GT 비용**: B의 교차검증 샘플 규모가 작으면 AI-judge-AI 편향 통제 불충분 가능.
