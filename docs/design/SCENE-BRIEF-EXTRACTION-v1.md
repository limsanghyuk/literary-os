# 씬별 소제목·설명(SceneBrief) 추출 — 검토안 + 파일럿 (v1, 2026-06-29)

- 성격: 검토(GO/NO-GO 판단용) + 실측 파일럿. 누적→수렴→본안 워크플로.
- 질문(사용자): 씬별 소제목+설명이 없으면 만들어야 하지 않나. → **검토 결론: GO.** 파일럿으로 가능·저비용·무GPU 확인.
- 연계: GENERATION_BODY_L4(SceneBrief 스키마)·DESIGN-PLANNER-LEARNING-CURRICULUM(loop-P 라벨)·DESIGN-LLM2-SYNOPSIS-ASSEMBLER(L5 씬)

## 0. 한 줄
각 씬 본문 → {소제목·설명·극적기능·인물}로 추상한 **SceneBrief** 층을 코퍼스에 추가한다. 이는 ①brief→산문 생성학습쌍 ②회차별 brief 배열=마이크로플롯 설계도("왜 이 구성인가") ③loop-P 극적기능·인물 라벨 — 세 용도를 한 번에 채운다.

## 1. 왜 GO인가 (파일럿 실측, 대장금 1화 14씬)
1콜로 14씬 brief 생성. 읽으면 회차 구성이 그대로 보임:
```
#1 [갈등촉발] 첫 시합의 실패 — 서천수가 활을 쏘나 화살이 떨어지고 손을 다친다
#2 [정보공개] 꿈에서 깨어나다 — 천수가 악몽에서 깨고 망돌이 깨운다
#4 [복선심기] 정치적 긴장 — 이세좌·종사관이 폐비 문제를 논의
#6~9 [복선심기] 수랏간 긴장·기미상궁 의심·별감의 약재
#14 [반전] 갑작스러운 침입 — 나인들이 박나인을 제압하고 노리개를 빼앗는다
```
→ 슬러그(heading)가 못 주던 **극적 기능·인과 흐름**을 brief가 포착. 설계도로 기능함이 실증됨.

## 2. 스키마 (SceneBrief, 기존 7-pass 정합)
```
{ scene_id, scene_no,
  소제목,            # 6~14자, 그 씬의 사건/전환 압축(장소슬러그 아님)
  설명,              # 1~2문장: 누가 무엇을 하고 결과/정보가 무엇인가
  극적기능,          # 갈등촉발|상승|반전|해소|복선심기|복선회수|관계전환|정보공개|잔향
  주요인물:[..],
  (v2) 소속군, 메인-서브 교차, payoff_id }   # loop-P/AEG 라벨과 합류
```
heading(소제목 슬러그)·text(본문)는 그대로 두고 **brief 필드만 추가**(하위호환).

## 3. 접근 ToT (3안)
| 안 | 방법 | 장점 | 단점/리스크 | 판정 |
|---|---|---|---|---|
| **A. LLM 추출** | 본문→brief(gpt-4o-mini 등) | 의미 포착, 파일럿 검증됨, 무GPU·저비용 | 환각·일관성·요약 충실성 | **채택** |
| B. 룰/feature | heading+conflict_intensity 등으로 기능 추정 | 0원·결정론 | 소제목/설명 의미는 못 만듦 | 보조(기능 라벨 교차검증용) |
| C. 하이브리드 | 메타(슬러그·인물·기능후보)=룰 + 소제목·설명=LLM | 비용↓·검증 가능 | 배선 복잡 | 부분 채택(v2) |

## 4. 비용·스케일 (실측 기반 추정)
- 입도: 회당 50~90씬(파일럿 확인). 전 코퍼스 ≈ 134,337씬(manifest), 드라마분 ≈ 10만+.
- 배치 ~14씬/콜 → 드라마 전수 ≈ 7~9천 콜. gpt-4o-mini 기준 **대략 $10~30 · 무인 수 시간**(과거 생성 배치와 동급). 증분·재개형으로 설계(중단 안전).
- **티어링**: Tier-1 = 드라마 129편 우선(영화 후순위). 더 좁게는 명작 20편 먼저 → 형식·기능 라벨 합의 후 전수.

## 5. 가치 (세 용도)
1. **생성 학습쌍**: (소제목+설명) → 씬 산문 = GenerativePort/loop-C용 정조준 학습 데이터(현 rejected=AI평이체 한계 보완).
2. **마이크로플롯 설계도**: 회차별 brief 배열 = "왜 이 회가 이렇게 구성됐는지"의 가독 설계도. SceneBrief가 곧 L5.
3. **loop-P 라벨**: 극적기능·인물·(v2)소속/교차가 PlannerPort/SubplotAllocator의 GT 라벨.

## 6. 위험·자기비판
- LLM 요약 **충실성**: 본문에 없는 내용 추가 위험 → 프롬프트에 "본문 근거만" 강제 + 표본 인간 점검.
- **극적기능 라벨 노이즈**: 9분류 경계 모호 → v1은 분포만 신뢰, 정밀 라벨은 교차검증(룰 feature 대조).
- **비용 누적**: 전수 전 Tier-1로 ROI 확인.
- 본문 절단(420자)로 긴 씬 정보 손실 → 긴 씬은 분할 요약.
