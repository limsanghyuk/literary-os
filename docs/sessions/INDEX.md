## 3. 보완 사항 (정리 점검 결과)
체계 정리를 위해 권고하는 개선 (실제 이동·삭제는 개발 모드에서 수행):

1. **빌드 산출물 분리**: `blueprint_v3.js` `proposal_v3.js`는 문서가 아닌 생성 스크립트 → `tools/` 또는 `build/`로 이동 권고. docs/sessions에 부적합.
2. **버전 스프롤 정리**: 동일 주제에 v1·v2·v3·final·final_v2 공존 → §1 정본만 남기고 구버전은 `docs/sessions/_archive/`로 격리 권고(삭제 아님).
3. **Preflight 로그 26개 분리**: `docs/sessions/preflight/` 하위로 이동하면 기획 문서 가독성↑.
4. **하위 폴더 구조(선택)**: `phase_a/ phase_b/ ... phase_e/ experiments/ preflight/`로 재배치 시 80+ 평면 목록 해소. 단 기존 링크 영향 주의.
5. **본 INDEX 유지**: 신규 문서 push 시 §1 정본 표 + §2 카탈로그 갱신.
