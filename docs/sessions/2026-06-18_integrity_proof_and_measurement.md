## B. 신규 데이터 실측

### 실측 0 — ★생성 vs 실명작 승률 (gpt-4o-mini, 신규 명작 풀)
사용자 제공 OpenAI 키로 `loop_c_dpo.py` 실행. 레퍼런스 풀에 **신규 명작 추가**(도깨비·대장금·비밀의숲·시그널·하얀거탑·불한당·마왕·선덕여왕·추적자·밀회 + 기존 살인의추억·올드보이·신세계·국제시장 등) = **729씬**. gpt-4o-mini가 씬 생성 → 3페르소나 패널 블라인드(위치스왑) → 생성 draft vs 실명작 씬 쌍대 판정.

| | 결과 |
|---|---|
| 총 대결 | **24쌍** |
| **생성(draft) 승** | **11 (46%)** |
| **실명작(ref) 승** | **13 (54%)** |

작품별: 국제시장 2/2·살인의추억 2/3·신세계 1/3 (실명작 우세). 장르별 생성 승률: crime 3/4, thriller **4/11(36%)**, melo 2/4, romance 2/5.

**해석**: 현 LLM-0 생성은 **실 한국명작과 막상막하(46% vs 54%) — 아직 못 이김**. 특히 스릴러(살인의추억·신세계·비밀의숲)에서 36%로 약함. 이것이 **loop-C 학습의 명확한 목표치**: 신규 명작 풀이 강해질수록 승률이 100%(열화 레퍼런스)→~50%(실명작)로 떨어지는 것 = **진짜 시험이 작동**한다는 실증. 직전 세션 2/4(50%)와 일관. 한계: n=24 소표본·단일라운드 노이즈 → N라운드 이동평균 필요.

### 실측 1 — 임베딩 검색 정합성 (신규 425작품)
작품별 씬 임베딩 평균 → 작품 벡터 → 코사인 최근접 탐색.
- 표본: 제빵왕김탁구_15 → top5 전원 제빵왕 타 회차(5/5), 킬미힐미_10 → 전원 킬미힐미(5/5), 힐러_09 → 전원 힐러(5/5).
- **집계: 신규 425작품 중 최근접 1위가 동일 작품 다른 회차 = 423편 (99.5%)**.
- 의미: 임베딩 공간이 작품 정체성을 정확히 포착 → **RAG/loop-C 레퍼런스 검색이 신규 데이터에서 정상 작동** 실증.
