# 실 모델 6지표 — OpenAI 자체평가 + 실 DPO 1라운드 (2026-06-18)

기준 v13.39.0 (HEAD 53a5408c) · 실행: 샌드박스 · LLM=gpt-4o-mini(실 클라우드) · 학습=tiny-gpt2 CPU 실 DPO

## 실행 방식 (Mock 아님)
- M1/M2의 **생성·심판을 실 OpenAI(gpt-4o-mini)** 로 수행(자체평가 닻 작동).
- 그 선호쌍으로 **실 DPO 학습**(tiny-gpt2, CPU) → W0→W1 측정 → G_LOOPC_WINRATE 수용판정.
- ※GPU 클라우드(RunPod) 키 미제공 → 가중치 학습은 CPU 소형 모델. LLM만 실 클라우드.

## 6지표 (실측)
| # | 지표 | 값 |
|---|---|---|
| 1 | M1 Critic 자격승률 (명작>열화 판별) | **1.0 · 통과** |
| 2 | M2 필적률 (생성 vs 은닉GT 쌍대) | 1.0 (채점 3·제외 0) |
| 3 | M3 분포 병리 감점 | 3 |
| 4 | W0 baseline 선호정확도 | 0.0 |
| 5 | W1 학습후 / ΔW | 0.0 / **+0.0** |
| 6 | G_LOOPC_WINRATE 수용판정 | **rollback** (ΔW≤0) |

## 정직한 해석 (중요)
- **M1=1.0 통과**: gpt-4o-mini가 명작>열화판을 정확히 가름 → 자체평가 심판 자격 확보(진짜 성과).
- **M2=1.0은 과대평가**: 닻이 *짧은 데모 명작풍 스니펫*(실 corpus_ko 455편 아님)이라 LLM 생성이 닻을 이김. 실 명작 코퍼스 닻이면 필적률 훨씬 낮아짐(학습여지 정상화).
- **W0=W1=0 → rollback**: 3쌍·tiny-gpt2·길이 교란(긴 생성 chosen이 logprob합 불리)으로 선호정확도 미이동. **게이트가 "향상 없음 → 채택 거부(rollback)"를 정확히 판정** = 시스템이 비향상을 정직하게 거른다는 증거.

## 결론
**자체평가→loop-C 사슬이 실 클라우드 LLM으로 처음 끝까지 돌았고, 6지표가 모두 실측됐다.** 메커니즘(M1 자격·M2 쌍대·M3 가드·수용게이트)은 작동하나, 이번 라운드는 **소규모·소형모델·약한 닻**으로 ΔW=0 → 게이트가 올바르게 rollback. 실제 ΔW를 움직이려면 **실 생성기(3B/8B)+실 corpus 닻+수백 쌍**이 필요 = 다음 단계(개발자 4070 로컬 / RunPod 클라우드 GPU).
