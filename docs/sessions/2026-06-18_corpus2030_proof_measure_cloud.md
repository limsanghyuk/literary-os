## 4. 개발자가 로컬에 DB를 보유하는 의미

우리 아키텍처 = **"판단은 로컬, 생성만 LLM, 학습은 누적"**. 로컬 DB는 단순 백업이 아니라 **시스템의 판단 기반(judgment substrate)이자 핵심 자산**이다.

1. **LLM-0 주권·저작권 방어**: 명작 corpus는 verbatim 저작물 → 외부 LLM/공개 클라우드에 절대 올리지 않음. 로컬 보유가 저작권 방어선이자 LLM-0 경계(corpus/constitution은 외부 금지)의 물리적 구현.
2. **결정성·투명성**: RAG 검색·공식 특징·NKG 일관성 판정이 전부 로컬에서 결정적으로 계산됨. 외부 LLM은 *비-verbatim 컨텍스트로 생성만* 호출. 같은 입력=같은 판단(재현성).
3. **자산·해자(moat)**: 누적된 2,030작품 tri-store가 loop-C 학습 선호쌍·RAG 레퍼런스·NextEpisodeBench 은닉GT의 원천. 데이터가 곧 경쟁력.
4. **비용·지연**: 판단마다 클라우드 호출 불필요 → per-query 비용 0, 지연 최소.
5. **제품 배치의 축소판**: Phase G 하이브리드 SaaS = **로컬/온프렘 = literary-os 코어(공식·NKG·일관성·critic·UI·프로젝트상태) / 클라우드 = 생성기·패널 GPU**. 지금 로컬 DB 구조가 그대로 제품 경계가 됨.

→ **요약**: 로컬 DB = 저작권 안전 + 결정적 판단 + 핵심 자산. 클라우드는 *보조*(생성·대규모 학습·백업)이지 판단·코퍼스의 주체가 아니다.

## 5. 신규 대본 지속 수집·저장을 위한 클라우드 아키텍처

**요구**: 개발자가 새 드라마 대본을 계속 조사·확보 → 우리 DB에 지속 편입·저장. **불변 제약**: verbatim 원문은 비공개·암호화만(공개 저장소·공개 데이터셋 절대 금지). **현 규모**: 2,030작품 / 210k 벡터(1536d) / emb_cache ~1.3GB / 원문 수 GB = **작음**.

### 5-1. 4가지 전략 비교 (장점·단점·리스크·비용)

| 전략 | 구조 | 장점 | 단점·리스크 | 월비용(현 규모) |
|---|---|---|---|---|
| **A. 로컬권위 + 비공개 객체저장 동기화** (권고) | 권위 DB=로컬. 원문·emb shard를 암호화해 B2/R2에 백업·회사↔집 동기화. ChromaDB는 로컬 재빌드 | 저작권 최안전·LLM-0 주권 유지·초저가·락인 없음 | 상시 클라우드 쿼리 없음·동기화 수작업성 | **~$0–1** (B2 $0.006/GB·R2 egress 0) |
| **B. 매니지드 벡터DB(Qdrant/Pinecone)를 라이브 스토어** | 벡터+id만 매니지드 DB 업서트, 원문은 비공개 객체저장 | 상시·다기기·확장 쉬움 | **payload에 verbatim 금지**(텍스트는 로컬). 주권 일부 약화·락인 | **$0–60** (Qdrant 무료 1GB/Pinecone 무료 2GB에 210k 적합) |
| **C. 자체호스팅 벡터DB(소형 VPS/컨테이너)** | $5–12 VPS에 Qdrant/Chroma + 암호화 볼륨 + 비공개 객체저장 | 통제+상시+소유(임시인스턴스 방어논리 적용) | 운영부담·암호화 at-rest 필수 | **$5–15** |
| ~~D. 공개 데이터셋 호스팅(HF public/공개 S3)~~ | — | — | **저작권 위반** | 제거 |

**제거**: D(공개 호스팅) = 저작권 위반으로 즉시 탈락.

### 5-2. 권고 (최적안 + 이유)
**기본 = 전략 A**(로컬 권위 + 비공개·암호화 객체저장). 이유: 우리 규모가 작아 매니지드 DB의 상시성 이점이 비용·주권 리스크를 정당화하지 못함. 저작권이 1순위 제약이고, 이미 V786 `CloudStore/PresignedHttpStore`(암호화 업로드)·ProviderRouter R2(민감 코퍼스→LOCAL)가 이 설계와 정합.
- **객체저장 선택**: 비용 최소=Backblaze B2($0.006/GB)+Cloudflare 무료 egress, 또는 egress 완전무료가 필요하면 Cloudflare R2($0.015/GB). 현 규모에선 둘 다 월 $1 미만.
- **선택적 add-on = 전략 B**: 상시·다기기 RAG 쿼리(Phase G SaaS)가 필요해지면, **벡터+id+메타만**(씬 텍스트 payload 금지) Qdrant 무료 티어에 업서트. 텍스트는 끝까지 로컬/비공개 객체저장.

### 5-3. 지속 편입 파이프라인 (HOW, 체크리스트)
```
신규 대본 확보(개발자)
  └─① 로컬 변환·파싱·청킹 (convert/parse 멱등)
  └─② 임베딩(OpenAI) → append-only emb_cache shard
  └─③ 원문 + 신규 shard 암호화 → 비공개 버킷(B2/R2) 버전 업로드   ← 저작권 안전(비공개·암호화)
  └─④ 로컬 재빌드: store_chroma.py(ChromaDB) + features.py + nkg.py (무API·멱등)
  └─⑤ 매니페스트(작품수·청크수·sha256)만 허브 push (verbatim 금지)
  └─(선택) 벡터+id를 매니지드 Qdrant 업서트(텍스트 payload 없이) — 상시 쿼리용
```
- [ ] 버킷은 **private + 서버측/클라이언트측 암호화**(Fernet/age 권장; 코드 기본 SHA256-CTR)
- [ ] 동기화 키(B2 app key/R2 token)는 env·시크릿 매니저, 코드·허브 커밋 금지
- [ ] 회사↔집 동기화: `rclone`(B2/R2 지원)로 emb_cache·scenes 단방향 미러
- [ ] 허브엔 **집계만**(쌍 수·승률·sha256). 원문·dpo_pairs verbatim 금지
- [ ] Drive `corpus_ko.zip`은 인코딩 교정 전 stale 가능 → 교정본으로 재동기화(모지바케 재발 방지)
