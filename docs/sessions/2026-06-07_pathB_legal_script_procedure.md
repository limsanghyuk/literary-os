# 경로 B — 합법 스크립트 기반 데이터 구축 절차 (Gold 깊이)

정확도 높은 Gold 20~30편 L2/L3를 합법 스크립트로 구축. 핵심: **원문은 분석 입력일 뿐, 저장은 구조·피처만(verbatim=false).**

## 1. 합법 열람 경로 (개발자 수행)
- KOFIC **시나리오마켓**(회원 가입→원문 열람/다운로드, 저작권 추적 동의) — 주로 영화·개발 시나리오.
- **한국영상자료원 영상도서관 / KMDB 시나리오·콘티** — 시나리오 검색·열람.
- **출판 대본집**(교보문고 등) 구매 — 드라마 방영작 대본 다수.
- 시나리오 공모 수상 공개작.
- (주의) 작품별 가용성 상이. 드라마 대본은 대본집/KMDB, 영화는 시나리오마켓 우선.

## 2. 권리 준수 게이트 (G_CORPUS_RIGHTS)
- 열람 라이선스·이용약관 준수, 열람 기록 보관.
- **대사 verbatim 저장·재배포 금지.** DB엔 분석 산출물만.
- pre_train_safety로 verbatim 비율 0·PII 0 강제.

## 3. 구조·피처 추출 워크플로
합법 열람 스크립트 → ① 씬 분할 → ② 인물·사건·거시/미시 플롯·복선(plant→payoff) 분석 → ③ 13필드 SceneFeature 정량 → ④ 작가 검수(일치율 게이트) → ⑤ 트라이-스토어 적재(벡터·그래프·피처, verbatim=false).
- 자동화: LLM-1 분석기 1차 + 작가 검수(정량 라벨 캘리브레이션).

## 4. A vs B 분담
- A(공개 구조분석): 폭 200편 L0~L2. 저렴·안전, 정확도 보조.
- B(합법 스크립트): 깊이 Gold 20~30편 L2/L3. 정확·풍부 → 공식 검증 정답셋·계수 학습.

## 5. 단계
1. 시나리오마켓·KMDB·대본집에서 Gold 후보 가용성 조사(장르 균형).
2. 합법 열람 + 권리 게이트 적용.
3. 추출 워크플로 → 트라이-스토어 적재.
4. 공식 검증 하니스(Stage1~)로 타당성 측정.
