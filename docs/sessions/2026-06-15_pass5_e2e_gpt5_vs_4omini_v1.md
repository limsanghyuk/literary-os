# Pass5 E2E 모델 비교 — GPT-5 vs gpt-4o-mini (b) v1.0

**작성일**: 2026-06-15 · **기준선**: main `3f563607` (V754 / v13.8.0) · **문서 ID**: LOS-E2E-GPT5-CMP-V1.0-2026-06-15
**계기**: 개발자 질문(모델 버전) → 가용 모델 확인(gpt-5/5-mini/5-chat-latest 존재, "5.5" 없음) 후 동일 프리미스 '균열'을 GPT-5로 재측정.

## 1. 측정 (동일 프리미스 '균열' 스릴러)
| 모델 | 씬 | Pass6 게이트 | 평균 R | 비용 | 비고 |
|---|---|---|---|---|---|
| gpt-4o-mini | 4 | **4/4 PASS** | **0.571** | ~$0.0005 | 직설적·사건 중심 |
| **gpt-5-chat-latest** | 3 | **3/3 PASS** | **0.518** | $0.0076 (~14×) | 절제·분위기·서브텍스트 중심 |

## 2. 핵심 발견 (정직)
1. **구조 게이트는 둘 다 전건 통과** — Pass6(길이·갈등·콜백)는 모델 무관하게 안정적.
2. **공식 R은 GPT-5가 오히려 낮다(0.518<0.571).** GPT-5 산문이 더 절제·분위기적(예: "검은 강가, 안개가 수면 위를 기어다닌다")인데, **키워드 기반 arc·tension 공식이 그 정교함을 못 잡아** 점수가 더 낮게 나온다. → **공식=구조 프록시(품질 미측정)** 진단의 3번째 독립 확증(longform·loop-B·E2E에 이어). 즉 *공식 R로 모델 품질을 비교하면 안 된다*.
3. **V754 5축 Critic(gpt-4o-mini 심판) = 전 축 TIE.** 두 정황: ⓐ저가 심판이 두 능숙한 스릴러 씬을 변별 못 함, ⓑ응답이 'WINNER:' 형식 미준수 → tie 기본처리(V753 스모크와 동일). → **critic 프롬프트 형식 강제·심판 상향(gpt-5/3페르소나 합의)이 필요**(V755 보강 항목).

## 3. 결론 + 후속
- **모델 비교의 진짜 기준은 공식이 아니다.** 구조 게이트는 통과 여부만, 공식 R은 품질 변별력 없음. 품질 비교는 **강한 critic(형식 강제) + 인간 GT + 실명작 레퍼런스**로만 가능.
- **GPT-5는 더 정교하나 14× 비용** — 생성 주력(Phase G LLM-2.0) 전까지는 4o-mini로 충분, critic 심판에만 선별적 GPT-5 검토.
- 후속(V755): critic 프롬프트 'WINNER:' 형식 강제 + 심판 상향 + 일치율(human_gt) 측정.
