## 1. 실험2 — 블라인드 3모드 평가 결과

씨드: "10년 만에 고향역에 내린 남자가, 마중 나오기로 한 동생이 보이지 않는 플랫폼에 선다."
3모드 생성 후 정답(A=인간문체/B=순수LLM/C=안티LLM)을 가리고 독립 전문작가 에이전트가 채점.

| | A 인간문체 | B 순수LLM | C 안티LLM |
|---|:---:|:---:|:---:|
| 합계 /50 | **43** | 10 | 36 |
| 순위 | 1 | 3 | 2 |
| 감정어 수 | 0 | 9 | 0 |
| 심사위원 정체 추정 | 인간 가능성 最高 | AI 가능성 最高 | **판별 불가(유보)** |

**판정:** 심사위원이 **안티LLM(C)을 인간과 구별하지 못함** → 개발자가 message에서 예측한 "AI 평가로는 안티LLM 목업이 인간급으로 나온다"를 그대로 재현. 순수LLM(B, 감정어 9·telling)은 만장일치 AI 적발·꼴찌.

### 1.1 이전 실험(§9 8쌍 목업)과의 차이 — 핵심 교훈

| | 실험1 (8쌍 목업) | 실험2 (3모드 블라인드) |
|---|---|---|
| 방법 | 결정론·모델프리 **지표** | 실 생성 + LLM **정성 판정** |
| 길이통제 후 결론 | per-char 3/8 — "정규화하면 우위 붕괴"(**회의적**) | 안티LLM을 인간급 2위로 **격상**(열광적) |
| 정체 | 정직한 측정자 | **§9에서 보조로 강등한 그 AI-judge-AI** |

**두 실험의 결론이 어긋난다는 것 자체가 데이터다.** 모델프리 정직지표는 깎아내렸고(per-char 3/8) AI 심사위원은 띄웠다. 이 **격차 = AI-judge-AI 순환편향의 크기**. 생성자=평가자 동일계열이라 자기 문체를 자기가 후평가. → 실험2는 "안티LLM이 좋다"의 증거가 **아니라**, 확정 판정을 **per-token held-out + 인간 블라인드**로 못 박은 §9 결정4의 정당성 실연이다. 수렴점: 순수LLM telling은 두 실험 모두 처벌(신뢰 가능).

## 2. 발견 — 구조계층 실측 분포 부재 (코드베이스 실조사)

개발자 질문("작품당 거시플롯 평균·화당 미시플롯·화당 씬 수를 여러 각도로 학습하나")을 코드로 검증한 결과:

| 항목 | 상태 | 근거 |
|---|---|---|
| 계층 스키마(씬→미시→거시→화→작품) | ✅ 정의 | 씬마다 macro_phase·subplot·beat 라벨 |
| 씬 분절 | ✅ | corpus_ko 122,681씬 |
| 작품당 거시플롯 평균 수 | ❌ 미측정 | macro_arc_compiler 0.25·0.7 **규칙분할** |
| 화당 미시플롯(K) 수 | ❌ 미측정 | episode_planner `BASE_K=4, K∈[2,8]` **상수** |
| 화당 평균 씬 수 | ❌ 미측정 | `BASE_SCENES_PER_MICROPLOT=5` **상수** |
| 60분 막구조 | ⚠️ 관례값 | 콜드오픈 2~4분·3막 54분 = 관례 |

**결론:** 거시·미시·씬 수는 코퍼스에서 **센 값이 아니라 손으로 정한 휴리스틱 상수**다. 또한 현재 학습(DPO/per-token)은 **씬·페어 단위 한 축뿐** — 작품 거시아크·화 미시구성에는 학습 신호가 흐르지 않음(= P4 광역아크 빈칸과 동일 구멍).

## 3. 신규 과제 등록 — 코퍼스 구조 실측 추출 패스 (P2.5, GPU 불요)

씬에 macro_phase·subplot 라벨이 이미 있으니 **집계만** 하면 됨:
1. 작품당 고유 macro_phase 수 → 거시플롯 분포(평균·장르별)
2. 화경계 × 고유 subplot 수 → 화당 미시플롯 분포
3. 화당 씬 수 분포 → `BASE_SCENES_PER_MICROPLOT=5` 실증 검증
4. **선결: 라벨 커버리지 점검** — 2,030 코퍼스 전체에 라벨 신뢰도 있게 붙었는지 먼저 확인

**산출:** 현재 휴리스틱 상수(K=4·씬=5·막 0.25/0.7)를 **실측값으로 교체** → P0 페어링·P4 플래너 양쪽 입력.
**차단점:** 이 허브 클론엔 5작품 시드뿐, 2,030 corpus_ko 실데이터는 집/회사 머신 → 스크립트는 작성·시드검증 후, 실행은 코퍼스 보유 환경에서.

## 4. 우선순위 갱신 (REPORT 대비)

기존 P0~P4 사이에 신규 **P2.5(구조 실측 추출, GPU 불요)** 삽입. P4(광역아크 플래너)의 **선결 입력**이 됨 — 플래너가 쓸 K·씬·막 분포의 실측 근거를 P2.5가 공급. 나머지 우선순위 불변.

## 5. 자기 점검 (논리적 약점)

1. 실험2는 n=1 씨드·1심사위원 → 일화적. 편향 *크기*의 정량화는 아니고 *존재*의 실연. 정량화는 per-token held + 인간 다수 블라인드 필요.
2. "씬마다 macro_phase 라벨이 있다"가 "거시플롯을 분석·학습했다"로 과장될 위험 — 라벨은 시드 5작품 산물이고 2,030 전체 커버리지 미검증. P2.5 1순위가 커버리지 점검인 이유.
3. P2.5 신설이 P3(실 GPU 라운드) 지연 핑계가 되면 안 됨 — P2.5는 P0·P1과 병렬 가능, P3 차단점(RunPod 키)과 무관.
