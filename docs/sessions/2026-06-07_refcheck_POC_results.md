# 레퍼런스 검증 POC 실행 결과 (OpenAI, 미스터 션샤인, 2026-06-07)

Mode 1(생성 vs 레퍼런스) 다중에이전트 블라인드 비교를 OpenAI(gpt-4o-mini 생성 / gpt-4o 평가 3페르소나)로 실제 실행.

## 실행 결과
- 조건: 구한말 신미양요, 소년(유진 초이) 군함 탈출 씬.
- arm B = 모델 생성 / arm A = "실제 씬" (단, **LLM 지식 재구성 = 프록시 레퍼런스**)
- 다중에이전트(문학평론가·드라마투르그·일반시청자) 블라인드 투표: **B 3 : A 0** (B 점수 8, A 6~7)

## 핵심 — 정직한 해석 (방법론적 경고)
표면상 "모델 생성이 실제를 이김". 그러나 **이 결과는 무의미하며 오히려 방법의 함정을 입증**한다:
- arm A는 **실제 김은숙 대본이 아니라 gpt-4o-mini가 기억으로 재구성한 열화 버전**이다.
- 즉 "모델의 새 생성 vs LLM의 열화된 회상"을 비교한 것 → 모델이 이기는 게 당연·무의미. "명작을 이겼다"가 아니다.

## 입증된 것
1. ✅ **다중에이전트 하니스 작동**: 3 에이전트가 일관된 블라인드 판정(3:0) 산출. 인간 GT 대행 메커니즘 동작.
2. ✅ **레퍼런스는 반드시 실제 합법 대본이어야 함을 실증**: LLM 회상 레퍼런스는 열화→비교 무효·오도. Mode 1은 **대본집(합법 원문) 비협상 전제**.
3. → 개발자의 대본집 입수가 검증의 진짜 관문임이 재확인.

## 결론·다음
- Mode 1(생성 vs 실제)·Mode 2(공식 on 실제)는 **실제 대본 확보 시에만 유효**. LLM 회상으론 안 됨.
