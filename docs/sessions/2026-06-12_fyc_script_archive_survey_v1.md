## §3. 추출 SOP (EP9 실증 방법의 표준화)
═══════════════════════════════════════════
1. **기사 접근**: Deadline 기사 페이지 내 임베드 PDF 뷰어(Issuu/Scribd류 아님 — 자체 호스팅 PDF 링크).
2. **추출**: 웹페치는 PDF 미지원 → **Chrome 페이지 컨텍스트에서 pdf.js로 텍스트 추출**(EP9 실증 06-11). 절차: 기사 열기 → PDF URL 획득 → pdf.js `getDocument→getTextContent` 페이지 루프 → 씬 헤딩(INT./EXT.) 기준 분할.
3. **저장**: `Projects/literary/scripts/{작품}_{회차}_official_eng.md` — 씬 번호·헤딩 보존. **허브 비커밋**.
4. **적재**: 트라이스토어 인입(scene_feature DDL) → SG-1~4 프로토콜(쌍대 변별·Δfitness·복선 잔향·시퀀스) 즉시 적용 가능.
5. **기록**: 평가 결과만 `docs/sessions/`에 커밋(원문 비전재). formula_ledger에 매체=screenplay 표기.

## §5. 병목 재평가 — 본 조사가 바꾸는 것과 못 바꾸는 것
═══════════════════════════════════════════
**바꾸는 것**:
1. 구조·복선·시퀀스 축의 데이터 풀: 1편 → **130편**(합법·무료·즉시). Stage 일부(씬 단위 쌍대·복선 잔향·시퀀스)의 N 확장 경로 확보.
2. **시즌 쌍 14작품** — 단일 회차 한계(EP9 §4 "회차 간 복선 불가")를 부분 우회: 동일 작품의 시즌 간 아크·복선 전이를 2점 비교 가능.
3. SG-2 FAIL(각본 포맷 recalibrate)의 보정 데이터: 매체별 캘리브레이션에 필요한 screenplay 표본이 충분해짐.

**못 바꾸는 것(정직하게)**:
1. **한국어 원문 문체 축** — 전부 영문. 전자책 대본집 구매(비밀의숲 리디 확인)는 여전히 개발자 유일 액션.
2. **단일 작품 전 회차**(Stage 3 가동 조건) — FYC는 작품당 1~2편. 회차 간 복선(시즌 아크)은 대본집 또는 시즌 쌍의 제한적 대용.
3. 인간 GT(박빙·문체 축) — 데이터가 아니라 작가 1명의 양자택일 필요.

**결론**: P2 병목 중 '입수' 관문은 **구조 축에 한해 사실상 해소**. 잔여 병목은 ①한국어 원문(개발자) ②전 회차 깊이(대본집) ③인간 GT(작가 섭외)로 좁혀졌다.
