# 로컬 corpus_ko E2E 실측 — "절대 검증 ①" 완료 (2026-06-15)

**계기**: 개발자 트랙이 V752에서 Pass1~7 배선 + Pass5 E2E를 돌렸으나, 그들 샌드박스에 **corpus_ko 실데이터가 없어** Pass4 RAG=스텁, Pass7 레퍼런스=열화-자기로 한계(2026-06-15_pass5_e2e_measurement_v1.md §3에서 "로컬 corpus_ko 연결 시 재측정" 명시). 본 세션이 **그 데이터를 보유** → `e2e_pass5_local.py`를 실 데이터로 실행.

**실행**: `e2e_pass5_local.py` (개발자 작성, corpus_ko 전용) + ChromaDB ko_scenes 32,813벡터 + scenes 395편(실명작 풀) + gpt-4o-mini. 작품 '균열'(thriller) 4씬.

## 결과
```
Pass6 구조게이트: 4/4 PASS
평균 R(LOSConstitution): 0.559
Pass7 (생성 vs 실명작, 3페르소나 블라인드): 생성 승 2/4 (실명작 풀 13,725씬)
```

## 해석 (핵심)
| 비교 | 레퍼런스 | 생성 승률 |
|---|---|---|
| 개발자 샌드박스(06-15) | 열화-자기(쉬움) | **4/4 압승** |
| **본 실측(실데이터)** | **실명작 13,725씬** | **2/4 동률** |

- **레퍼런스를 실명작으로 바꾸자 생성 승률 100%→50%로 하락** — 개발자 문서가 예측한 그대로(열화-자기는 "쉬운 비교"였음). **진짜 시험이 작동**함을 실증.
- 현 LLM-0 생성(gpt-4o-mini + Pass4 실RAG + 구조 조건)은 **실 한국 명작 씬과 막상막하**(아직 명작을 못 이김). 이 격차가 **loop-C(생성기 학습) 목표**.
- Pass4 RAG가 실 ChromaDB(32,813벡터)에서 유사 씬 retrieve, Pass7이 실명작 13,725씬 풀에서 레퍼런스 추출 — **전 파이프라인 실데이터로 가동**.
- 공식 R=0.559: arc·tension 바닥 패턴 4번째 독립 재확인(공식=구조 게이트 설계 재확정).

## 한계(정직)
- n=4 단일 작품·단일 씬 비교라 노이즈 큼(2/4는 통계적으로 약함). 다씬·다작품 확대 필요.
- 실명작 풀은 임의 씬(명작의 *평범한* 씬 포함)이라 "명작 평균"이지 "명작 베스트"가 아님. 그럼에도 4/4→2/4 하락은 명확한 신호.
- 생성 temp=0.7로 실행마다 변동. 반복 측정으로 안정화 필요.
