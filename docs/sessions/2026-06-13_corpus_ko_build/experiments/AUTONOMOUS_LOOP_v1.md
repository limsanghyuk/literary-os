# 자가 학습 루프 골격 설계 v1 (① F-43~F-55 연결, 2026-06-13)

이번 세션에 **실증된 조각만으로** 닫힌 자가 학습 루프를 조립한다. 추가 가정 없이 검증된 부품을 배선한다.

## 검증된 부품 (이번 세션 실측)
| 부품 | 역할 | 실증 |
|---|---|---|
| corpus_ko tri-store | 데이터 기질(컨텍스트·레퍼런스) | 106편·9,744씬·ChromaDB+NKG+SceneFeature |
| **fitness_v2** (conflict_mean·conflict_arc·thematic_complexity·curiosity) | 값싼 1차 sanity baseline | LOO-CV 0.586(등가중≈CV, 과적합 거의 0) |
| **멀티에이전트 패널**(쌍대·레퍼런스 앵커) | 고정밀 보상신호 | 73%(만장일치 82%) |
| 객관 명성 사실(수상·관객수) | 패널 캘리브레이션 닻 | 공개 팩트(작가 불필요) |
| 가중치 학습(쌍대 concordance) | 장기 보정 | 0.458→0.64 일반화 |

## 닫힌 루프 — 3중 중첩 (추론→행동→관찰→재결정)

```
                 ┌─────────────────────────────────────────────┐
   [RAG/NKG]────▶│ ① 생성(행동): LLM-k 초안 (씬/회차)           │
   컨텍스트       │   - ChromaDB 유사 레퍼런스 + NKG 인물/관계 주입│
                 └───────────────┬─────────────────────────────┘
                                 ▼
                 ┌─────────────────────────────────────────────┐
   fitness_v2 ──▶│ ② 1차 추론: 공식 sanity 게이트                │
                 │   - 구조 최소선 통과? conflict_arc 존재?      │
                 │   - FAIL → 즉시 재생성(짧은 루프 A)            │──┐ 짧은루프A
                 └───────────────┬─────────────────────────────┘  │ (생성 재시도)
                                 ▼ PASS                            │
                 ┌─────────────────────────────────────────────┐  │
   실제 대본 ───▶│ ③ 관찰: 멀티에이전트 패널 쌍대 평가           │◀─┘
   (레퍼런스)     │   - 생성물 vs 실제 레퍼런스 씬 (쌍대만)        │
                 │   - 3역할 앙상블, 만장일치=신뢰가중            │
                 │   - 객관 명성으로 주기 캘리브레이션           │
                 └───────────────┬─────────────────────────────┘
                                 ▼ 선호신호(pairwise preference)
          ┌──────────────────────┴───────────────────────┐
          ▼ 중간루프B                                      ▼ 긴루프C
 ┌────────────────────────┐              ┌──────────────────────────────┐
 │ ④-B 공식 재가중         │              │ ④-C 생성기 RLHF/PPO(F-43~55) │
 │  fitness_v2 weights ←   │              │  패널 선호 → 보상모델 → LoRA  │
 │  패널 concordance 최대화 │              │  (G_NO_ABSOLUTE_REWARD: 쌍대) │
 │  (값싸고 빠름, 매 배치)  │              │  (느림, 주기적)               │
 └────────────────────────┘              └──────────────────────────────┘
          └──────────────────► 다음 생성에 반영(재결정) ◄────────────┘
```

## 3중 루프 사양

- **짧은 루프 A — 생성 재시도(추론 즉시)**: fitness_v2 sanity 게이트 FAIL 시 즉시 재생성. 비용 0(공식). 절대점수 아님 — 구조 최소선(예: conflict_arc>0, 씬길이 분포) 충족 여부만.
- **중간 루프 B — 공식 재가중(매 배치)**: 패널 쌍대 선호를 정답으로 fitness_v2 가중치 갱신. EXP-D 메커니즘. 값싸고 일반화 실증됨. 공식이 패널을 점점 잘 근사 → 패널 호출 절감.
- **긴 루프 C — 생성기 학습(주기적)**: 패널 선호쌍 → 보상모델 → 생성기 LoRA/RLHF/PPO(F-43~F-55). 절대보상 금지(쌍대 선호만). 느리지만 생성 품질 자체를 끌어올림.

## 순환성·안전 차단 (설계 불변식)
1. **쌍대만**(G_NO_ABSOLUTE_REWARD) — 절대점수 금지.
2. **실제 대본 앵커** — 패널은 항상 인간이 쓴 레퍼런스와 비교. 생성물끼리만 비교 금지.
3. **생성기≠심판** — 패널 모델군과 생성 모델군 분리(상관오류·자기편애 차단).
4. **객관 사실 캘리브레이션** — 패널 쌍대가 수상·관객수와 어긋나면 패널 교정(주기 검증).
5. **인간 체크포인트** — Gold 셋·고위험 결정은 작가 검수(루프 외부 게이트).

## 단계적 가동 (Phase E~G 매핑)
- **Phase E (지금 가능)**: 짧은루프A + 중간루프B 가동. fitness_v2 게이트 + 패널 재가중. 데이터=corpus_ko. 생성은 LLM-1 초안만.
- **Phase F**: 긴루프C 1차(보상모델+LoRA), 패널 이질화(gpt-4o+claude+gemini), 코퍼스 확장(영화+드라마).
- **Phase G**: 자율 평가루프 상시화, 생성 주력, 작가 개입 최소화.
