# 문학 생성 요소 갭 기획안 v1.0 — 능동 발굴 7요소 + 기존 빈칸 2건 정식화 (2026-06-13)

**기준선**: HEAD `c83b411` · **작성**: 본 모드 — **개발자 미제시 영역의 능동 발굴**(개발자 지시: "내가 생각하지 못한 부분이 있다면 너가 제시하여 기획을 하라")
**원칙 준수**: 모든 요소는 ①기존 모듈과의 차이 명시 ②**측정 가능한 검증**(쌍대·블라인드 정합) 동반 — 검증 불가능한 기획은 제출하지 않음(미검증 부채 방지).
**ID 체계**: LE-1~LE-9 (Literary Element). 공식 번호(F-56+)는 검증 통과 후 등재 시 부여 — 선등재 금지.

## §0. 협업 방식의 전환 (메타 — 1인 개발자 한계에 대한 구조적 답)
═══════════════════════════════════════════
지금까지: 개발자 질문 → 본 모드 답변(수동). 문제: 개발자가 생각하지 못한 요소는 영원히 설계되지 않는다.
**전환**: 본 모드가 **주기적 갭 감사(Gap Audit)**를 능동 수행 — 서사학·창작론의 요소 전집을 기존 공식 인벤토리(F-01~F-55)와 대조 → 빈칸을 측정 가능 형태로 제안 → 개발자는 **우선순위 결정만**. 역할 재확인: 개발자=결정·데이터·취향(GT) / 본 모드=열거·형식화·검증설계. 본 문서가 1회차 감사다.

## §1. 기존 빈칸 2건 정식화 (전 세션 합의 — 필수 개발)
═══════════════════════════════════════════
### LE-1. 사건 다양성·창발 엔진
- **빈칸**: 붕괴 방지 장치는 두텁고, 뻔하지 않게 하는 장치는 Noise 항뿐. LLM은 방치하면 최빈 사건(재회·오해·고백)으로 수렴 — 안티LLM 커버리지 0 실측과 동근원.
- **설계 골자**: ①사건 타입 온톨로지(30~50종) + 코퍼스 빈도 분포 ②씬 브리프 생성 시 **분포 제약**(최근 N씬 사건 타입 엔트로피 하한) ③명작의 사건 전환 행렬(어떤 사건 뒤에 어떤 사건이 오나 — 신품 428씬·FYC로 학습 가능) 대비 생성물의 KL 거리.
- **검증(사전등록 후보)**: 명작 vs LLM 무제약 생성의 사건 엔트로피 차이가 유의한가(명작이 높아야 가설 성립) → 제약 적용 생성이 쌍대 비교에서 무제약 대비 승률 상승하는가.

### LE-2. 인물 창조 알고리즘
- **빈칸**: 인물 간 역학(CIM·SBT)은 정밀하나, 새 인물을 언제·왜·어떤 속성으로 탄생시키나는 미설계.
- **설계 골자**: ①기능 공백 탐지 — 서사 상태 텐서에서 결핍 차원(예: DL 딜레마 부하 만성 저하)을 읽어 "필요한 기능의 인물"을 역산 ②속성 생성 시 기존 인물과의 **대조 거리 하한**(가치관·말투·욕망 벡터) ③탄생 시점 규칙(아크 위상과 결합) ④사망·퇴장 규칙(기능 소진 판정)도 동일 체계의 짝.
- **검증**: 명작의 신규 인물 등장 시점·기능을 모델이 역예측(신품 1~10부: 회차별 신규 인물 등장을 가린 채 "이 시점에 어떤 기능의 인물이 필요한가" 예측 → 실제와 대조).

## §2. 능동 발굴 7요소 — 개발자 미제시 영역
═══════════════════════════════════════════
기준: 문학 이론의 핵심 요소 중 F-01~F-55와 코드베이스 어디에도 생성·측정 장치가 없는 것. 우선순위순.

### LE-3. 인물 음성 분리 (Character Voice) ★최우선 권고
- **무엇**: 대사만 읽고도 누가 말했는지 알 수 있는 인물별 언어 지문 — 어휘 폭·문장 길이·종결어미·화법(직설/우회)·관용구. **StyleDNA는 작품 단위 문체이지 인물별 음성이 아니다** — 현재 모든 인물이 같은 목소리로 말할 위험이 측정조차 안 된다.
- **왜 최우선**: 김은숙 드라마의 식별력이 바로 이것이고, 신품 428씬(인물별 대사 수천 개)이 **이미 손에 있어** 즉시 착수 가능. LLM 생성물의 최대 약점이기도 하다.
- **검증(즉시 실행 가능)**: **블라인드 화자 판별 시험** — 신품에서 인물명 가린 대사를 분류기/판정자에게 주고 화자 적중률 측정(명작 기준선) → 생성물에 동일 시험 → 기준선 대비 갭. 정량·누설 0·LLM 키 소액.
- **생성 측 장치**: 인물별 음성 벡터를 씬 브리프에 주입 + 생성 후 화자 판별 자기 시험 통과 의무.

### LE-4. 서브텍스트·아이러니 (말과 속의 괴리)
- **무엇**: 인물이 말하는 것 ≠ 의도하는 것. "잘 가"가 "가지 마"인 장면. 드라마 대사의 품격이 갈리는 지점.
- **기존**: KnowledgeBoundary SUSPECTS=0.5(암시 주입)가 유일한 근접 장치 — 정보 차단이지 표현 기법이 아님.
- **설계 골자**: 씬 브리프에 표면 발화/이면 의도 2층 명세 → 생성 → 판정자가 이면 의도를 역추출할 수 있는가(추출 가능=서브텍스트 성립, 표면=직설 노출 감점). 극적 아이러니는 독자 지식>인물 지식 상태(이미 NKG에 있음)를 **활용하는 씬 비율**로 측정.
- **검증**: 명작 씬의 서브텍스트 밀도(이면 의도 존재 비율) vs 생성물 — 쌍대 trait 판정("어느 쪽이 말 속에 더 많은 것을 숨기나").

### LE-5. 스테이크 에스컬레이션 (걸린 것의 크기 관리)
- **무엇**: 긴장 곡선(T_ideal)은 **리듬**이지 **크기**가 아니다. 위험에 걸린 것(목숨·사랑·명예)이 단계적으로 커지는 구조는 별도 축 — 같은 긴장 리듬이라도 스테이크가 제자리면 늘어진다.
- **설계 골자**: 씬별 스테이크 등급(개인 불편→관계→생계→정체성→생사→공동체) 주석 → 회차·시즌 스케일 단조 상승 추세 + 국소 완급 검증. 명작의 스테이크 궤적을 신품·FYC에서 추출해 표준 궤적군 학습.
- **검증**: 명작 궤적 vs 셔플 궤적 판별(SG-4 방식 재사용) → 생성물 궤적의 명작군 근접도.

### LE-6. 결말 설계 — "놀랍지만 필연적인" (Surprise-yet-Inevitable)
- **무엇**: 반전·결말의 품질 공식화. 좋은 반전 = 사전 단서가 충분히 깔려 있되(재독 시 필연) 사전 예측은 어려움(초독 시 놀라움). payoff 회수율은 '회수했나'만 보고 '어떻게 회수했나'를 못 본다.
- **설계 골자**: 이중 측정 — ①필연성: 결말 씬과 선행 단서 씬들의 잔향 연결 강도(DRSE 재사용, 이미 검증된 인프라) ②놀라움: 결말 직전까지의 정보로 결말을 예측시킨 적중률(낮아야 좋음). 두 축의 곱이 반전 품질.
- **검증**: 명작 결말(오징어게임 EP9 '말' 회수가 원형)에서 두 축 측정 → 평작·열화판 대비 분리되는가.

### LE-7. 시점·서술자 관리 (POV)
- **무엇**: 누가 이야기를 보고 있는가 — 시점 인물 선택, 시점 전환 규칙, 시점 위반(시점 인물이 알 수 없는 것을 서술) 탐지. 소설 생성에서 특히 치명적(드라마는 Na로 부분 표면화 — 신품도 4인 Na 교차 구조).
- **기존**: KnowledgeBoundary가 인물 인식은 관리하나 **서술 시점**은 미관리.
- **설계 골자**: 씬별 POV 필드 의무화 + 시점 위반 린터(시점 인물 비인지 정보의 서술 노출 탐지 — NKG 대조로 결정론 구현 가능) + 시점 배분 통계(주연 편중도).
- **검증**: 시점 위반 주입 픽스처를 린터가 잡는가(단위 시험) + 명작의 시점 배분 패턴 추출.

### LE-8. 유머·이완 (Comic Relief)
- **무엇**: 긴장 사이의 의도된 웃음. 거의 모든 AI 서사 시스템의 사각지대이고 우리도 측정·생성 장치 0. 김은숙의 무기가 정확히 이것(신품 = 로맨틱 코미디 — **확보 데이터가 곧 교본**).
- **설계 골자**: 유머 생성 공식화는 무리(솔직히) — 대신 ①배치 측정: 명작의 유머 비트 위치 vs 긴장 곡선(이완이 고긴장 직후 오는가) ②쌍대 trait 판정("어느 쪽이 의도된 웃음을 주나")로 존재 여부만 측정 ③생성은 씬 브리프 tone 필드+패스 분리(유머 패스).
- **검증**: 신품에서 유머 비트 주석(소표본) → 긴장 곡선과의 위상 관계 실측 — 가설: 유머는 무작위 배치가 아니다.

### LE-9. 주제 일관성 엔진 (Theme)
- **무엇**: 작품이 말하려는 것(주제)이 사건 선택을 구속하는 메커니즘. MAE R_writer가 주제 일관성을 **평가**하지만 주제를 **유지·주입하는 생성 장치**는 없다. 모티프(이미지 반복)는 있으나 주제는 그 상위 — "복수는 복수를 낳는다"는 명제 수준.
- **설계 골자**: 작품 헌장에 주제 명제 1~3문 고정 → 주요 사건(스테이크 상위 씬)마다 주제 정합 판정(명제를 강화/시험/배반하는가 — 셋 다 정당, '무관'만 감점) → 주제 무관 사건 비율 상한.
- **검증**: 명작에서 주제 명제를 역추출시키고(판정자 합의) 주요 씬의 주제 관련 비율 측정 → 생성물 대비. 단 9요소 중 가장 주관적 — **마지막 순위 권고**.

## §4. 자기 점검
═══════════════════════════════════════════
1. 9요소 전부를 동시 추진하면 미검증 부채 재생산 — **한 번에 1~2개**, 각각 측정 통과 후 다음(공식 생애주기와 동일 규율).
2. LE-8·9는 형식화 난도가 정직하게 높다 — 측정 우선, 생성 장치는 측정이 성립한 뒤.
3. 본 감사는 서사학 일반론 기준 — 한국 드라마 특수 요소(예: 회차 엔딩 훅의 시청률 관습)는 개발자 도메인 지식이 더 정확할 수 있음. 추가·반박 환영.
4. 갭 감사 주기: 분기 1회 또는 Phase 전환 시 — 차기는 Phase E 중간.
