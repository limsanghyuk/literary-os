## §1. 공식의 5역할 정식화 — 결합의 공통 어휘
═══════════════════════════════════════════
기존 기획은 공식을 "baseline sanity check"로만 규정했다. 공식 코퍼스 전수 구조를 보면 실제 역할은 5종이며, 이를 정식 어휘로 고정해야 단계별 결합이 설계된다:

| 역할 | 정의 | 해당 공식군(ID 참조) | 에이전트 관계 |
|---|---|---|---|
| **R1 상태추정기** | 서사 상태를 수치 텐서로 유지 | DRSE 상태전이·서사상태텐서·시간감쇄·동시성충돌 (F-01~F-05 계열) | 모든 에이전트의 **공유 월드스테이트** — 에이전트는 R1을 읽고 쓴다 |
| **R2 보상함수** | 생성물 품질을 스칼라/벡터 보상으로 | fitness 목적함수·MAE 가중보상·Advantage·EMA baseline·RLHF/PPO 군(F-06~F-11, F-44~F-48 계열) | RLAIF/DPO의 **R(x)** — LLM이 아닌 공식이 보상의 1차 원천 |
| **R3 제약·가드** | 생성 중 실시간 위반 차단 | Drift Guard 삼중족쇄·Agency Conservation·KL 발산 한계(F-14, F-48 계열) | 에이전트 행동공간을 **하드 클램프** — 사후 평가가 아닌 진행 중 개입 |
| **R4 거버넌스 게이트** | 릴리즈·권한·정책 결정 | (GPT 트랙 Stage184 거버넌스 계층 — default deny·정책우선·승인규칙 — 의 **Claude 트랙 이식 제안**, 신규 F-G군) | 자율 루프의 **헌법** — LLM-2.0+에서 비중 급증 |
| **R5 설명기** | 평가 근거의 인간 가독 표면화 | 각 공식의 컴포넌트 분해(fitness 6컴포넌트 등) | UI Formula Lens의 데이터원 — **공식은 블랙박스 LLM의 대척점 자산** |

**핵심 명제**: LLM 완화(0→2.5)는 "공식 축소"가 아니라 **역할 이동**이다 — R2(평가 주력)에서 R3·R4(가드·헌법)로 무게중심이 이동하고, R1·R5는 전 단계 불변.

## §2. 단계별 결합 메커니즘 (LLM-0 → 2.5+, 구체)

| 단계 | 생성 | 평가 | 공식 역할 배치 | 신규 결합 메커니즘(구체) |
|---|---|---|---|---|
| LLM-0 (~V745) | 공식·템플릿 | 공식 전부 | R1+R2 주력 | (현행) |

| **LLM-1.5 (F)** | LLM 초안+공식 구조 | 공식 R2 = RLAIF의 보상 | R2가 **DPO/PPO 보상으로 주입**: reward = w₁·fitness + w₂·MAE합의 + w₃·LLM축점수, KL 한계는 R3가 클램프 | 공식 보상의 **미분가능 근사**(아래 §5-2) 착수 |

| **LLM-2.5 (G후반)** | 자율 생성-평가 루프 | 자율 + R4 헌법 | R4 **default-deny**: 루프가 산출한 어떤 결과도 거버넌스 공식 통과 전 미릴리즈, R2는 주기 감사로 강등 | 루프 예산·안전·윤리 공식(신설 F-G군) |
| **LLM-3 (H 스케치)** | 자율 | 자율 | **공식 자가 재보정**: GD 자가갱신(F-07)·AMW α 자가학습(F-13)을 메타 수준으로 — 모델이 공식의 가중치를 재학습하되 R4가 재보정 자체를 게이트 | 스케치만. 구체화 조건 = G.2 무사고 1년(전 문서 §8 불변) |

### Arbitration Protocol v1 (LLM-1의 핵심 신규 설계)
공식 점수와 LLM critic 점수가 불일치할 때의 결정 규칙 — 기존 기획의 "일치율≥0.80"은 측정만 했지 **불일치를 어떻게 쓰는지**가 없었다:
1. 씬별 |z(공식)−z(LLM)| > 1.5σ → **불일치 레코드** 생성(공식ID·축·양측 근거 R5 첨부).
2. 불일치 레코드는 3분기: ①공식 결함 후보(레퍼런스 GT로 재검 → 공식 생애주기 recalibrate) ②LLM 결함(공식이 GT와 일치 → critic 프롬프트 개선) ③진성 모호(둘 다 GT와 등거리 → **인간 개입 큐**, UI로 노출).
3. 분기 ③의 인간 판정 = 최고가치 학습신호(InterventionEvent type=annotate, DPO 페어 가중 2배).
4. 월간 불일치 통계가 G_LLM1_ALIGNMENT의 실측 기반(κ 계산의 원천 데이터).

### 에이전트 토폴로지 — 공식 소유권 매핑
| 에이전트 | 책임 | 소유 공식군(R 역할) |
|---|---|---|
| Orchestrator | 7패스 조율·예산 | R4 일부(패스 전이 게이트)·비용 공식 |

## §3. UI/UX 구체 설계 — Claude Design 반영
═══════════════════════════════════════════
기존 blueprint(3-zone·개입6종·best-of-n)를 유지하고 **컴포넌트·상태·API·디자인 시스템** 수준으로 상향.

### 3.1 디자인 원칙 4 (Claude Design 언어로)
1. **공식 가시성**: 모든 AI 판단 옆에 R5 근거 칩(chip) — 신뢰는 설명에서 나온다.
2. **개입 최소 마찰**: 모든 개입은 1클릭+선택적 사유 — 마찰이 높으면 학습신호가 죽는다.
3. **상태 정직성**: draft/under_review/accepted 상태가 항상 보임 — 어떤 씬도 출처(사람/AI/혼합) 은닉 금지.
4. **잔액 시각화**: 미회수 복선(payoff_debt)·긴장 곡선 등 R1 텐서를 상시 주변시야에.

## §5. 진보 방향 — 추가 제안 4 (기존 기획에 없던 것)
═══════════════════════════════════════════
1. **공식 자가 재보정 메타루프(H 전조)**: GD 자가갱신·AMW α 학습이 이미 '공식이 스스로 배우는' 씨앗이다. Phase F부터 재보정 이력을 원장(formula_ledger)에 적재 → G에서 "재보정 제안은 자동, 승인은 R4+인간" 반자율 → H에서 완전 메타루프. **지금 할 일은 이력 적재 시작뿐** — 성급한 구현 불요.
2. **공식의 미분가능 근사(soft formula)**: R2 보상이 비미분이면 PPO 신호가 sparse하다. fitness·긴장 곡선의 **신경망 근사기**(공식 출력을 distill)를 만들면 LoRA 학습에 dense gradient로 주입 가능. F.3과 병행 연구과제로 — 근사기는 공식의 대체가 아니라 **그림자**(원 공식이 항상 감사).
3. **듀얼 트랙 수렴 계획**: GPT 트랙 Stage184의 거버넌스 계층(평가체·거버넌스체·릴리즈 인장)은 Claude 트랙에 없는 자산이다. §1 R4로 개념 이식하되, 수식·코드 이식은 트랙 혼동 금지 원칙상 **재유도**(개념→Claude 트랙 고유 명세)로만. 수렴 비교표를 Phase F 초 1회 작성.
4. **서사 월드모델로서의 ReaderSim 승격**: 독자 시뮬레이터를 단순 평가기가 아니라 **생성 전 탐색기**로 — 후보 플롯 분기를 R1 텐서 위에서 시뮬레이션해 기대 보상이 높은 분기만 Generator에 전달(tree search와 공식의 결합). G.1의 생성 품질 도약 후보. 비용이 크므로 W4 비용 미터와 함께만.

## §7. 자기 점검
- §2의 LLM-2.0 "토큰 스트림 가드 체크포인트"는 씬 경계 granularity로 한정 — 토큰 단위 개입은 비용·복잡도상 기각(컴파일러 관점).
- §5-2 soft formula는 공식의 권위를 약화시킬 위험 — '그림자' 원칙(원 공식 상시 감사)으로 방어하나, 근사 오차 감사 게이트 설계가 후속 필요.
