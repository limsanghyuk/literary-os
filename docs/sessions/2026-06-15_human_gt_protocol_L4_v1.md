# 인간 GT(작가 평가) 운영 프로토콜 — L4 설계 v1.0

**작성일**: 2026-06-15 · **기준선**: main `6610451` (V749 / v13.3.0) · **문서 ID**: LOS-HUMAN-GT-PROTOCOL-L4-V1.0-2026-06-15
**계기**: P2 최상위 갭. 허브 정독 결과 "인간 GT 운영 프로토콜" 문서 **0건** 확인 → 절대 타당성 검증의 유일 기준이 통째로 비어 있어 본 L4 설계로 메운다.
**선행 근거**: 패널 POC 73%(만장일치 82%) · LW-5(문체 축=LLM 선호 무효지대, S 6/6) · LE-3 v2(생성 음성 −30%, 프롬프트론 복원 불가) · WP-4b 쌍대 패러다임(G_NO_ABSOLUTE_REWARD) · Arbitration v1.

---

## 0. 왜 (문제 정의)
공식(구조 sanity)·LLM 패널(73% 프록시)만으로는 "우리 글이 진짜 좋은가"의 **절대 판정**이 불가하다. 특히 LW-5에서 **절제·저온 문체(거장 특질)를 LLM 심판이 감점**함이 실측됐다 — 즉 LLM 선호 심사가 무효가 되는 지대가 존재한다. 인간 작가 GT는 그 무효지대(박빙·문체 축)에 집중 투입하는 **최종 닻**이다.

## 1. 평가 설계 5원칙 (LW-5 교훈 반영)
1. **쌍대 비교만** — 절대 점수 금지(`G_NO_ABSOLUTE_REWARD`). "A/B 중 어느 쪽"만 묻는다.
2. **선호 질문 금지(문체 축)** — "어느 쪽이 좋은가" 대신 ① 목표 문체 달성도(특성 판단) ② 정전 근접도("현진건/노희경 발췌와 어느 쪽이 가까운가")로 묻는다.
3. **블라인드** — A/B 라벨 은닉·좌우 순서 무작위·출처(생성/실제) 비공개.
4. **실제 대본 앵커** — 비교 기준은 항상 합법 입수 실제 씬(DB `scene_id`). 생성물끼리 비교 금지.
5. **인간 판정 = 최고 가치 신호** — 수집만 하고 즉시 학습하지 않음(학습은 loop-C, Phase F).

## 2. 평가 3모드 (기존 파이프라인과 통합)
- **GT-A 생성 vs 실제(Turing식)** — 동일 dramatic_function 실제 씬 vs 생성 씬. 목적=생성기 품질. (refcheck Mode1의 인간판)
- **GT-B 명성차 실제 씬쌍(캘리브레이션)** — 수상·관객수로 명성차 확실한 실제 씬쌍. 목적=**평가자/패널 신뢰도 측정**(인간이 객관 명성과 일치하는지).
- **GT-C 공식↔인간 중재** — Arbitration `disagreement_queue`(|z공식−z인간|>1.5σ) 항목을 인간이 판정. 목적=공식 생애주기 recalibrate 신호.

## 3. 평가자 모집·자격 (개발자 액션)
- **1차 베타 2~3명**: 시나리오/소설 창작 경력자 또는 문예창작·영상원 전공자.
- **보상**: 쌍당 정액(예: 쌍당 2~3분 작업 기준 시간보상) — 개발자 결정.
- **동의·윤리**: 평가결과의 연구·학습 활용 동의서 + 실제 대본은 **평가 화면 표시(변형적 내부 평가)만, verbatim 미저장**.

## 4. 표본 설계
- **난이도 층화**: ① 격차 大(명작 vs 열화) — LLM으로 충분(11/11), 인간 최소. ② **박빙·문체 축 — 인간 필수 투입**(예산 집중).
- **장르 균형**: thriller/crime/melo/romance/comedy 분산(EXP-C 장르곡선 정합).
- **규모(1차)**: 작가당 30~50쌍, **쌍당 3중 평가**(평가자 3명 교차).

## 5. 신뢰도·일치율 지표
- **Krippendorff α**(순서형/이항) + 쌍 단위 다수결. **임계 α≥0.6**(substantial)에서 GT 채택.
- **인간↔패널 일치율** 산출 → 패널이 인간과 어긋나는 축(특히 문체) 식별 → **패널 캘리브레이션**(객관 명성·인간과 어긋나면 패널 가중 교정).
- 결과는 `validation/ledger.py`에 기록(생애주기 추적).

## 6. 학습 신호 연동 (수집만, 학습은 Phase F)
- 인간 판정 → `learning/signal_schema.py`의 **InterventionEvent(type=annotate)** 로 적재, **DPO 페어 2배 가중**(loop-C).
- GT-C 인간 판정이 "공식 결함"으로 분류되면 → 해당 공식 `formula_registry` 상태 `recalibrate` 전이 트리거.

## 7. 하니스 통합 (코드 연결점 — 차기 V750/751 구현 대상)
- 재사용: `validation/pairwise.py`(Bradley-Terry), `validation/arbitration.py`(disagreement_queue), `learning/signal_schema.py`(InterventionEvent).
- **신규 `validation/human_gt.py`**(차기 버전): `GTRecord`(TypedDict: pair_id, scene_id_a/b(또는 generated_id), mode(A/B/C), evaluator_id, choice, trait_judgment, canon_proximity, ts) · 평가지 생성 CLI(`tools/build_gt_sheet.py`) · 집계(α·승률·인간↔패널 일치율).
- **게이트 후보 `G_HUMAN_GT_ALIGNMENT`**: 인간 α≥0.6 AND 패널-인간 일치율 보고(미달 시 패널 보상 비활성).
