**기준선**: HEAD `0a22018` · 재현: `tools/formula_validation/pilot_unsu/m3/run_lw*.py` · 비용 <$0.50
**설계**: 60분 드라마 1편의 압축 프록시 = 6씬 미니 에피소드(기승전결+복선 회수, 새 전제: 자갈치시장 모자극·놋숟가락 모티프). **모델의 실제 기능 적용**: `StyleDNAEngine.compile('literary')`(절제 저온 프로파일·금지어)와 `KoreanAntiLLMFilter`(클리셰 50+쌍)를 생성 사슬에 실배선. arm S(기능 적용) vs arm P(플레인 생성) 대조. 전부 사전등록.

## 2. 해석 — LW-5가 모순을 푼다
표면적으로 "기능 적용판이 진다"(2a·2b·4 FAIL). 그러나 LW-5가 원인을 분리했다:
- **작가 문체 기능은 완벽히 작동한다** — 목표한 절제·저온 문체를 6/6 달성. 기능 고장이 아니다.
- **LLM 심판이 그 문체를 선호하지 않는 것이다.** LLM 심판은 화려하고 유려한(=LLM스러운) 산문을 선호한다 — 후속실측 D(절대 채점에서 생성 모작>현진건 역전)와 **같은 편향의 쌍대 버전**. 절제는 정확히 현진건·노희경 같은 인간 거장의 특질인데, 그 특질이 LLM 선호 심사에서 감점된다.
- **삼각측량 완성**: LLM 심판 선호는 절대 채점에서도(D), 쌍대 선호에서도(LW-2) LLM스러움 쪽으로 기운다. 신뢰 가능한 것은 **특성 판단**(문체 달성도 6/6, 명작 vs 열화 11/11 — 품질 격차가 클 때)뿐. → **'AI답지 않게 쓰기'를 목표로 하는 모델을 LLM 선호 심사로 평가하면 목표 달성이 감점된다.** 이것이 본 실측의 최상위 발견이다.

## 3. 기능별 정직한 진단
| 기능 | 상태 | 조치 |

| KoreanAntiLLMFilter | **커버리지 공백** — 사전의 정확 문자열 매칭이 실제 LLM 출력에서 0건 발화(시연 문장에선 정상 작동) | 사전 → 패턴/의미 매칭으로 일반화 필요. score_only 천장(10.0) 보정. 단 생성 지시문에 금지어를 선반영하는 우회는 유효했음 |

## 4. 평가 설계 수정 (3인 심의 §5에 추가)
1. **문체 축 평가는 선호 질문 금지** — "어느 쪽이 좋은가" 대신 ①목표 문체 달성도(특성 판단) ②정전 근접도("현진건 발췌와 어느 쪽이 가까운가") 2종으로.
2. 안티LLM 검증도 동형: "어느 쪽이 AI스러운가"(선호성 판단, 편향 오염)가 아니라 **클리셰 밀도·문장 통계 등 측정 가능 특성**으로.
3. 인간 GT의 우선 투입처 확정: 박빙·문체 축(LLM 심판 무효 지대). 명작 vs 열화 같은 격차 大 판정은 LLM으로 충분 — **인간 평가 예산을 가장 가치 있는 곳에만** 쓰는 설계가 가능해졌다.
