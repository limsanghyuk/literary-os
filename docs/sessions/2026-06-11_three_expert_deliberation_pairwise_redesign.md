## §2. 아키텍트 (A) — 개념 차원의 해법
═══════════════════════════════════════════
- **존재론 교정**: 문학 품질은 절대량이 아니라 **관계적 속성**이다. 인간 평단도 절대점수가 아니라 정전(canon) 대비 비교로 작동한다. 따라서 공식의 기본 연산을 `score(x)`에서 **`compare(x, y) → Δ근거벡터`**로 재정의한다.
- **절대 점수가 필요한 곳(게이트 임계·대시보드)**: 쌍대 승패의 누적에서 **Bradley-Terry/Elo 잠재 점수**를 유도 — 절대 스케일은 측정이 아니라 **집계의 산물**로 강등.
- **F-3 해법**: 쌍대 판정의 입력에 NKG·DRSE 구조 신호(복선 회수 차·상태 텐서 거리)를 포함 — 구조 자산이 비로소 보상 경로에 배선된다.
- **F-4 해법**: 비교의 한 변에 항상 **실제 명작(정전 anchor)**을 둘 수 있다 — "정전과의 거리"는 자기선호 순환을 구조적으로 차단하는 가장 싼 방법.

## §3. 컴파일러 (C) — 구현 차원의 해법
═══════════════════════════════════════════
- **비용 통제**: 쌍대는 전수 O(n²) — 기각. **anchor set k=5(고정 기준 씬)와만 비교 = O(kn)**, 순위 필요 시 스위스 페어링 O(n log n). 판정 캐시(쌍 해시).
- **즉시 구현 가능 목록**: ①비교 주석 1콜 양씬 동시(콜 수 동일, 컨텍스트 공유로 변별↑ 기대) ②F 프로토콜을 회귀 단위테스트로 고정(G_PAIRWISE_REGRESSION) ③DPO 파이프는 이미 선호쌍 입력 — InterventionEvent(before/after)가 자연 쌍, 변환 코드 불요.
- **결정성**: 판정 temp=0·위치 무작위 시드 고정·근거(R5) 로그 의무 — 재현 가능.
- **C의 우려 1건**: 쌍대 주석이 절대 주석보다 실제로 나은지는 **추정이지 실측이 아니다** → 본 심의가 끝나기 전에 실험 H로 검증할 것을 요구 (§6).

## §6. 실험 H — 합의안 1단계 즉시 실증 (사전등록)

**해석 (3인 합동)**:
1. **합의안 1단계가 즉시 입증됐다.** 같은 공식, 같은 가중치인데 주석을 비교 형식으로 바꾸는 것만으로 7/11→11/11. 변경 비용: 프롬프트 형식 1곳. 콜 수 동일(오히려 절반 — 두 씬을 1콜로).
2. **절제의 정직한 함의 (P 지적)**: 무가중·단일 컴포넌트도 11/11 — 이 난이도(명작 vs 강열화)에서는 신호가 강해 **가중치의 부가가치는 아직 미입증**이다. 가중치 값어치는 박빙 쌍(명작 vs 준수작, 생성 vs 생성)에서만 가려진다 → 차기 실험: 난이도 사다리(쉬움→박빙) 위에서 가중 vs 무가중 분리. 이것이 fitness 가중 재보정(생애주기)의 정식 입구.
3. **경로 확정**: 측정·보상·검증의 쌍대화는 추정이 아니라 실측 근거를 갖춘 정식 채택안이 됐다. WP-4b로 편입하라.
