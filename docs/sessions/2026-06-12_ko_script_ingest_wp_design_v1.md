## §1. WP-KO 집행 명세 (Sonnet) — `tools/ingest/ko_script_pipeline.py`
═══════════════════════════════════════════
**입력**: `Projects/literary/scripts/*.{pdf,hwp,txt,md}` (개발자 수동 입수분 — 출처 메타 필수)
**처리 6단계**:

| 단계 | 내용 | 명세 |
|---|---|---|
| 1. 포맷 파서 | 한국 시나리오 양식 정규화 | 씬 헤딩 패턴: `S#\d+` · `씬\s*\d+` · `#\d+` · `장면\s*\d+` + 장소/시간 행(`INT/EXT/실내/실외/낮/밤`). 지문 vs 대사 구분: 인물명 행(2~6자+개행) 휴리스틱. hwp는 hwp5txt, pdf는 pdf.js 산출 텍스트 입력 |
| 2. 씬 레코드 | scene JSONL 생성 | 스키마: `{work_id, scene_no, heading, location, time, body, dialogue[], action[], chars[], lang:"ko", medium:"screenplay", source, license_status, sha256}` — scene_feature DDL(P1-P5 §3) 정합, license_status ∈ {official, purchased, member_gray, pd} |
| 3. 청킹 | 이중 청크 | 1차=씬 단위(정본, 공식 입력용) / 2차=512~1024자 슬라이딩(임베딩 검색용). 청크에 scene_no 역참조 의무 |
| 4. 임베딩 | EmbeddingProvider 경유 | WP-3 인터페이스 재사용(gemini→BGE-M3→OpenAI 폴백). 캐시 키: sha256+provider_id. **신규 임베딩 코드 금지** |
| 5. 적재 | 트라이스토어 3중 | ChromaDB(벡터) + 피처테이블 SQL(scene_feature) + NKG(인물 공기어→CIM 부트스트랩 F-36 경로). build_tristore.py 확장, 신규 모듈 금지 |
| 6. 자동 리포트 | 검증 즉시 실행 | SG-1(쌍대 변별)·SG-3(복선 잔향)·SG-4(시퀀스) + 문체 축(trait/canon 판정, 선호 질문 금지 D-PW3) → `out/ko_script_report_{work_id}.md` 자동 생성 |

**게이트 연동**: ①`license_status=member_gray`인 레코드는 finetune/ 경로 유입 시 G_CORPUS_RIGHTS FAIL(타입 수준 차단 — G_NO_ABSOLUTE_REWARD와 동일 패턴) ②인입 시 ledger에 medium=screenplay·lang=ko 이벤트 기록.
**DoD**: TC 8(파서 픽스처 3양식·스키마 검증·청크 역참조·license 차단·캐시 적중·리포트 생성) + 파일럿 1편 전 단계 통과. **RULE-0 Preflight 선행.**
**버전 후보**: WP-2와 통합 권장(WP-2b, 트라이스토어 DDL과 동시 집행) — 실행 큐 변경 없음.

## §2. 분석 로직 — 한국어 각본이 공식 체계에 주는 것
═══════════════════════════════════════════
1. **2×2 매트릭스 측정**: 동일 공식을 [한국어 산문(PD)·영어 각본(FYC)·한국어 각본(신규)]에 적용 → 매체 효과와 언어 효과 분리. SG-2 FAIL(Δfitness 각본 포맷)이 매체 탓인지 언어 탓인지 판별 가능해짐.
2. **fitness recalibrate 정식 실행**: medium=screenplay 분기(stage_registry v2, D-SR1)의 첫 캘리브레이션 데이터.
3. **문체 축 한국어 가동**: StyleDNA·안티LLM 패턴을 영어 번역 없이 원문에서 — LW-5 프로토콜(특성 판단) 한국어판.
4. **F.4b 예비**: 한/영 동일 매체 피처 상관 ρ 측정의 첫 표본.

## §3. 드라마 데이터 추가 입수 (개발자 병행 검색 환영) — 평가 체크리스트
═══════════════════════════════════════════
개발자가 신규 소스 발견 시 아래 4문항으로 1차 판정 후 전달(본 모드가 §1 출처 메타에 반영):

| # | 질문 | 판정 |
|---|---|---|
| 1 | 권리자(방송사·제작사·작가·출판사)가 직접 공개/판매했나? | YES→official/purchased(전 용도 가능) |
| 2 | 회원제 커뮤니티 업로드인가? | YES→member_gray(평가 전용·소수 한정) |
| 3 | ToS에 수집 금지 조항이 있나? | YES→수동 소수만 |
| 4 | 최종 촬영고인지 드래프트인지 표기 있나? | 메타에 기록(검증 해석에 필요) |
