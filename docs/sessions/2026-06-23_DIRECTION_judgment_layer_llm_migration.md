## 2. 핵심 논점 — "판단도 LLM이 해야 LLM-2~3"
### 2-1. 전제 교정 (현 아키텍처 실측 기준)
- 시놉시스 *뼈대*(거시아크/사건관계도/미시플롯수/회차분량)는 **현재 결정론 공식(LLM-0)이 산출**. LLM 판단 아님.
- 불변식 "판단은 로컬" = 로컬은 **공식+Critic**이지 로컬 호스팅 LLM이 아님.
- ∴ **현재는 로컬 LLM이 약해도 구조 뼈대 품질 거의 불변**. 뼈대는 공식 타당성이 좌우(corpus_ko 검증 대상).
- 로컬 LLM 성능이 실제로 품질을 가르는 지점 = **빈칸 2개(로그라인·주제 역생성, 인물속성 채우기) + 산문 윤문 + Critic 판단**.

### 2-2. 방향 합의 + 단서 (Tree-of-Thought 결론)
- **방향 동의**: 사다리(LLM-0→3)가 인코딩한 그림. LLM-1에서 이미 Critic 부분 LLM화(pairwise). 갈수록 판단 무게중심이 공식→모델로 이동.
- **단서①: "대체"가 아니라 "내재화 + floor 잔존."** 생성도 LLM·판단도 LLM이면서 외부 앵커 없으면 모델이 자기를 속이는 방향으로 드리프트(Goodhart). 거대 모델일수록 자기정당화가 더 그럴듯. 공식 = 외부 기준점이라 가치.
- **측정 근거**: 블라인드 3모드 평가에서 심사 LLM이 안티-LLM↔인간 문체 구별 실패 + 패널 자기계열 후한 평가(AI-judge-AI 편향 실측). = floor 제거 시 self-judging 붕괴의 실증.
- **자율주행 비유**: 완전자율도 차선이탈·충돌한계를 학습정책 아래 floor로 유지. 운전 늘어도 가드레일 안 뗌.
- **LLM-2~3 도달의 조작적 정의**: 판단 LLM이 공식의 일을 학습 흡수 → 공식 **발동(bind) 빈도가 0에 수렴**. 그때도 공식은 *그대로 둔다* — 0임을 측정하고 회귀를 잡기 위해. loop-C = floor 앵커된 채 판단 개선. = "공식 = floor not reward"의 진짜 의미.
- **단서②: 판단모델은 프론티어급.** 8B는 판단자 불가(자기편향 더 심함, 장기 인과 못 잡음). 판단까지 LLM이면 그 LLM이 GLM 5.2급/프론티어 → 자체 호스팅 원하면 DGX급 HW 전제.

## 4. 누적 설계 원칙 (이번 세션 산출)
> **판단층의 LLM 이양 = 공식 floor 잔존 + 프론티어급 판단모델 + loop-C 앵커.**
- 공식은 폐기 대상이 아니라 강등(reward→floor) 대상. 발동빈도 0 수렴이 LLM-2~3 지표.
- 로컬 소형(라마3.1-8B/Qwen) = Critic 베이스라인·윤문. 판단 주체 ✗.
- 프론티어 판단모델: API(Claude/GLM) 또는 sovereign 호스팅(GLM 5.2 + DGX급).
