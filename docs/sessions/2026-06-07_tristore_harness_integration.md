# 트라이-스토어 ↔ 공식 검증 하니스 연결 (Stage 1 확장, 2026-06-07)

트라이-스토어 ③ 피처테이블(SQLite)을 공식 검증 하니스에 **직결**해 실행. 5편 30씬.

## 파이프라인 (연결 입증)
피처테이블(SQL 질의) → NarrativeFitnessScore(실 공식) → Spearman 상관 → 재가중.

## 결과 (N=30)
- 기존 가중치 fitness vs 품질: Spearman = **+0.698**
- 상관기반 재가중: **+0.731**
- 컴포넌트 상관: motif 0.82 · surface 0.79 · arc 0.71 · energy 0.68 · conflict 0.60 · curiosity 0.53 (전부 양·고)

## 해석 (정직)
- ✅ **연결 입증**: 피처테이블이 공식 하니스의 입력으로 직접 동작(SQL→공식→상관). 트라이-스토어가 검증 인프라로 가동.
- 주의: N=18(이전 0.40)→N=30(0.70) 큰 변동 = 소표본 노이즈. proxy GT(LLM)·동일LLM 순환으로 절대값은 낙관적.
- 결론: 파이프라인은 작동, 절대 타당성은 규모(30~50편)+인간 GT 필요.
