# 원본 재분할 파이프라인 설계 (v1)

작성 2026-06-26. 역할: 20년차 코퍼스·내러티브 데이터 엔지니어 관점.
사용자 방향: "회당 설계도를 구성하려면 씬/시퀀스/의미비트 카드 단위로 원본 씬을 분석해 코퍼스에 트리를 만들고 데이터화해야 한다. 특히 원본을 제대로 분석해야 한다."
선행 근거: CORPUS-vs-ORIGINAL-SCENE-CROSSVAL-v1.md (적재기는 번호마커엔 충실, 무번호 원본은 slug/fallback 병합 / max_textlen>4000=286화 11.5% 결함).

## 0. 목표 한 줄
원본 한국드라마00~04(173편)의 **진짜 씬 경계를 무손실 복원**하여, 그 위에 카드-트리(다음 문서)를 얹을 수 있는 깨끗한 씬 레이어를 만든다. GPU 불요가 기본, LLM은 폴백 한정.

## 1. 입력 현황 (실측 인벤토리)
- 원본 루트: `C:\claude\db\Scripts` (한국드라마00~04).
- 고유 작품 173편 / 약 118편 데이터화 / 약 55편 미데이터화.
- 코퍼스 씬 적재 전수 2,497화. 그중 **max_textlen>4000 = 286화(11.5%)** 병합결함, **>8000 심각 = 150화(6.0%)**.
- 깨끗한 골드(번호마커+병합없음) 1,446화: 씬밀도 평균 69·중앙 64·p10~p90 46~84.
- 원본 포맷: txt/HWP5/HWP3·doc 혼재. 마커 분포 = `#N`·`S#N`·`N.`·`씬N`·번호없는 장소헤딩.

## 2. 핵심 설계 원칙
1. **트리아지 우선** — 전량 재처리가 아니라 결함 286화만 재분할(작업량 절반↓). 골드 1,446화는 손대지 않음(이미 1:1 정확).
2. **결정론 우선, LLM 최후** — 번호마커·장소헤딩 정규식으로 풀리는 건 GPU/API 없이. 실패분만 LLM.
3. **무손실 불변식** — 재분할은 경계 재배치만. 원본 text 글자(공백제거) 보존율 100% 강제(분할 전후 합산 일치 검증). 실패 시 fail-closed.
4. **in-place 아님, 사이드카** — 원본 코퍼스 레코드를 덮어쓰지 않고 `scenes_resplit/` 신규 스토어에 기록. 검증 통과 후 사용자 확인 하에 승격.

## 3. 파이프라인 단계 (Reasoning+Acting 루프)
```
S0 분류  → S1 마커탐지 → S2 결정론분할 → S3 무번호휴리스틱 → S4 LLM폴백 → S5 검증 → S6 사이드카적재
            (관찰)         (행동)          (행동)            (행동)       (관찰·재결정)  (행동)
```

### S0. 트리아지 분류
각 화를 max_textlen·marker_method로 라벨:
- **GOLD**(max<4000 & 번호마커): 스킵(재분할 불요).
- **MERGE-NUMBERED**(max>4000 & 번호마커 존재): 마커 재적용으로 결정론 복구 — 1순위, 가장 쉬움.
- **MERGE-UNNUMBERED**(max>4000 & slug/fallback): 장소헤딩 휴리스틱 → 실패 시 LLM.
- **CATASTROPHIC**(단일레코드·명명붕괴·텍스트유실<95%): 원본 재입수 필요(별도 트랙, 본 파이프라인 범위 밖 — 표시만).

### S1. 마커 탐지 (관찰)
원본 raw text에서 씬 헤딩 후보 정규식을 우선순위로 스캔:
```
P1 num:#n     ^\s*#\s*\d+
P2 num:S#n    ^\s*S#?\s*\d+
P3 num:n.     ^\s*\d+\s*\.
P4 num:씬n    ^\s*씬\s*\d+
P5 slug       장소-헤딩식(아래 S3)
```
가장 많이·규칙적으로 매칭되는 패턴을 그 화의 primary marker로 채택(단일 화 내 혼용 대비 카운트 기반 선택).

### S2. 결정론 분할 (MERGE-NUMBERED 행동)
primary marker 라인에서 경계를 끊어 씬 배열 재생성. 모래시계 24화 1:1 검증으로 이 경로의 정확도 0오차 확인됨. 286화 중 다수가 여기서 해소될 전망(slug 31.8% 결함이지만 번호계열도 일부 병합).

### S3. 무번호 휴리스틱 (MERGE-UNNUMBERED 행동)
번호 없는 장소-헤딩식 원본의 씬 경계 추정. 3신호 결합:
- **장소+시간 헤딩**: `(S#|#)?\s*(장소명)\s*[\./,]\s*(아침|낮|밤|새벽|오후|저녁|D|N)` — 한국 대본 표준 슬러그.
- **공간전환 큐**: 줄 시작의 `INT./EXT.`·`실내/실외`·신규 고유 장소명(이전 헤딩과 다른 슬러그).
- **인물큐 제외**: `이름:` 형태(대사 화자)는 헤딩 아님 → 경계 후보에서 배제(병합 원흉 = 인물큐를 헤딩으로 오인 or 헤딩 미인식).
세 신호 가중 스코어가 임계 초과한 라인만 경계. 산출 씬수가 골드밀도대(46~84)에 들어오는지 sanity check.

### S4. LLM 폴백 (S3 실패분 행동)
S3가 골드밀도대 밖(과소/과대분할)이거나 신호 모호 시에만. 청크 단위로 "이 텍스트의 씬 경계를 장소·시간 전환 기준으로 표시하라" 프롬프트. 비용 한정 위해 결함 286화 중 휴리스틱 실패분(소수 추정)만. **API 과금 → 사용자 승인 게이트.**

### S5. 검증 (관찰·재결정)
- **무손실**: 분할 씬 글자 합(공백제거) == 원본 글자 합. 불일치 → 해당 화 REJECT, 재처리 큐.
- **밀도 sanity**: 씬수 ∈ [40, 110] 권장대. 벗어나면 플래그(롱테이크 몽타주 정당 가능 → 사람 확인).
- **경계 품질**: 신규 씬 max_textlen<4000 재확인. 여전히 >4000이면 미해소 표시.

### S6. 사이드카 적재
`scenes_resplit/{work}/{ep}.jsonl` 신규 기록 + `resplit_manifest.json`(화별 method·before/after 씬수·손실율·검증결과). 원본 미변경. 사용자 확인 후 승격.

## 4. 전략 비교 (ToT)
| 전략 | 방식 | 장점 | 단점/리스크 | 판정 |
|---|---|---|---|---|
| A 전량 LLM 재분할 | 286화 전부 LLM | 최고 정확 잠재 | API 과금 큼·결정론 가능분까지 낭비·비결정성 | 기각 |
| B 전량 결정론 정규식 | 마커+휴리스틱만 | 무비용·재현성 | 무번호 모호화 일부 미해소 | 부분채택 |
| **C 캐스케이드 트리아지(채택)** | S0분류→번호결정론→무번호휴리스틱→실패분만LLM | 비용 최소·정확·무손실 게이트·재현성 | 휴리스틱 임계 튜닝 필요 | **채택** |
A는 비용·비결정성, B는 모호분 미해소. C만 비용과 정확을 동시 만족 + 무손실 불변식으로 안전.

## 5. 자기 검토 (논리적 약점)
- **휴리스틱 임계(가중 스코어·골드밀도대)는 가설** → 모래시계(번호)·비밀의숲(깨끗한 slug)으로 보정셋 삼아 캘리브레이션 선행 필요.
- **인물큐 vs 헤딩 구분**은 HWP raw 추출에서 미완(기존 한계) → S3 전에 추출기를 헤딩↔대사 구분 가능하도록 보강해야 함(태그/들여쓰기/콜론 패턴).
- **CATASTROPHIC(텍스트유실)은 재분할 불가** → 재입수 트랙 분리, 본 파이프라인은 경계결함만 책임.
- **LLM 폴백 비결정성** → 동일 입력 재현 위해 temperature 0 + 결과 캐시 + 사람 spot-check.

## 5.5 ★PoC 실측 캘리브레이션 (2026-06-26, S0~S3 실행)
설계 직후 코퍼스 scenes 전수 2,497화에 실제로 돌려 트리아지 가정을 검증. **두 가지 정정이 나옴.**

**(1) 결함 유병률 정정 — 225화(무공백 글자기준).** 이전 286화는 raw len 기준. max(무공백씬글자)>4000 = **225화**, >8000 = 105화. 트리거를 무공백 글자수로 통일 권고.

**(2) ★트리아지 비중 역전 — 결정론 경로는 소수, LLM이 주력.** 결함 225화에 번호마커 재분할(S2) 자동선택 적용:
| 경로 | 복구 수 | 비고 |
|---|---|---|
| S2 번호마커(#n·S#n·n.·씬n) 결정론 | **3** | 마더·동이13·아는여자(무손실+max<4000) |
| S2+ 숫자붙은 슬러그(`8만복네 마당D`) 패턴 추가 | **12** | num:n글 패턴이 +9 기여 |
| 잔여 → S3 휴리스틱/S4 LLM 필요 | **213** | 대부분 무번호 슬러그·산문 |

설계 v1은 "MERGE-NUMBERED가 다수 쉬운 승리"라 가정했으나 **실측은 반대**: 결함의 ~95%가 번호 없는 슬러그/산문이라 결정론 정규식으로 안 풀린다. **S4 LLM이 폴백이 아니라 주력 경로.**

**(3) ★결함 4계층 택소노미(신규, PoC서 발견):**
1. **회내 병합·번호有** → S2 결정론(rare, ~3~12). 예 마더(73→90, max 9869→3556 무손실).
2. **회내 병합·숫자붙은슬러그**(`8만복네 마당D`) → glued 헤딩 정규식이 무손실 분할. 단 밀도게이트 필요.
3. **다회차 메가병합**(한 파일에 여러 회 융합) → 회/부 경계 먼저 분리 후 씬분할. 예 그대웃어요_01=원본"23씬"에 53,197자 단일블록, glued분할시 406씬(≈5~6회분)·max3238 무손실 → 밀도게이트(40~110)가 정확히 "단일회 아님"으로 reject. 귀여운여인·커피프린스 동류.
4. **순수 무번호 산문**(장소헤딩 약함) → S4 LLM 필수. 예 선덕여왕_00.

**함의:** S0 분류에 **회경계 탐지(부/회 마커)**를 S1 앞에 추가해야 함(계층3 대응). 그리고 GPU/LLM 예산을 "소수 폴백"이 아니라 **결함 다수(>200화)의 주 처리수단**으로 재산정. 무손실 불변식·밀도게이트는 PoC서 정상 작동 확인(메가병합 자동 격리).

**(4) ★최종 트리아지 확정(triage_final.csv, 225화 전수 분류):**
| 클래스 | 정의 | 처리경로 | 화수 |
|---|---|---|---|
| class1 | 번호마커 회내병합 | S2 결정론(무LLM) | **12** |
| class2 | glued 슬러그 회내병합 | S3 휴리스틱(무LLM) | **0** |
| class3 | 다회차 메가병합 | 회분리 후 재분할 | **2** |
| class4 | 순수 무번호 산문 | **S4 LLM 필수** | **211** |
- 무LLM(결정론+휴리스틱) 복구 합계 = **12/225 (5%)**.
- LLM 또는 회분리 필요 = **213/225 (95%)**.
→ 설계 v1의 "번호결정론 다수 승리" 가정 **완전 기각**. S4 LLM이 결함셋 재분할의 **주 경로**(95%)임이 전수 분류로 확정.
