# 데이터 소싱 방식 명확화 (2026-06-07) — Claude 트랙

## 1. "50편 → 200편"의 의미
분량 목표(작품 수)다. SP-E.1=50편(파일럿20+Gold30), Phase F=200편 확대. '방법'이 아니라 '규모'.

## 2. 데이터화 단위 = 씬 (방법, 출처 불문)
한 작품을 씬으로 분할 → 각 씬에서 인물·사건·거시플롯·미시플롯(서브플롯)·복선(plant→payoff)을 정의·정리 → 정량 피처(13필드 SceneFeature) 산출. 이 구조·분석·피처가 데이터의 본체.

## 3. 두 소싱 경로 (상보)
### A. 무(無)스크립트 구조 분석 (저작권 안전, 스크립트 불필요)
- 출처: 공개 회차 리캡·위키·리뷰·작품 시청·LLM 지식.
- 방법: 위 공개 자료에서 씬·인물·플롯·복선을 분석.
- 장점: 비용↓, verbatim 0(안전), 대량(폭). / 한계: 정확도·환각 위험 → 다출처 교차+작가 검수 필수.
- 용도: 폭(breadth) 200편 L0~L2.

### B. 합법 스크립트 기반 (정확, 입수 제약)
- 출처(실재·합법): KOFIC **시나리오마켓**(회원 원문 열람/다운로드, 저작권 추적), **한국영상자료원 영상도서관/KMDB 시나리오·콘티**, **출판 대본집**(교보문고 등), 시나리오 공모 수상작.
- 방법: 원본 스크립트를 읽고 씬을 정확히 분할·분석.
- 장점: 정확·풍부(대사 행동·서브텍스트). / 제약: 입수 비용·권리, **대사 verbatim 저장·재배포 금지**.
- 용도: 깊이(depth) Gold 20~30편 L2/L3.

## 4. 핵심 저장 원칙 (출처 불문)
어느 경로든 **저장하는 것은 '구조·분석·정량 피처'이지 대사 원문이 아니다.** 스크립트를 읽어 분석하더라도 DB에는 씬 분할·인물 상태·플롯·복선·SceneFeature만 들어간다(verbatim=false). → 저작권 안전 + 공식이 실제로 먹는 것과 일치.

## 5. 이전 언급 정리
- 'YouTube/검색으로 대사 스크립트 읽고 씬 분할' = 경로 A(공개 자료 구조분석)의 한 형태. 단 대사 원문은 저장 안 함.
- 'MBC 한 편 100원' = 합법 스크립트 구매(경로 B)의 취지. 정확한 'MBC 100원' 서비스는 미확인이나, **합법 등가물(KOFIC 시나리오마켓·KOFA 영상도서관·대본집)이 실재**하므로 그 경로로 대체.

## 6. 권고 (하이브리드 = 트라이-스토어 채우기)
