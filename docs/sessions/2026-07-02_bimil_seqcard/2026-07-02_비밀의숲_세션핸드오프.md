## 2. 어떻게 했나 (방식) — 재현 절차

워크플로: **원본추출 → 회차별 Sonnet 병렬저작 → series_arc → 검증 → 허브 push.**

### 2-1. HWP5 원본 추출 (★커스텀 파서 필수)
표준 변환기(hwp5txt / `python -m hwp5` / soffice) **전부 실패**. 직접 작성한 `hwp5extract.py`(olefile + zlib)로 16파일 클린 추출.
- HWP5 = OLE 복합문서 (magic `d0cf11e0a1b11ae1`).
- FileHeader byte 36 bit0 = 압축 플래그 → raw deflate (`zlib.decompress(data, -15)`).
- 레코드 헤더 4바이트 LE: tag_id(bits 0-9) / level(10-19) / size(20-31), size==0xFFF면 다음 4바이트가 실제 크기.
- **PARA_TEXT tag=67**, 텍스트 UTF-16LE. 제어문자 {0,10,13}=break, INLINE{4~9,19,20}·EXTEND{1,2,3,11,12,14~18,21~23}는 16바이트(8워드) 소비, ≥32는 일반 문자.

### 2-2. 씬 분절 (★헤딩 포맷 회차별 상이)
- EP01-04 = "장소**.** 낮/밤" (마침표), EP05-16 = "장소 **-** 밤" (하이픈/en-dash). **같은 작품인데 회차마다 다름.**
- S#(씬번호)는 이 대본엔 Flashback 참조에만 등장 → 통합 정규식 + Flashback/Insert 제외 필터로 1,037씬 분절.

### 2-3. 저작
Sonnet 16 에이전트 병렬(에피소드당 1개, 2웨이브). 각 에이전트가 해당 회차 씬 본문을 정독하여 9필드 라벨. core/core2는 16기능 택소노미에서만 선택. **Opus 순차저작 금지 규칙 준수.**

### 2-4. 검증 → series_arc → push
- `validate.py`: 필드셋 일치, scene_no 연속, core/core2 택소노미 소속, 비어있지 않은 문자열, by/work_id/meta 정합. → TOTAL 1037, ERRORS 0.
- `build_arc.py`: 비밀의숲_series_arc.json 생성.
- `rebuild_all.py`: `_ALL_series_arc.json` 22작 재집계.
- git add/commit/push → 원격 fresh clone 재검증.

## 4. 이번 분석에서 발견한 규칙 (silver-label 가설)

> ⚠️ 전부 `by=*_reading` 단일모델 라벨. κ게이트 미통과 → **검증사실 아님, prior 주입 금지.** 거시플래너 설계 힌트로만 사용.

1. **core_dist = 장르 지문.** 설계에 없던 창발. 비밀의숲·마왕=CONFLICT/PERIL/REVELATION(스릴러), 도깨비=ROMANCE/DESIRE, 베토벤바이러스=CONFLICT/BOND(군상극).
2. **CONFLICT_persist = 군상극↔단선극 축.** 베토벤 0.071(분산=군상극) vs 비밀의숲 0.254(단선 수사).
3. **core는 덩어리(런)로 발생.** 전이문법 top이 자기루프 지배 → 거시플래너는 씬 낱개가 아니라 **시퀀스(기능 런) 단위**로 배치해야 함.
4. **택소노미 반증이 실제 작동.** 파일럿에서 episode_role 8→6 병합, tension_role 앵커 재정의 유발(스키마 v2.1).
5. **씬헤딩 포맷은 회차 단위로 흔들린다** (2-2 참조) → 파싱 정규식은 회차별 패턴 선탐지.
6. **자기점검:** 측정가능 작업으로의 도피 경향 + silver-label 순환(설계·채움·κ검증을 한 주체가 하면 AI-judge-AI 폐쇄루프). 1~3이 "가설"에 머무는 이유.

**가설→지식 승격의 유일 병목:** 독립 인간 어노테이터 블라인드 라벨(κ 앵커). 이 모드에선 실행 불가.
