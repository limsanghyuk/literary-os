# 레퍼런스 기반 다중에이전트 검증 프로토콜 (P2 인간GT 보강, 2026-06-07)
개발자 제안 채택: 인간 작가 GT가 현 단계 비현실적 → (1) 다중에이전트가 인간 평가 대행, (2) **실제 시나리오를 GT 레퍼런스로 교차검증**.

## 0. 왜 이게 더 타당한가
- 기존 제안(작가 블라인드 선호)·LLM-judge(추상 품질판정)는 **순환·인력 문제**.
- 핵심 통찰: **실제 드라마 씬 자체가 최고 전문가(김은숙 등)의 산물 = GT 레퍼런스.** LLM에게 추상적 '좋은가'를 묻지 않고, **명작과 비교**한다 → 순환 제거.
- 인간 최고전문가 부재 → **고품질 다중에이전트 앙상블**이 임시 대행(레퍼런스 그라운딩으로 단일 judge보다 강건).

## 1. 두 검증 모드
### Mode 2 (우선·순환 최소) — 공식 on 실제 명작
- 실제 명작 씬(합법 대본집)에 공식(SceneFeature·fitness·DRSE·NKG)을 적용.
- 가설: 공식이 **명작을 높게 평가**하고 **실제 복선을 탐지**하는가? (명작의 평단·시청 acclaim = GT)
- LLM 불필요 → **순환 0**. 공식 타당성의 가장 깨끗한 1차 검증.

### Mode 1 — 생성 vs 실제 (Turing식 블라인드)
- 동일 조건(인물·직전상태·거시목표)으로 모델이 씬 B 생성 → 실제 씬 A(대본집).
- **다중에이전트 블라인드 비교**(A/B 라벨 은닉, 순서 무작위) → 선호·점수.
- 지표: B가 A에 **대등/근접**할수록 모델 우수. 실제가 명백히 이기면 = 척도가 품질 변별력 보유(정상). verbatim 일치가 아니라 **구조·품질 대등성**.

## 2. 다중에이전트 평가자 (인간 GT 대행)
| 페르소나 | 기준 |
|---|---|
| 문학평론가 | 문학성·주제밀도·문장 |
| 드라마투르그 | 극적구조·인과·긴장·인물 |
| 일반시청자 | 몰입·감정·다음 궁금 |
| (옵션) 장르전문가 | 장르 관습·톤 |
- 블라인드·순서무작위·다수결 + 일치율(분산/krippendorff). 단일 LLM judge보다 강건.
- 한계: 여전히 LLM 프록시(인간 최고전문가 아님). 단 레퍼런스 그라운딩으로 추상판정보다 타당. 인간 작가 확보 시 상위 검증으로 승격.

## 3. 교차검증 구조
- Gold 대본집 작품의 여러 씬을 레퍼런스로 → Mode2(공식 적용) + Mode1(생성 비교) 풀링.
- 작품·장르 균형, N 확대(30~50편 → 수백 씬).
- 사전등록: Mode2 임계(공식이 명작 상위 X% 정확 평가), Mode1 임계(B 대등률).

## 4. 저작권
- 실제 씬은 **합법 대본집 열람**으로 평가 입력(변형적·내부 평가). 저장은 평가결과·피처만(verbatim 미저장).

## 5. 하니스 (작성됨, 크레딧 충전 시 실행)
- tools/formula_validation/refcheck.py: 생성 B + 레퍼런스 A + 3페르소나 블라인드 비교.
- (현 상태: Anthropic·OpenAI·Gemini 크레딧 전부 소진 → 라이브 실행 보류. 충전 또는 키 교체 시 즉시 실행)
- 예시 조건: 미스터 션샤인 1화 특정 씬 → 생성 vs 실제 비교.

## 6. P2 보강 효과
- **인간 GT 병목 대폭 해소**: 실제 시나리오=GT, 다중에이전트=임시 평가자. 개발자는 대본집 입수만 하면 됨(작가 섭외 불요).
- 단, 다중에이전트는 프록시이므로 최종 절대 검증은 인간 작가가 이상적(차후).
