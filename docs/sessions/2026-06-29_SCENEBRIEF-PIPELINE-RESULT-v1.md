# ①②③ 씬 설계도 파이프라인 — 1회차 E2E 실증 (2026-06-29)

verbatim 본문 비커밋. 본 보고는 방법·집계·브리프(추상 메타)만. 데이터는 로컬 db.

## 한 줄
재분할(①)→Claude 직접 브리프(②)→brief→산문 학습쌍+회차 설계도(③)를 천만번사랑해 1화에서 끝까지 실증.

## ① 재분할
- s#N. 마커 재파싱 → **53씬**(이전 저장 6씬, 47씬 손실 회복). 천만번사랑해 110회차 전체 동일 복구(평균 57.5씬).

## ② Claude 직접 브리프 (GPT 미사용)
- 53씬 각각 {소제목·설명·극적기능·주요인물}을 **본 세션 Claude가 직접 작성**(Anthropic 키 크레딧부족으로 API 배치 불가 → 직접 작성이 최상 품질).
- 극적기능 분포(1화): 복선심기 10·인물성격 7·관계전환 6·잔향 6·인물소개 5·인물관계 5·갈등촉발 3·정보공개 3·반전 2·상승 2·관계갈등 2·인물심리 1·**회말훅 1**.
- → **1화 구성 법칙이 수치로 보임**: 전반=인물·세계 소개+복선 대량 심기, 후반=부도 확정(반전)·난정 간 적합(반전)·금자 유혹(회말훅).

## ③ 산출
- **brief→산문 학습쌍 53쌍**(`brief2prose/`): prompt=(소제목+설명+기능+인물), completion=씬 본문. = 생성 학습 재료.
- **회차 설계도 뷰**(md): 53씬 한 줄 요약 배열 = "왜 이 회차가 이렇게 구성됐는지" 가독 설계도.

## 스케일 결정거리 (★)
- Claude(상위 모델) 직접이 품질 최상이나, **Anthropic API 키 크레딧 부족으로 배치 자동화 막힘**. GPT는 사용자 지시로 배제.
- 옵션: (a) Anthropic 크레딧 충전→Claude API 배치(haiku/sonnet, 드라마 ~10만 씬) (b) 세션별 Claude 직접(최상품질·소량) (c) 회사 Claude 세션 분담 배치.
- 권고: Tier-1 명작 우선 (a) 또는 (c). 결함 회차(원본/​txt 누락분)는 제외.
