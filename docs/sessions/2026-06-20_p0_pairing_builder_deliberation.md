# 세션 기록 — P0 선호쌍 빌더 멀티에이전트 설계 심의 (2026-06-20)

## 지시
사용자: "우선순위를 정해 단계적으로 진행. 최고 수석 아키텍트·컴파일러·데이터 전문가가 협의·교차 비교·논의하여 설계 초안서 작성. 최고 수석 시스템 프린시펄 엔지니어가 감독·문제조사·결론 검토·최종안 확정하여 허브에 로드. 정직하게 각 전문 에이전트를 충분히 동원."

## 수행
독립 서브에이전트 4기 실제 기동:
1. **수석 아키텍트** — 파이프라인 골격·소프트쿼터 자동충원·느슨한 0.20 임계 제안.
2. **수석 컴파일러** — per-token 하드코딩·sum 차단 가드·E4 통합 명세.
3. **수석 데이터** — 타이트 임계(token 5%/char 8%)·길이매칭 후 E4 재실행·혼합비 15/55/20/10 보정.
4. **수석 시스템 프린시펄 엔지니어** — 교차검토 감독. 충돌 6건 판정 + 자체 결함 2건(G1 토크나이저, G2 ablation) 발견.

## 확정 결론 (DESIGN-P0-PAIRING-BUILDER-v1)
- ARCH의 소프트쿼터 자동충원·0.20 느슨임계 **기각**(오염 재유입). DATA의 타이트 임계 채택.
- per-token 전용 + sum 3중 차단. 길이매칭 후 E4. 작품단위 train/held(held>=250).
- 프린시펄 발견: 단일 토크나이저 + tokenizer_sha 잠금; ablation은 P3 이연, P0은 input-set 해시만 동결.
- 파일별 작업지시서 9단계 + DoD 7항.

## 자기검증
임계·혼합비는 미실측 휴리스틱(P3+인간블라인드 캘리브레이션 의무), per-token 마진 미세, AI-judge-AI 편향은 크기 미정량, P4 크레딧할당 스텁.
