## 0. 위치 — 왜 지금 LLM-1 '이후'를 기획하나
현재 loop-C(show/tell 선호쌍 DPO)가 졸업 임박 = LLM-1(쌍대 Critic의 '노트 능력') 실증. 그러나 이건 **문장 기교 한 칸**일 뿐, "작가팀 대체"의 거시 기획(16/24부작 통째 구성)은 빈칸. 이 문서가 그 빈칸(LLM-2)을 채우는 기획을 누적한다.

## 1. 북극성 재확인
인간 드라마/영화 **작가팀 대체** — 16/24부작 통째 생성. 불변식: 판단은 로컬(공식 floor + Critic), 생성만 LLM, 학습은 loop-C 누적, 공식=floor(보상 아님).

## 2. 작가팀 모델 (사용자 제기) → 오케스트레이터 매핑
메인 작가팀(전체 구성·회차 플롯·씬 분할) + 서브 작가팀(개별 씬 생성). **정정 3**: ① 내려보내는 단위=회차 통째(비트로 묶임), 고립 씬 아님 ② 단방향 아니라 되먹임 루프(메인 검토·노트·재수정) ③ 한국드라마는 전통적 스타작가 단독집필 多(팀 모델은 지향).
| 인간 | 시스템 |
|---|---|
| 메인(거시) | 플래너/오케스트레이터: intent→story_bible→season_arc→beat→scene |

## 3. 허브 기존 자산 vs 빈칸 (실측)
- **이미 코드 존재(Phase C SP-C.2)**: DirectorAgent.generate_blueprint(씬청사진 5요소) → ScriptAgent(최대3 regen) → CriticAgent(5축) → request_regeneration 되먹임 → EditorAgent.finalize. = 서브+비평+편집은 있음.
- **시놉시스(story_bible) 7요소 중 5개는 결정론 엔진 존재**(인물: character_birth_gate·ledger_builder·grid_builder·pressure_cast_planner·character_intent_agent / 사건도 유사).
- **★빈칸 2개 = ① Synopsis Assembler ② 로그라인/주제 역생성.** 이 둘이 LLM-2 플래너의 첫 실체. `world.story_bible.StoryBibleAggregator`는 노트만·미구현.

## 4. ★3-전문가팀 심의 (작가·비평·PD) — 한 결론으로 수렴
- **수렴①**: 분포는 명작의 *증상*이지 *원인* 아님. 곡선 모양(변곡=막경계·소멸=퇴장)이 핵심, 절대값 아님. **음성게이트(범작 거르기)엔 강하나 양성(명작 만들기)엔 무력** — 평균분포 모방=가장 정교한 범작. 흥행작 코퍼스 생존편향 경고.
- **수렴②(핵심)**: 진짜 빈칸 = **인과 척추 + 주제(aboutness)**. 작가팀 "씬→씬 강제력" = 비평팀 "이 작품은 무엇에 관한가". → **Synopsis Assembler의 1급 산출물 = 인과 척추(주제 질문 포함). 분포는 부산물 검증으로 강등.**
- **수렴③**: **Goodhart는 floor로 방향까지 못 막음**. 공식=floor는 하한만, 최적화 쏠림은 못 막음. show/tell이 먼저 졸업=측정 쉬운 축으로 쏠리는 증거(자랑 아니라 경고).
- **채워진 전문지식**:
  - 메인/서브 구분 3기준: 종결위치(메인=클라이맥스 닫힘) / 인과종속(서브가 메인 선택 바꾸면 종속, 아니면 평행=테마거울) / 회수책임(메인=의무, 서브=폐기가능).
  - 측정가능 구조요소: 떡밥-회수거리(이봉) · 회말훅 5종 · 감정선 valence곡선(미드포인트 거짓승리) · 톤 교대 자기상관 · 미드포인트 반전 · 씬 인과연쇄 깊이 · 변화회계 · 설정-회수율 · 선택의 비용 · 서브텍스트 밀도.
  - 시놉시스 표준 10요소(PD): 로그라인/기획의도/타깃/인물관계도/세계관/시즌아크·막구조/회별시놉+회말훅/서브플롯트랙표/톤레퍼런스/차별화.
- **결정 필요 2건**: (a) 제작가능성·시장 게이트 스코프? → 권고: 1차=집필(인과척추), 제작게이트는 입력조건 인터페이스만 빈칸(Phase G로). (b) floor가 방향 못 막음 → 양성축(변화회계·설정회수·선택비용) + 축간 균형 메타게이트.

## 5. 분포 실측 (floor 데이터화 = 단계 ① 완료)

## 6. 판단층 LLM 이양 방향 (2026-06-23, 미확정)
"판단(공식+Critic)도 결국 거대 LLM이 해야 LLM-2~3 도달". 보정 합의 = (a) 공식은 폐기 아니라 **floor 잔존**(발동빈도 0 수렴이 LLM-2~3 지표) (b) **판단모델은 프론티어급**(8B는 주플래너✗·로컬Critic/윤문만) (c) **loop-C 앵커**. GLM 5.2(MIT 오픈웨이트) = sovereign 판단층 로컬화 미래카드(744B라 4070 불가).
