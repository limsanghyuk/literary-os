## 3. 우선순위맵의 핵심 진단 (그대로 인용·반영)
> "이미 충분: 검증 방법론·측정 인프라·로드맵. **실질 부족 1순위 = 생성 본체의 L4 부재** — orchestration/ 7패스는 여전히 의사코드 골격. 다음 큰 기획은 '무엇으로 평가하나'가 아니라 **'무엇을 생성하나'**."
> "경고: 기획은 **실측이 만든 빈칸에만** 투입. 실측 없이 쌓는 기획은 '정교한 미검증 부채'."

→ 함의: 우리가 만든 건 **평가·학습 절반**(critic·보상·loop-C·GPU). **생성 절반**(7-pass 실체)이 골격이면 loop-C가 개선할 *대상*이 빈약하다. 생성 본체가 최대 레버리지 코드 빈칸.

## 5. 권고 인터리빙 (Gate 기반, 절대일정 아님)
1. **T1 즉시 가동**(개발자): 단독킷으로 4070 첫 실 라운드 → ΔW 측정. *나머지와 독립 병행.*
2. **T3 착수**(나, 코드 1순위): 7-pass 중 골격→실구현. loop-C에 개선할 실체 공급 = T1의 가치 증폭.
3. **T2 병행**(나, 코드 2순위): NextEpisodeBench 구체 설계→PoC. T3 생성물을 연속생성으로 평가.
4. **T4 후순위**(나): T1~T3로 자율생성 품질이 측정상 오른 뒤 UI/공식 Exit.

원칙: 각 트랙은 **실측→빈칸→기획→실행** 사이클 준수. 기획 누적 금지(우선순위맵 경고).

## 6. 결론
세 트랙 전부 진행이 맞고, 생성 본체(T3)를 더해 4트랙으로 본다. **개발자=T1 실측 / 나=T3 생성본체→T2 연속평가→T4 UI** 순. 다음 즉시 착수 후보 = **T3 생성 본체 7-pass L4 1단계**(loop-C가 개선할 실체 확보).
