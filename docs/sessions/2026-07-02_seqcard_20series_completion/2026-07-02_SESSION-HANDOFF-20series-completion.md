## 2. 작업 방식 (파이프라인, 시리즈당 동일하게 반복)

1. **원본 확보**: `C:\claude\db\Scripts\` 하위 폴더에서 zip(hwp 묶음) 또는 PDF 확보. 완결성(전 회차 존재) 확인 후 착수.
2. **텍스트 추출**:
   - hwp: `hwp5txt`(pyhwp) 사용. 이번 세션에서 바이너리가 PATH에 없어(`/sessions/.../.local/bin/hwp5txt`) 매번 절대경로 지정 필요했음(샌드박스 리셋마다 재발).
   - zip 내 한글 파일명: `cp437`→`cp949` 라운드트립 디코드.
   - **태양의후예 PDF(2단 촬영고)**: 일반 `pdftotext`/`pdftotext -layout` 둘 다 좌우 컬럼이 섞여 읽기 순서가 깨짐 확인. `pdfplumber`의 `page.within_bbox((0,0,w/2,h))`/`(w/2,0,w,h)`로 좌/우 컬럼을 각각 추출 후 좌→우 순서로 이어붙이는 방식으로 해결(사전 테스트로 검증 후 전 16화 일괄 적용).
3. **씬 경계 형식 사전 검증**: 회차 샘플(1화·중간·마지막화) 대상 `grep -c`로 마커 패턴과 개수를 먼저 확인한 뒤 저작 착수. 마커가 없는 작품(너의목소리가들려·녹두꽃)은 "장소+시간대" 단독행을 씬 경계로 채택(역전의여왕에서 확립한 패턴 재사용).
4. **웨이브 저작**: `Agent` subagent를 회차당 1개씩, 웨이브당 최대 6개 병렬 파견. 프롬프트에 9필드 스키마·16기능 택소노미·해당 작품의 정확한 씬 경계 규칙·"core_dist는 core 필드만 집계하고 0인 항목은 넣지 말 것"을 매번 명시.
5. **웨이브 후 독립 재검증**(에이전트 자체보고를 신뢰하지 않고 매번 bash로 재확인):
   - jsonl 실제 레코드 수 = episode_meta.json의 `scene_count`
   - `core_dist` 합계 = 실제 레코드 수, 그리고 실제 jsonl에서 재계산한 분포와 dict 값 자체가 정확히 일치
   - `work_id` 일관성, `scene_no` 1..N 연속성, 16기능 택소노미 값 유효성, 9필드 셋 완비성 — 5개 항목 전부 체크하는 python 스크립트를 매 웨이브 반복 실행
6. **시리즈 집계**: 전 회차 jsonl을 합쳐 총 씬수·전체 core_dist·top4·CONFLICT→CONFLICT 마르코프 자기전이확률(`CONFLICT_persist`) 계산.
7. **`_ALL_series_arc.json` 갱신**: series 딕셔너리에 신규 항목 추가, `totals`(series_count/eps/scenes) 갱신, `note`에 진행상황 한국어 요약.
8. **허브 반영**: uploads의 "키 (1).docx"에서 GitHub PAT 재추출(세션 간 미유지) → `/tmp/hubN`에 매번 새로 클론(이전 클론은 샌드박스 리셋으로 소실) → `authored/*.jsonl`·`*.episode_meta.json`·`_ALL_series_arc.json`만 복사(원본 텍스트는 저작권상 절대 커밋 금지) → 커밋(한국어 상세 메시지) → push → `git fetch`+`git show origin/main:...`+`json.load`로 원격에 실제로 반영됐는지 재확인.

## 3. 발견된 이슈와 수정 사례

**(1) bash 마운트 지연으로 인한 파일 잘림 — 이번 세션에도 3회 재발.**
파일 도구(Read/Write/Edit)로 방금 쓴 내용을 bash가 즉시 보지 못해, 잘리거나 NUL 바이트가 섞인 상태로 커밋될 뻔한 사례가 반복됨. 매번 Read 도구로 확정한 정확한 내용을 python 스크립트(dict 하드코딩 후 `json.dump`)로 강제 재작성하고, `python3 -m json.tool`로 재검증하는 방식으로 우회했다.
- `_ALL_series_arc.json` 갱신 시(너의목소리가들려 반영 단계): Edit 직후 bash `json.tool` 파싱 실패 → python dict 하드코딩 재작성으로 해결.
- `녹두꽃_08.seqcard.jsonl`: 마지막 줄에 NUL 바이트가 남아 71번째 줄에서 파싱 실패(Read로는 70줄 정상) → `bytes.replace(b'\x00', b'').rstrip()`로 정리 후 재검증 통과.

**(2) 에이전트 세션 한도 도달 — 너의목소리가들려 14·17·18화(3건).**
"You've hit your session limit · resets 4:30pm (Asia/Seoul)" 응답으로 실패. 재시도 시 14화는 이미 파일이 정상 저장돼 있어 재검증만 수행, 17·18화는 정상 재실행으로 완료.

**(3) episode_meta와 실제 jsonl 불일치 — 공주의남자_13화.**
`scene_count=83`으로 기록됐으나 실제 jsonl 레코드는 82개(HOOK 카운트도 5→4로 스테일). 실제 jsonl을 기준으로 재계산해 수정.

**(4) 카인과아벨 씬 마커 회차별 4종 혼재.**
`N.\t`/`■\t`/`.\t`/번호없는 순수 프로즈가 회차마다 다르게 쓰였으나 각 회차 내부는 일관됨을 확인 — 과거 세션에서 실격 판정한 "궁"류 불량 소스와는 다른 케이스로 판단, 회차별로 정확한 마커를 개별 지정하여 정상 처리.

**(5) 태스크 리스트 자동 스트레이 생성.**
에이전트 완료 메시지에서 파생된 중복/조기완료 태스크(#23·#24·#25·#26)가 자동 생성되어 매번 수동으로 삭제·상태 정정(특히 16화 중 12화만 완료된 시점에 자동으로 "completed" 처리된 사례) 처리함.
