# ADR-119: ReaderFeedbackCollector — PIPA 준수 피드백 수집 + Gate G68

## 상태
Accepted (V659, 2026-05-27)

## 맥락
PublicSDK와 B2B API가 가동된 후, 독자 피드백을 수집하여 RLHF 파이프라인에
연결해야 한다. 한국 개인정보 보호법(PIPA 2024-03-15 기준) 준수가 필수다.

## 결정

### PIPA 준수 4원칙
| 원칙 | 구현 |
|------|------|
| 동의 확인 | ConsentLevel 열거형 — NONE/ANONYMOUS/PSEUDONYMOUS/IDENTIFIED |
| 최소 수집 | 점수(1~5)·유형·씬 ID만 저장, 상세 개인정보 미수집 |
| 익명화 | 이메일·전화·주민번호·이름칭호 자동 마스킹 + reader_id → SHA-256 해시 |
| 보유 기간 | RAW_RETENTION_DAYS=14일, purge_expired() 호출 시 자동 파기 |

### Gate G68 합격 조건
| 항목 | 기준 |
|------|------|
| PII 잔류 | 0건 (100% 익명화) |
| 피드백 수 | ≥ 10건 (MIN_FEEDBACK_COUNT) |
| 동의 차단 | 추적 가능 (차단율 보고) |

### PII 마스킹 패턴
- 이메일: `[EMAIL]`
- 전화번호(한국): `[PHONE]`
- 주민등록번호: `[RRNO]`
- 이름+칭호: `[NAME]`

## 결과
- `literary_system/feedback/` 패키지 신설
- `literary_system/gates/feedback_collection_gate.py` (G68)
- TC 33개 추가
