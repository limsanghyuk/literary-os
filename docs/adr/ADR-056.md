# ADR-056: LoRA Dataset Format, ProvenanceLedger, DatasetRegistry, DSR SLA Policy

**상태**: Accepted  
**날짜**: 2026-05-21  
**버전**: V596 (v10.1.0)  
**작성자**: Literary OS Engineering

---

## Context

Phase B SP-B.1 에서 Llama-3.1-8B 기반 LoRA 파인튜닝 파이프라인을 구축한다.  
학습 데이터셋의 재현성, 출처 추적, GDPR/PIPA DSR 처리, DVC 버전 관리를 위한
공통 정책이 필요하다.

## Decision

### 1. Alpaca JSONL 형식 (LoRADatasetBuilder)

```json
{
  "instruction": "당신은 한국 드라마 작법 전문가입니다...",
  "input": "장르: 멜로 | 작품: 사랑의 계절\n\n씬 내용...",
  "output": "씬 텍스트 전문",
  "entry_id": "entry-uuid",
  "content_hash": "sha256[:16]",
  "source_type": "synthetic|licensed|public_domain",
  "license": "CC-BY-4.0"
}
```

- DSR 삭제 마킹된 entry_id는 build() 시 자동 제외

### 2. 8:1:1 분할, seed=42 고정 (DatasetSplitter)

- train:val:test = 0.8:0.1:0.1
- `random.Random(42)` 결정론적 셔플
- `stratified=True` 시 source_type 별 균등 분배

### 3. ProvenanceLedger — sha256 체인

```
record_id = sha256(f"{entry_id}:{content_hash}:{prev_hash}:{created_at}")
```

- append-only, GENESIS_HASH = "0"×64
- `verify()` 체인 무결성 검증 (ProvenanceChainError)
- `mark_dsr_deleted(entry_id, request_id)` DSR 삭제 마킹
- JSONL 영속화

### 4. DatasetRegistry + DVC (DatasetRegistry)

- sha256 체크섬 검증 (`verify()`)
- DVC add 실패 시 `RuntimeWarning` (graceful degradation)
- 3-tag: version_tag + split_tag + source_hash

### 5. DSRHandler — 30-day SLA (DSRHandler)

- `DSR_SLA_DAYS = 30`, `DSR_SLA_SECONDS = 30 * 86400`
- 상태: PENDING → PROCESSING → COMPLETED / REJECTED / EXPIRED
- `overdue_requests()` SLA 초과 시 EXPIRED 전이

## Consequences

- LoRA 학습 데이터의 법적 책임 추적 가능
- DSR 요청 30일 내 처리 의무 이행 자동화
- DVC 미설치 환경에서도 graceful degradation으로 동작
- DatasetRegistry를 통한 sha256 기반 재현성 보장
