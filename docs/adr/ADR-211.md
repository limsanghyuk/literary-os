# ADR-211: WP-4b Pairwise 패러다임 프로토콜

- **Status**: Accepted
- **Date**: 2026-06-12
- **Version**: V748 / v13.2.0
- **Phase**: Phase E SP-E.0

## Context

Phase E 검증지 로드맵 WP-4b는 Literary OS 내 품질 비교 방식을 절대 점수 기반에서
쌍대 비교(pairwise) 패러다임으로 전환한다. 기존 `F-06_fitness` 등 절대 점수 보상은
스케일 편향, 판단자 간 불일치, 캘리브레이션 비용이 높아 신뢰도 문제를 야기했다.

## Decision

### §0 금지 사항 (Prohibition Set)

1. **절대 점수 보상 금지** — `rlhf/`, `finetune/` 내 `reward = <float literal>` 패턴 금지
2. **문체 축 선호 질문 금지 (D-PW3)** — `mode=preference` + `trait` 필드에 문체/스타일/style/tone 포함 시 `ValueError`
3. **naive 임베딩 anchor tension 금지** — anchor 유사도 기반 단순 집계 금지 (BT 점수만 허용)
4. **전수 O(n²) 쌍대 금지** — `tournament()` 함수는 O(kn) 이하만 허용 (k 기본값 5)
5. **docs/sessions commit 시 `git add -f` 필수** — .gitignore 포함 파일은 강제 추가

### Bradley-Terry (BT) 모델

- 판정 결과에서 잠재 점수 추정: iterative MLE 200 iter
- `bt_scores(judgments) → Dict[str, float]`
- 대칭 매치 결과 → 점수 동일 (±epsilon)

### Anchor Set V1

- 5개 PD 씬 고정: `운수좋은날_s02`, `운수좋은날_s10`, `운수좋은날_s11`, `pd_anchor_s01`, `pd_anchor_s02`
- SHA256 핀: `anchor_sha = sha256(sorted IDs joined with ',')`
- 하위 호환 pin 변경 시 별도 ADR 필요

### 회귀 픽스처 (G_PAIRWISE_REGRESSION)

- 명작 vs 강열화 11쌍 → 승률 ≥ 9/11 (≥81.8%)
- V749에서 Gate 구현 예정

### Transitivity Check (G_TRANSITIVITY)

- 판정 그래프 3-cycle 비율 < 5% 요구
- V749에서 Gate 구현 예정

## Gates

| Gate | 설명 | 버전 |
|------|------|------|
| G_NO_ABSOLUTE_REWARD | rlhf/finetune/ 절대 float reward 패턴 차단 | V748 (본 ADR) |
| G_PAIRWISE_REGRESSION | 명작 vs 강열화 11쌍 승률 ≥9/11 | V749 |
| G_TRANSITIVITY | 판정 cycle rate <5% | V749 |

## Consequences

- `literary_system/validation/pairwise.py` 신규 (compare, tournament, bt_scores, transitivity_check)
- `tools/run_release_gate.py` — `_check_no_absolute_reward()` 게이트 추가
- `tests/validation/test_pairwise.py` — 17 TC (DoD 8)
- 향후 `rlhf/reward_model.py` 신규 시 `# G_NO_ABSOLUTE_REWARD_OK` 예외 주석 또는 BT 기반 재설계 필요

## Alternatives Considered

| 대안 | 이유 기각 |
|------|-----------|
| ELO rating | 순서 의존성 높음, 수렴 불안정 |
| Thurstone Case V | 정규성 가정 강함, 소표본 부적합 |
| 절대 점수 + 정규화 | §0 금지 사항 ①에 해당 |
