# ADR-053: CorpusGovernance — 코퍼스 수집 거버넌스

**Status**: Accepted  
**Date**: 2026-05-21  
**Version**: V592 (v9.7.0)  
**Deciders**: Literary OS Core Team  
**Tags**: corpus, provenance, pii, governance, sp-a5

---

## 컨텍스트

Literary OS의 드라마 데이터 파이프라인이 고품질 시나리오 코퍼스를 필요로 한다.
V591까지는 합성 데이터(CorpusIngestor)만 사용했으나, Phase A 완료 목표인
6,000 PASS 및 v10.0.0을 위해서는 다양한 출처의 실제 코퍼스가 필요하다.

핵심 제약:
- **저작권 준수**: 모든 항목의 라이선스 추적 필수 (Provenance 100%)
- **PII 제거**: 개인식별정보 수집 금지 (PIPA 준수)
- **LLM-0**: 수집 파이프라인에서 외부 LLM 호출 없음
- **폴백 전략**: 단일 소스 의존 금지 (KOFICE 협약 실패 시 대안 필요)

---

## 결정

### 1. 3종 폴백 옵션 (우선순위 A → B → C)

| 옵션 | 클래스 | 소스 | 라이선스 | 비용 |
|------|--------|------|----------|------|
| A | PublicDomainIngestor | 공공도메인 한국고전 | public_domain | $0 |
| B | SyntheticCorpusIngestor | 합성 템플릿 | CC-BY-4.0 | ~$50 |
| C | AcademicCorpusIngestor | KOFICE/KOCCA LOI | academic_license_v1 | 협약 |

`CorpusFallbackPipeline.collect(count)` — 자동으로 A→B→C 순서로 보충.

### 2. CorpusProvenanceIndex

모든 `CorpusEntry`에 대해 entry_id / source_type / license / sha256 / ingested_at 추적.

```python
index = CorpusProvenanceIndex()
index.register_batch(entries)
coverage = index.coverage(entries)  # ADR-053 완료 조건: coverage == 1.0
index.to_jsonl("/data/provenance.jsonl")  # 영속화
```

완료 조건: **coverage == 1.0** (5천 신 Provenance 100%)

### 3. CorpusPiiFilter

수집 단계에서 PII 제거.

```python
flt = CorpusPiiFilter(strict=False)  # strict=True이면 PII 엔트리 제거
clean, removed = flt.filter_entries(entries)
```

지원 패턴: 주민등록번호 / 전화번호 / 이메일 / 계좌번호

### 4. CorpusEntry 데이터 모델

기존 `ScenarioEntry`(V557)와 별도로 `CorpusEntry` 정의 — Provenance 필드 강화.

## 결과

### 긍정적 결과
- 3종 폴백으로 KOFICE 협약 지연 시에도 5천 신 달성 가능
- Provenance 100% 추적으로 저작권 감사 대응
- PII 자동 제거로 PIPA 준수
- JSONL 영속화로 재현성 확보

### 부정적 결과 / 트레이드오프
- AcademicCorpusIngestor는 현재 플레이스홀더 (LOI 체결 전)
- PublicDomainIngestor는 10개 고전 작품 기반 (V593에서 확장 예정)

### 중립적 결과
- SP-A.6(V593)에서 CorpusValidator(4단 필터) 및 1만 신 검증으로 이어짐
