# ADR-050 — BackendHealthMonitor / PartialAvailability

| 항목 | 내용 |
|---|---|
| **버전** | V589 |
| **날짜** | 2026-05-21 |
| **상태** | Accepted |
| **Gate** | G48 (PartialAvailabilityGate) |
| **SP** | Phase A / SP-A.2 |

---

## 문제

V588 QueryInterface는 LOSDBClient의 모든 백엔드가 정상이라는 가정 하에 동작한다.
실 운영 환경에서 SQL / Vector / Graph 중 일부가 장애 상태일 때 쿼리를 계속 시도하면
타임아웃·캐스케이드 장애로 SLO(1.0s)를 초과할 위험이 있다.

---

## 결정

`literary_system/db/health_monitor.py`에 **BackendHealthMonitor** 클래스를 구현하고,
QueryInterface에 선택적으로 주입하여 부분 가용 시 폴백 로직을 수행한다.

---

## 설계

### AvailabilityState

| 상태 | 설명 | 가용 백엔드 수 |
|---|---|---|
| `FULL` | 모든 백엔드 정상 | n/n |
| `PARTIAL_DEGRADED` | 일부 장애 | ≥ 2 |
| `CRITICAL` | 대다수 장애 | 1 |
| `OFFLINE` | 전체 장애 | 0 |

### CircuitState (per backend)

```
CLOSED → (연속 3회 실패) → OPEN → (60s 경과) → HALF_OPEN → (성공) → CLOSED
                                                            → (실패) → OPEN
```

### BackendHealthMonitor API

```python
monitor = BackendHealthMonitor(
    ping_interval_sec=30.0,   # ping 캐시 간격
    failure_threshold=3,      # OPEN 전환 연속 실패 수
    recovery_timeout_sec=60.0 # OPEN → HALF_OPEN 대기 시간
)
monitor.register(BackendType.SQL,    ping_fn=lambda: True)
monitor.register(BackendType.VECTOR, ping_fn=check_qdrant)
monitor.register(BackendType.GRAPH,  ping_fn=check_neo4j)

monitor.check_all()               # → Dict[BackendType, CircuitState]
monitor.get_available_backends()  # → List[BackendType]
monitor.overall_state()           # → AvailabilityState
monitor.health_report()           # → Dict[str, Any]

# 테스트/시뮬레이션용
monitor.force_open(BackendType.VECTOR)
monitor.force_closed(BackendType.VECTOR)
```

### QueryInterface 통합

```python
qi = QueryInterface(
    client=losdb_client,
    health_monitor=monitor,   # 선택적 주입 (None이면 전체 허용)
)
# health_monitor가 있으면 _get_available_backends()가 가용 백엔드만 반환
# VECTOR OPEN → find_characters() 빈 결과 반환 (SLO 보호)
```

### HybridRetrieverV2 통합 (SP-A.2)

```python
v2 = HybridRetrieverV2(
    bm25=bm25_retriever,
    dense=dense_retriever,
    health_monitor=monitor,   # 선택적
)
# VECTOR OPEN → BM25 단독 폴백 (source="bm25_fallback")
```

## T1~T4 시나리오

| 시나리오 | 상태 | 동작 |
|---|---|---|
| T1 | FULL | 전체 쿼리 정상 |
| T2 | PARTIAL_DEGRADED | 장애 백엔드 스킵, 나머지 계속 |
| T3 | CRITICAL | SQL 전용 쿼리 (Vector/Graph 스킵) |
| T4 | OFFLINE | 빈 결과 반환 (SLO 보호) |
