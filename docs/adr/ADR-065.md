# ADR-065 — CanaryController + ModelServingEndpoint 설계 결정

**날짜**: 2026-05-22  
**상태**: Accepted  
**담당**: SP-B.2 V605  
**연관 ADR**: ADR-063 (PPOTrainer), ADR-064 (RLHFMonitor)

---

## 1. 배경

SP-B.2 RLHF 루프(V601~V610)에서 훈련된 모델(PPO + LoRA)을
프로덕션 환경에 점진적으로 배포할 수 있는 Canary 인프라가 필요하다.
단순 일괄 전환(Big Bang)은 리스크가 크기 때문에 트래픽 비율을
4단계로 나눠 각 단계에서 Gate를 통과해야 다음 단계로 진입한다.

---

## 2. 결정 사항

### 2.1 4단계 Canary 트래픽 비율

| 단계 | 트래픽 비율 | 설명 |
|------|------------|------|
| STAGE_0 | 5 % | 초기 카나리아 — 소수 요청에만 노출 |
| STAGE_1 | 25 % | 부분 롤아웃 — 1/4 트래픽 |
| STAGE_2 | 50 % | 절반 롤아웃 — A/B 동등 비율 |
| STAGE_3 | 100 % | 전면 전환 — 기존 모델 완전 교체 |

### 2.2 단계 Gate 조건 (AND 조건)

다음 4가지를 모두 충족해야 `gate_passed = True`:

| 조건 | 기본 임계값 | 근거 |
|------|-----------|------|
| requests ≥ min_requests_per_stage | 10 | 통계적 유의성 확보 |
| error_rate ≤ error_rate_threshold | 0.05 (5%) | SLO 오류율 상한 |
| latency_p95 ≤ latency_p95_threshold_ms | 1500 ms | SP-B.4 G60 목표 (P95 ≤ 1.5초) |
| mean_reward ≥ reward_threshold | 0.75 | G56 보상 임계값 (ADR-063) |

### 2.3 P95 레이턴시 산출 방식

`sorted(latencies)[int(n * 0.95) - 1]` — numpy 없이 순수 Python으로 산출.
LLM-0 원칙(외부 의존 최소화)을 준수한다.

### 2.4 ModelServingEndpoint FastAPI 소프트-임포트

FastAPI 미설치 환경(CI, 단위 테스트)에서도 `get_model_card()` 메서드가
정상 동작해야 한다. `try/except ImportError` 로 소프트-임포트 처리.

---

## 3. 거부한 대안

| 대안 | 거부 이유 |
|------|---------|
| 단일 50/50 A/B | 초기 리스크 노출이 너무 큼 |
| 연속적 트래픽 램프 | Gate 판정 시점이 불명확 |
| Kubernetes Argo Rollout 의존 | 로컬 LLM-0 원칙 위반 |
