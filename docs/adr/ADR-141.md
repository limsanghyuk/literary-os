# ADR-141: V679 Enterprise 컴플라이언스 감사 익스포터 (G78)

## 상태
Accepted

## 날짜
2026-05-27

## 컨텍스트
SP-C.4 Enterprise Layer 5번째 안정화 단계. 테넌트별 SLO·Revenue·Cost·Isolation
이벤트를 표준 감사 로그로 집계하고 규정 준수 여부를 판정하는 감사 레이어가 필요하다.
기존 `compliance/audit_trail_db.py`의 `AuditEventType`·`AuditSeverity`와 충돌하므로,
Enterprise 전용 `EnterpriseAuditEventType`·`EnterpriseAuditSeverity`로 분리한다.

## 결정
`literary_system/enterprise/compliance_audit.py` 신규 파일에 다음을 구현한다.
- `EnterpriseAuditEventType`: SLO_VIOLATION / REVENUE_DISPUTE / COST_EXCEEDED /
  ISOLATION_BREACH / POLICY_CHANGE / ACCESS_DENIED
- `EnterpriseAuditSeverity`: INFO / WARNING / CRITICAL
- `ComplianceStatus`: COMPLIANT / NON_COMPLIANT / UNDER_REVIEW
- `EnterpriseAuditEvent`: 단일 감사 이벤트 (is_critical 속성)
- `TenantComplianceRecord`: 테넌트별 컴플라이언스 상태 집계
- `EnterpriseAuditReport`: 전체 테넌트 감사 리포트
- `EnterpriseAuditEventStore`: 이벤트 저장소 (new_event 팩토리)
- `EnterpriseComplianceAuditor`: CRITICAL ≥3 → NON_COMPLIANT, CRITICAL ≥1 → UNDER_REVIEW
- `EnterpriseAuditExporter`: JSON / summary 포맷 내보내기
- `EnterpriseComplianceAuditGate` (GATE_ID="G78"): 4-테넌트 데모 시나리오
