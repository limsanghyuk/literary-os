# ADR-192: SP-D.4 착수 전 위험 신호 완화 전략 (Pre-V731 Risk Signal Guard)

**날짜**: 2026-05-29
**상태**: ACCEPTED
**범위**: tools/spd4_risk_guard.py, SP-D.4 개발 가이드라인

## 배경

Phase D handoff 문서 §7에는 11종의 위험 신호가 정의되어 있다.
SP-D.4 착수(V731) 전에 SP-D.3 이전 항목을 확인하고,
SP-D.4 신규 위험(FL 수렴, DR RPO, Phase E manifest)에 대한
사전 방어 전략을 확정한다.

### 🔶 SP-D.4 신규 위험 — 사전 방어 전략 확정

#### RS-08: FL PoC delta 수렴 실패 (10라운드 초과)

**원인**: FedAvg 초기 epsilon=0.001이 데이터 분포에 비해 너무 작거나,
2-테넌트 가중치(n_k) 편차가 커서 delta가 수렴하지 않을 수 있다.

**완화 전략**:
```
1. FedAvgAggregator 구현 시 epsilon 파라미터를 외부 주입 가능하게 설계
   - 기본값: epsilon=0.001
   - 완화값: epsilon=0.01 (10배 완화)
2. max_rounds=10 초과 시 EarlyStopException 발생 (무한루프 방지)
3. 매 라운드 delta 값을 로그 기록 — 수렴 여부 가시화
4. G90 TC에 "epsilon=0.01 완화 시나리오" 1건 포함 필수
```

**결정**: `FedAvgAggregator(epsilon=0.001, max_rounds=10)` — epsilon은 키워드 인자로 노출.

#### RS-09: DR RPO > 1h

**원인**: 백업 주기가 1h를 초과하거나 복원 로직에 오류가 있을 경우.

**완화 전략**:
```
1. G91 DR Gate 구현 시 RPO 검증을 명시적 assertion으로 강제
   - backup_interval_seconds ≤ 3600 (1h = 3600s)
   - restore_time_seconds ≤ 3600
2. DRBackupManager에 _validate_rpo() 내부 검증 메서드 필수
3. TC에 "RPO = 3601s → FAIL" 경계값 테스트 포함
```

**결정**: G91 구현 시 `assert backup_interval_seconds <= 3600` 강제.

#### RS-11: Phase E manifest validate 실패

**원인**: V740에서 생성할 Helm + KEDA + ArgoCD manifest의
스키마 오류 또는 필수 필드 누락.

**완화 전략**:
```
1. V740 구현 전 manifest JSON Schema를 tools/phase_e_manifest_schema.json 에 정의
2. tools/spd4_risk_guard.py가 schema 유효성 사전 검증
3. D-M-12 manifest에 필수 필드 목록 명시:
   - helm: name, chart, namespace, values
   - keda: scaledobject.spec.triggers[].type
   - argocd: application.spec.source.repoURL
4. validate_phase_e_manifest() 함수를 tools/에 구현 (V740 전)
```

**결정**: V740 구현 전에 `tools/phase_e_manifest_validator.py` 스텁 작성.
