# ADR-227: LocalGPUAdapter — 로컬 4070 QLoRA 학습 경로 (GPU 3-모드 ①)

## 상태
ACCEPTED (V767, Phase E.4 확장)

## 맥락
RLAIF DPO 데이터셋을 클라우드(RunPod/Lambda)만으로 학습하던 구조에, 개발자 워크스테이션의 **RTX 4070(12GB)** 을 로컬 학습 경로로 추가. 3인 합의 설계도(docs/sessions/2026-06-16_gpu_3mode_blueprint_v1.docx) 기반.

## 결정
`finetune/gpu_adapter.py`:
- `GPUProvider.LOCAL` enum 추가, registry 등록 → `get_adapter(LOCAL)`.
- `LocalGPUAdapter`(GPUAdapterContract 구현): cost_per_hour=$0(전기료 metadata 별도), QLoRA 4bit 모델별 VRAM 추정(3B 6/7B 11/8B 11.5/70B 46GB), 12GB 초과 시 fallback_cloud 신호.
- `LocalPreflight`: nvidia-smi(GPU·VRAM) + torch/transformers/peft/trl/bitsandbytes 점검. 미충족→ok=False→CLOUD 폴백.
- `finetune/train_local.py`: 사용자 PC에서 실행하는 QLoRA DPO 학습 스크립트(무거운 의존은 main 내부 import → 레포 LLM-0 무부하).
RLAIFTrigger(V765)는 provider 인자로 LOCAL 라우팅 가능.

## 결과
14 TC PASS. 로컬 dry_run(cost $0, 전기료 추정, VRAM 가드) 동작. 샌드박스(GPU 없음)는 fallback_cloud=True, 4070에선 Preflight PASS 예정. 3-모드 중 ①로컬 완성. LLM-0 유지.
## 대안
sandbox 직접 학습: GPU 부재로 불가. 사용자 PC 실행(train_local.py)으로 위임.
