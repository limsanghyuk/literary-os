# ADR-152: Prometheus /metrics OTel TraceContext 통합 (V689, D-M-02 완성)

## Status
Accepted — V689 (2026-05-28)

## Context
V688에서 W3C TraceContext Level 1 전파기를 도입했으나,
실제 관측 지점인 Prometheus /metrics 엔드포인트에 trace context가 연결되지 않았다.
D-M-02는 "모든 관측 가능 경계(HTTP endpoints 포함)에 traceparent 전파"를 요구한다.

### 문제
| 항목 | 현황 |
|------|------|
| trace_context.py | V688에 구현됨 ✅ |
| otel_adapter.py | V688에 구현됨 ✅ |
| /metrics 엔드포인트 trace 전파 | **미구현** ❌ |
| span 관련 Prometheus 메트릭 | **미구현** ❌ |

## Decision

### 1. 신규 모듈: `prometheus_trace_extension.py`

기존 `prometheus_exporter.py`를 상속·확장:

```
literary_system/ops/prometheus_trace_extension.py
  ├── TraceMetricSnapshot(MetricSnapshot)    ← span 통계 4종 추가
  ├── TraceAwareExporter(PrometheusExporter) ← render_trace() 추가
  ├── MetricsEndpoint                        ← /metrics 시뮬레이터
  └── MetricsResponse                        ← 응답 dataclass
```

### 2. trace 전용 메트릭 4종

| 메트릭 | 타입 | 설명 |
|--------|------|------|
| `literary_os_spans_exported_total` | counter | 누적 exported span 수 |
| `literary_os_active_traces` | gauge | 현재 활성 trace 수 |
| `literary_os_trace_errors_total` | counter | 에러 상태 span 수 |
| `literary_os_p99_span_duration_ms` | gauge | P99 span 지연시간 (ms) |

### 3. /metrics 엔드포인트 trace 전파 흐름

```
클라이언트 scrape 요청
  └─ traceparent 헤더 포함 (선택)
       ↓
MetricsEndpoint.handle_request()
  ├─ TraceContextPropagator.extract_or_create(headers)
  │   → 부모 컨텍스트 획득 (없으면 신규 생성)
  ├─ child_context(parent) → 이 scrape 요청의 child span
  ├─ OtelSdkAdapter.start_span("metrics_scrape")
  ├─ TraceAwareExporter.render_trace(snapshot)
  │   → 기본 메트릭 + trace 메트릭 4종 포함
  ├─ TraceContextPropagator.inject(child_ctx, response_headers)
  └─ MetricsResponse(status=200, traceparent=child_traceparent)
```

### 4. 설계 원칙
- **비파괴적 확장**: 기존 `PrometheusExporter` 코드 미수정
- **상속 기반**: `TraceAwareExporter(PrometheusExporter)` - 기존 테스트 영향 없음
- **외부 의존성 없음**: prometheus_client 라이브러리 불필요

## Consequences

### 긍정적
- D-M-02 완전 이행: TraceContext Level 1이 /metrics 경계까지 전파
- Prometheus Exemplars 도입 준비 완료 (trace_id를 metric에 부착 가능)
- Grafana Tempo 연동 기반 확보

### 부정적/트레이드오프
- `TraceMetricSnapshot`의 span 통계 값은 수동 입력 — 자동 집계는 향후 구현
- MetricsEndpoint는 실제 HTTP 서버 미포함 (FastAPI/Flask 통합은 Phase D 후반)
