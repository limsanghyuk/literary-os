# ADR-177: Plugin Registry Gate G87

**날짜**: 2026-05-28  
**상태**: Accepted  
**버전**: V716 (12.3.0)  
**작성자**: Claude (SP-D.3)

---

## 문제 제기

V711~V715에서 구현된 플러그인 서브시스템(PluginManifest, PluginRegistry,
PluginSandbox, PluginLifecycleManager, BasePlugin/PluginContext, PluginTestHarness)은
독립적으로는 검증되었으나, SP-D.3 릴리즈 게이트(G87)가 부재하여
전체 파이프라인에 통합되지 않았다.

ADR-128에 따르면 `plugins` 패키지가 2개 연속 버전(V714, V715)에서 고립되었으므로
V716에서 반드시 게이트 연결이 이루어져야 한다.

## 결정

`literary_system/gates/plugin_registry_gate.py`를 신설하고
`release_gate.py`에 G87로 등록한다.

### G87 체크포인트 (7개)

| ID   | 검증 대상                              | 기준 |
|------|---------------------------------------|------|
| PR-1 | PluginManifest 스키마 검증            | plugin_id ≥ 3자, entry_point 점 분리, version X.Y.Z |
| PR-2 | PluginRegistry CRUD + 훅              | 등록·조회·해제·on_register/on_unregister 훅 |
| PR-3 | PluginSandbox 화이트리스트            | 허용 모듈 실행 성공, 차단 모듈 실행 실패 |
| PR-4 | PluginLifecycleManager 상태 머신      | INACTIVE→ACTIVE→INACTIVE→ACTIVE (restart) |
| PR-5 | BasePlugin / PluginContext SDK        | emit/get_outputs, require_permission, MANIFEST 강제 |
| PR-6 | PluginTestHarness 격리               | activate + sandbox 실행 |
| PR-7 | `__all__` 공개 API 심볼               | 20개 필수 심볼 노출 |

## 결과

- `plugins` 패키지 ADR-128 고립 위반 해소
- 릴리즈 게이트 G87 추가 → GATES 총 84개
- 누적 테스트: 9,898 TC (V716 신규 33 TC)
