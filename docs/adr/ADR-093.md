# ADR-093 — TrainPlane Helm Chart 검증 전략

- **날짜**: 2026-05-25
- **상태**: Accepted
- **구현**: V626 / literary_system/ops/helm_validator.py
- **관련**: ADR-057 (TrainPlane/ServePlane 격리), ADR-092 (biweekly_train)

---

## 컨텍스트

V625에서 biweekly_train CI가 도입됨에 따라, 학습 Job을 배포하는 Helm chart
(deploy/helm/train_plane/)의 구조적 무결성을 자동 검증하는 메커니즘이 필요하다.

잘못 구성된 Helm values.yaml이 배포될 경우:
- GPU 리소스 미설정 → 학습 Job 스케줄 불가
- 네임스페이스 혼동 → ServePlane과 GPU 충돌 (ADR-057 §5 위반)
- 비용 SLO 논리 오류 → 비용 제어 메커니즘 무력화 (ADR-057 §4 위반)

## 결정

`HelmValidator` 클래스를 `literary_system/ops/helm_validator.py`에 구현하여
CI 파이프라인에서 TrainPlane chart를 정적 검증한다.

### 9개 체크포인트

| 번호 | 체크포인트 | 검증 내용 |
|---|---|---|
| C1 | chart_dir_exists | 차트 디렉토리 존재 |
| C2 | chart_yaml_valid | Chart.yaml 필수 필드 + apiVersion=v2 |
| C3 | values_yaml_valid | values.yaml 필수 키 + 하위 구조 |
| C4 | namespace_isolation | literary-train만 허용, literary-serve/default 금지 |
| C5 | gpu_resources_valid | nvidia.com/gpu >= 1 (requests + limits 모두) |
| C6 | cost_slo_valid | 0 < soft < hard < emergency, monthly <= hard |
| C7 | lora_hyperparams_valid | rank 범위, alpha >= rank, dropout 0~0.5, epochs 1~20 |
| C8 | template_files_exist | lora-job.yaml + cronjob.yaml 존재 |
| C9 | service_account_valid | SA 이름 'default' 미사용 (경고) |

## 결과

- 에러 0건 → `HelmValidationResult.valid = True`
- 에러 1건 이상 → `valid = False`, CI 실패 처리
- 경고는 CI를 차단하지 않음 (개선 권고)

## 보안 원칙

- LLM-0: 외부 LLM API 호출 없음
- PyYAML 선택적 의존성: 없을 경우 최소 파서 폴백 (_minimal_yaml_parse)
- G37 안전: HelmValidator, HelmValidationResult, TrainPlaneChartSpec — literary_system/ 내 유일 이름 확인됨

## 대안 검토

1. **helm lint 직접 사용** — Helm 바이너리 의존성 추가 필요, CI 컨테이너 복잡도 증가. 기각.
2. **JSON Schema 검증** — 외부 라이브러리 의존. 현재 범위에선 과도. 추후 V628+ 검토.
3. **현재 선택: 순수 Python 검증** — 의존성 최소, CI 호환성 최대.
