## 컨텍스트

`tools/formula_validation/` 하위 스크립트들은 `sys.path.insert(0, "/tmp/...")` 하드코딩을 포함한 임시 스크립트였다. Phase E WP-1 목표는 이를 정규 모듈로 승격해 ~25개 공식 일괄 검증의 상설 인프라를 구축하는 것이다.

## 결정

`literary_system/validation/`에 4개 모듈을 신규 추가한다.

1. **`formula_registry.py`** — `FormulaEntry` TypedDict + `REGISTRY` dict  
   - lifecycle: `candidate → validated → recalibrate → deprecated`  
   - 1차 등록: `F-06_fitness` (domain=physics)

2. **`stage_registry.py`** — Stage 임계값 사전등록 상수 (코드 변경 없이 런타임 변경 불가)  
   - Stage 1~6: spearman/f1/dtw_pct 임계, GT 필드명, 최소 샘플 수

3. **`formula_harness.py`** — `Harness` + `StageReport` + `FormulaResult`  
   - SQLite / JSONL 이중 데이터 소스, LLM 호출 0회  
   - `cost_cap` 파라미터: 미래 LLM 확장 대비 (현재 0.0 소비)

4. **`ledger.py`** — `record()` + `transition()` + `get_lifecycle()`  
   - `docs/formula_ledger.md` append (커밋 대상)  
   - 2회 연속 recalibrate → `deprecated` 자동 승격

## CLI

```bash
python tools/run_formula_validation.py --stage 1 --db data/corpus_seed/scenes_5works.jsonl
python tools/run_formula_validation.py --stage 1 --db data/tristore.db --cost-cap 0.5
```
