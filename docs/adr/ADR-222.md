# ADR-222: loop-C 선호쌍 + DPO 적재 (E.4 RLAIF)

## 상태
ACCEPTED (V762, Phase E.4)

## 맥락
Pass7 패널(생성 vs 실명작) 판정을 생성기 학습 신호로 전환해야 한다(loop-C). 개발자가 dpo_pairs.jsonl(17쌍, 생성 승률 59%)을 수집함.

## 결정
`learning/loop_c.py`: PreferencePair(chosen/rejected, 절대점수 금지) + 개발자 dpo_pairs.jsonl 직접 인입 + to_dpo_dataset(DPO 표준) + generation_win_rate(격차 지표) + reference_strength(pairwise BT). finetune.lora_*로 GPU 학습 결선(Phase F).

## 결과
16 TC PASS. 개발자 실 17쌍 인입 → 승률 0.588 재현. 다음 V763=보상모델, GPU LoRA 학습은 Phase F.
## 대안
절대점수 보상: G_NO_ABSOLUTE_REWARD 위반(기각). 쌍대 DPO 채택.
