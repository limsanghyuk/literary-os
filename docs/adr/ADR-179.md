# ADR-179: TenantAuthority — 테넌트 격리·권한 관리 서비스 (V718)

**상태**: 채택  
**날짜**: 2026-05-28  
**버전**: V718 / v12.3.2

## 맥락

ZeroTrustTokenService(V717)가 토큰 서명을 보장한다면, TenantAuthority 는 "그 토큰을 소지한 주체가 특정 테넌트의 리소스에 접근할 자격이 있는가"를 판단한다.

## 결정

1. **TenantRecord** DTO: `tenant_id`, `display_name`, `allowed_roles(frozenset)`, `active`, `tags`
2. **CRUD**: `register / get / disable / enable / add_role / remove_role / list_all`
3. **접근 판정**: `authorize(claims, required_role, target_tenant_id) → AccessDecision`
4. 크로스-테넌트 접근은 `AccessDecision(granted=False, reason="Cross-tenant access denied")` 반환
5. `authorize_or_raise()` 편의 메서드: denied 시 `AccessDeniedError` 발생

## 결과

- `literary_system.security.TenantAuthority` 완성
- 33 TC PASS (TC01~TC33)
- 다음 단계: V719 ZeroTrustMiddleware (ADR-180)
