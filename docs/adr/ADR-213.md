# ADR-213: G_HUMAN_GT_ALIGNMENT — 인간 GT(작가 평가) 정합 게이트

## 상태
ACCEPTED (V750)

## 맥락
공식(구조 sanity)·LLM 패널(73% 프록시)만으로는 "절대 타당성"을 판정할 수 없다(LW-5: LLM 심판이 거장 절제 문체를 감점). 인간 작가 GT가 유일한 절대 기준이나 운영 프로토콜 설계가 부재했다(허브 0건).

## 결정
인간 GT 운영 인프라(`validation/human_gt.py`)와 게이트 `G_HUMAN_GT_ALIGNMENT`를 도입한다. 게이트는 평가자 간 Krippendorff α≥0.6(substantial)을 요구하고, 패널-인간 일치율을 보고하여 패널 캘리브레이션 신호로 사용한다.

## 근거
- 쌍대만(절대점수 금지, G_NO_ABSOLUTE_REWARD 정합) · 블라인드 · 실제 DB 씬 앵커 강제(LLM 회상 레퍼런스 차단, WP-4 규칙) · 수집만(학습은 loop-C, Phase F).
- 기존 자산 재사용: pairwise.bt_scores, constitution.KrippendorffAlpha.

## 결과
- 긍정: P2 절대검증의 비어 있던 설계·코드를 채움. 패널 캘리브레이션 신호 확보.
- 부정/유의: 다중 평가자 표본 노이즈 → 만장일치 가중·N 확대 필요. 실 작가 섭외는 개발자 액션.

## 대안 검토
- LLM-judge 단독: 순환·문체 편향(기각). 절대 점수 채점: G_NO_ABSOLUTE_REWARD 위반(기각).
