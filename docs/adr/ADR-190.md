# ADR-190: G89 Chaos Resilience Gate (V729)

**날짜**: 2026-05-29
**상태**: ACCEPTED
**범위**: literary_system/gates/chaos_resilience_gate.py

## 배경

V724~V728 (SP-D.3)에서 구현된 Chaos Engineering 레이어의 완성도를 검증하는
공식 릴리즈 게이트가 필요하다.

- V724: ChaosEngine + FaultInjector (ADR-185)
- V726: ChaosScenario (ADR-187)
- V727: ChaosCircuitBreaker (ADR-188)
- V728: ChaosRunner + AutoRecovery (ADR-189)

또한 G88 (ZeroTrustSecurityGate)이 release_gate.py에 미등록 상태임을
V729 Preflight 분석에서 확인 → 동시 등록 처리.

## 결정

1. `literary_system/gates/chaos_resilience_gate.py` 신설 — G89 (6 CP)
2. `release_gate.py`에 G88 + G89 동시 등록 (87 Gates → 87 PASS 목표)

## 체크포인트 설계 (CR-1 ~ CR-6)

| CP | 검증 대상 | 기준 |
|----|-----------|------|
| CR-1 | ChaosEngine | 등록·활성화·주입·이력·통계 정상 |
| CR-2 | FaultInjector | BEFORE/AFTER/wrap 주입 정상 |
| CR-3 | ChaosScenario | preset 실행·ScenarioState 전이 정상 |
| CR-4 | ChaosCircuitBreaker | CLOSED→OPEN 전이·reset 정상 |
| CR-5 | ChaosRunner+AutoRecovery | resilience_ratio ∈ [0,1] · RECOVERED |
| CR-6 | G88 ZeroTrust 통합 | run_zero_trust_security_gate() PASS · ZT-7 확인 |

## 결과

- 85 Gates → 87 Gates (G88+G89 등록)
- Chaos Engineering 전 계층 공식 검증
- SP-D.3 V730 Exit Gate 진입 조건 충족

## 연결 다이어그램 (신경망 관점)

```
G89 [ChaosResilienceGate]
  └─ CR-1: chaos_engine ← fault_injector
  └─ CR-2: fault_injector ← chaos_engine
  └─ CR-3: chaos_scenario ← chaos_engine
  └─ CR-4: chaos_circuit_breaker ← agents/circuit_breaker + chaos_engine
  └─ CR-5: chaos_runner ← chaos_scenario + chaos_engine (AutoRecovery)
  └─ CR-6: zero_trust_security_gate ← security/ 전체 + chaos/
```
