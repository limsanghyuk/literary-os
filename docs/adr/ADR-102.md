# ADR-102: MetaLearnerCycle V642 — DataAugmentationController 실 통합 + α 안정성 측정

**날짜**: 2026-05-26  
**상태**: Accepted  
**버전**: V642 (v11.13.0)  
**관련 ADR**: ADR-081 (DataAugmentationController), ADR-101 (MetaLearnerCycle V641)

---

## 배경

V641에서 MetaLearnerCycle은 Krippendorff α 측정과 `augment_ratio` 동적 조정 기능을 도입했다.  
그러나 조정된 비율이 실제 `DataAugmentationController.augment()` 호출로 이어지지 않아,  
훈련 데이터 증강 루프가 실질적으로 닫히지 않는 상태였다.

V642는 이 갭을 해소하고, 다사이클 간 α 분산 추적(`alpha_stability()`)을 추가한다.

---

## 결정

### D1: `run_cycle()` `sample_texts` 파라미터 추가

```python
def run_cycle(
    self, l_final, rater_data=None, r_scene=None, *,
    cycle_number=None,
    sample_texts=None,   # [V642 신규]
) -> CycleReport:
```

- `sample_texts` 제공 시 `DataAugmentationController.augment()` 실제 호출
- `augment_count=DEFAULT_AUGMENT_COUNT_PER_CYCLE`, `augment_ratio=aug_after` 적용
- 결과 `AugmentationBatch`를 `CycleReport.augmentation_batch`에 포함

### D2: `CycleReport.augmentation_batch` 필드 추가

```python
@dataclass
class CycleReport:
    ...
    augmentation_batch: Optional[AugmentationBatch] = None  # [V642 신규]
```

- 기본값 `None` — 하위 호환성 유지 (V641 테스트 33/33 PASS 보장)
- `augmentation_performed` 프로퍼티: `batch is not None and augmented_count > 0`

### D3: `AlphaStability` 데이터클래스 + `alpha_stability()` 메서드

```python
@dataclass
class AlphaStability:
    cycle_count: int
    alpha_values: List[float]
    mean_alpha: float
    variance: float           # population variance
    is_stable: bool           # variance < ALPHA_STABILITY_MAX_VAR

ALPHA_STABILITY_MAX_VAR: float = 0.01  # 분산 허용 최대값
ALPHA_STABILITY_MIN_CYCLES: int = 2   # 최소 측정 사이클
```

- `statistics.pvariance()` 사용 (모집단 분산)
- 2사이클 미만 → `None` 반환

### D4: `run_n_cycles()` `sample_texts_list` 파라미터 확장

- V645 통합 검증에서 사이클별 증강 텍스트 지정 가능하도록 확장
- 미제공 시 기존 동작 그대로 유지
