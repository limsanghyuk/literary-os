# ADR-041: LOSDB Phase A — SQLiteRealAdapter REAL 어댑터 + LOSDB CLI

**버전**: V582  
**상태**: Accepted  
**날짜**: 2026-05-20  
**작성자**: limsanghyuk  

---

## Context

V581에서 LOSDB Phase A를 시작하여 SchemaRegistry와 MOCK-only MigrationManager를 구축했다.
그러나 실제 데이터베이스 마이그레이션 실행 능력이 없어, Phase A를 완성하려면 REAL 어댑터와 CLI가 필요하다.

## Decision

1. **SQLiteRealAdapter** 구현 (`literary_system/db/sql_real_adapter.py`)
   - `BaseMigrationAdapter`의 REAL 구현체
   - stdlib `sqlite3`만 사용 (외부 의존 0)
   - `sqlite:///:memory:` 및 `sqlite:///path` URL 지원
   - `losdb_migrations` 테이블로 마이그레이션 이력 영속화
   - SchemaRegistry와 연동 (`register` + `record_migration`)
   - `apply()`, `rollback()`, `list_applied()`, `table_exists()`, `close()` 제공

2. **LOSDB CLI** 구현 (`literary_system/db/cli.py`)
   - argparse 기반, `main(argv)` 진입점
   - G32 준수: `print()` 사용 금지, `_emit()`/`sys.stdout.write()` 사용
   - `--json` 최상위 플래그 (`losdb --json status`)
   - 서브커맨드: `status`, `analyze`, `health`, `migrate`
   - CI 통합 가능 (반환값 0=성공, 1=실패)

3. **Gate G41** (`sql_real_adapter_g41`) 신설
   - Layer L1 (위생 규칙 강제)
   - 7개 검증 포인트: import, MOCK check_connection, REAL apply, list_applied, rollback, CLI parser, CLI status

## Consequences

- LOSDB Phase A 완성: MOCK + REAL 양 모드 동등 지원
- SQLite 의존은 sqlite3(stdlib)만 — 추가 패키지 불필요
- V583+ Phase B에서 Graph/Vector REAL 어댑터 구현 예정
- CLI는 향후 ops 스크립트, CI 체크, 마이그레이션 자동화에 활용
