# ADR-198: FLOrchestrator — FL E2E 파이프라인 오케스트레이터 (V736)

**날짜**: 2026-05-29  
**상태**: ACCEPTED  
**서브페이즈**: SP-D.4

## Context

V732~V735에서 FLCoordinator, FedAvgAggregator, FLClient, FLPrivacyNoise가 개별 모듈로 구현되었다.  
각 컴포넌트를 직접 조합하는 작업이 중복되므로 단일 진입점이 필요하다.

## Decision

`literary_system/federation/fl_orchestrator.py`에 `FLOrchestrator` 클래스를 구현한다.

**핵심 API**:
- `FLOrchestrator(clients, max_rounds, dp_epsilon, dp_delta, use_privacy)` — 컴포넌트 초기화 및 클라이언트 등록
- `run_federation() → FLRunResult` — 전체 FL 루프 실행
  1. `coordinator.start_round()` → 참여 클라이언트 결정
  2. `client.receive_global_model()` → 전역 모델 broadcast
  3. `client.train()` → 로컬 훈련
  4. (선택) `privacy.privatize()` → DP 노이즈 적용
  5. `aggregator.aggregate()` → FedAvg
  6. `coordinator.finalize_round()` → 수렴 판정
  7. `coordinator.should_continue()` → 반복 결정

**FLRunResult 필드**: `total_rounds`, `converged`, `final_global_loss`, `loss_trend`, `privacy_budget`, `round_summaries`, `elapsed_seconds`

## Consequences

- FL PoC E2E를 단일 호출로 실행 가능
- G90 FL Gate (fl_gate.py FL-5)에서 검증
- 추후 실 LLM 어댑터 연결 시 클라이언트 구현체만 교체하면 됨
