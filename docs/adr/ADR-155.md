# ADR-155: SP-D.1 ObservabilityDashboard — 멀티패널 운영 지표 계층

**Date:** 2026-05-28
**Status:** Accepted
**Deciders:** SP-D.1 Core Team
**Supersedes:** —

## Context

SP-D.1 관측성 스택(V688~V691)은 TraceContext 전파, OTel SDK 어댑터, Prometheus /metrics
연동, TraceSampler를 구축했다. 그러나 이 지표들을 운영자가 한 눈에 파악할 수 있는
집계 계층이 없었다. V692에서 멀티패널 대시보드 계층을 도입해 5개 핵심 메트릭을 단일
인터페이스로 관리한다.

## Decision

`literary_system/ops/observability_dashboard.py` 에 다음을 구현한다:

| 클래스 | 역할 |
|--------|------|
| `ObsMetricPoint` | 단일 측정 포인트 (value + timestamp + labels) |
| `ObsAlert` | 임계값 알림 (fire_when_above / fire_when_below) |
| `ObsDashboardPanel` | 단일 메트릭 패널 — 시계열 기록 + 평균/최대/최소 |
| `ObservabilityDashboard` | 패널 집합 + 알림 집계 + health() + summary() |
| `create_spd1_dashboard()` | SP-D.1 표준 5패널 팩토리 |

### SP-D.1 표준 패널

| 패널 이름 | 타입 | 알림 조건 | 심각도 |
|-----------|------|----------|--------|
| `gates_pass_ratio` | GAUGE | < 0.95 | CRITICAL |
| `spans_exported_total` | COUNTER | ≥ 50,000 | WARNING |
| `active_traces` | GAUGE | ≥ 100 | WARNING |
| `p99_span_duration_ms` | GAUGE | ≥ 500 ms | WARNING |
| `trace_error_ratio` | GAUGE | ≥ 0.05 | CRITICAL |

### fire_when_below 설계

`ObsAlert.fire_when_below=True` 는 "값이 임계값 **아래**로 떨어지면 발화" 의미다.
`gates_pass_ratio` 처럼 **높을수록 좋은** 지표에 필수적이다.

## Consequences

- 운영자는 `dashboard.summary()` 하나로 전체 관측성 상태 파악 가능
- `health()` → "healthy" / "warning" / "degraded" 3단계 상태
- `record_otel_metrics()` / `record_gate_metrics()` 헬퍼로 크로스 모듈 연동 단순화
