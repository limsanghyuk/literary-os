# ADR-069: MultiWorkCIMV2 설계 결정

**상태**: Accepted  
**날짜**: 2026-05-23  
**버전**: V609 (v10.14.0)  
**ADR 계열**: SP-B.3 MultiWork 협업 레이어

---

## 컨텍스트

V567에서 구현된 `MultiWorkCIM` (v1)은 프로젝트별 독립 CIM과 공유 캐릭터 전역 집계를 지원한다.  
V607~V608에서 `SharedCharacterDBV2` (RewardTrace 포함)와 `MultiWorkOrchestratorV2`가 추가되었다.  
V609에서는 CIM 레이어를 보상 정보와 연동하고, 프로젝트 간 CIM 유사도 분석 기능을 추가한다.

---

## 결정

### D-1: 상속 기반 v2 (비파괴적 확장)

`MultiWorkCIMV2`는 `MultiWorkCIM`을 상속하여 v1 API를 완전 보존한다.  
`_project_cims` dict를 `ProjectCIMV2` 인스턴스로 채워 v1 메서드(`record`, `global_weight`, `stats`)가
v2 인스턴스를 통해 작동하도록 duck-typing 활용.

### D-2: 보상 가중치 연동 (옵셔널)

`char_db=None`이면 기본 reward 0.5를 사용한다.  
`record_v2(reward=None)`일 때 char_db가 있으면 두 캐릭터의 RewardTrace 평균을 사용.  
`reward_weighted_weight = weight × clamp(avg_reward, 0, 1)`

### D-3: CIMEntryV2 설계

```python
@dataclass
class CIMEntryV2(CIMEntry):
    reward_weighted_weight: float = 0.0
    
    def update_with_reward(self, decay, avg_reward):
        count += 1
        weight = 1 - exp(-decay * count)
        reward_weighted_weight = weight * clamp(avg_reward, 0, 1)
```

v1 `CIMEntry`를 상속해 기존 직렬화 코드와 호환.

### D-4: CIM 스냅샷 ID 생성

```
snapshot_id = f"{project_id}-cim-{sha256(project_id+timestamp+label)[:8]}"
```

UUID 대신 결정론적 prefix+hash 방식으로 디버깅 편의성 향상.

### D-5: 프로젝트 간 CIM 코사인 유사도

공유 캐릭터 쌍 (i,j)의 v1 weight 벡터로 코사인 유사도 계산:

```
cos_sim = dot(vec_A, vec_B) / (||vec_A|| × ||vec_B||)
```

- 두 벡터 모두 0이면 cos_sim = 1.0 (동일 상태)
- 한쪽만 0이면 cos_sim = 0.0 (완전 상이)
- `is_compatible = weight_delta_max < conflict_threshold (0.30)`

### D-6: 직렬화 스키마 (7-key)

```json
{
  "version": "2.0.0",
  "exported_at": "ISO-8601",
  "decay": 0.95,
  "conflict_threshold": 0.30,
  "project_cims": { "pid": ProjectCIMV2.to_dict() },
  "snapshots": { "pid": [CIMSnapshot.to_dict()] },
  "total_interactions": N
}
```

## 결과

- v1 `MultiWorkCIM` API 완전 호환 (register, record, global_weight, aggregate_warm_start, stats)
- 보상 가중 CIM: `record_v2()` + `reward_weighted_global_weight()`
- 프로젝트 간 유사도: `inter_project_cim_score()` → `InterProjectCIMScore`
- 스냅샷: `snapshot_project()` / `restore_project()`
- 직렬화: `export_state_v2()` / `import_state_v2()`
- 22 TC ALL PASS, 6458 tests, EA-6 PASS
