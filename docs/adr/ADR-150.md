# ADR-150: G82 Static Type Safety Gate 신설

**Status:** Accepted

**Date:** 2026-05-28  
**Author:** limsanghyuk  
**Supersedes:** —  
**Related:** ADR-147 (CI 4단), ADR-148 (type_stubs), ADR-149 (pre-commit)

---

## Context

SP-D.1 Gate G82는 정적 타입 안전성 인프라 전체를 단일 Gate로 검증한다.
개별 구성 요소(pre-commit, type_stubs, mypy config, lock file)가 각각
독립적으로 존재하더라도 통합적으로 검증하는 Gate가 없으면:
- 특정 구성 요소 누락을 릴리즈 후에야 감지 가능
- 배포 Pipeline(release_gate.py)과의 통합 불가

---

## Decision

`literary_system/gates/static_type_safety_gate.py`를 신설하여
G82 Gate를 5축으로 구성한다.

| 축 | 검증 항목 | 실패 조건 |
|----|----------|----------|
| ST-1 | .pre-commit-config.yaml + 4 hooks | 파일 없음 또는 hook 누락 |
| ST-2 | core/type_stubs.py + 4 Protocol | 파일 없음 또는 Protocol 누락 |
| ST-3 | pyproject.toml [tool.mypy] strict=true | 섹션 없음 또는 strict 누락 |
| ST-4 | requirements-lock.txt + 핵심 패키지 | 파일 없음 또는 패키지 누락 |
| ST-5 | ST-1~ST-4 ALL PASS | 1개 이상 FAIL |

---

## Rationale

1. **통합 검증**: 4개 인프라 구성요소를 단일 Gate로 묶어 누락 방지
2. **release_gate.py 통합**: `run_g82_gate()` 딕셔너리 인터페이스로 배포 파이프라인 연동
3. **LLM-0 준수**: 외부 API 호출 없음, 파일시스템 검사만 수행
4. **독립 실행**: `python3 literary_system/gates/static_type_safety_gate.py` 로 직접 실행 가능

---

## Gate 구조

```python
GATE_ID = "G82"

@dataclass
class TypeSafetyCheckpoint:
    name: str
    passed: bool
    detail: str = ""

@dataclass
class StaticTypeSafetyReport:
    checkpoints: List[TypeSafetyCheckpoint]
    gate_passed: bool
    passed_count: int  # property
    failed_count: int  # property

def run_static_type_safety_gate() -> StaticTypeSafetyReport: ...
def run_g82_gate() -> dict: ...  # release_gate.py 통합용
```

---

## Consequences

- 80 Gates -> 81 Gates (G82 추가, 총 Gates 수 1 증가)
- 향후 pre-commit hook 추가 시 REQUIRED_HOOKS 집합 확장으로 대응
- ST-5는 ST-1~ST-4 ALL PASS 논리: 하나라도 실패하면 gate_passed=False
