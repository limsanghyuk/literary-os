# ADR-112: AgentEnsembleEvaluator — Multi-Agent 후보 집계 설계

**날짜**: 2026-05-26  
**상태**: 승인됨  
**버전**: V652 (SP-C.2)  
**관련 ADR**: ADR-083 (NarrativeFitnessArbiter), ADR-110 (AgentCoordinator)

---

## 컨텍스트

SP-C.2 멀티-에이전트 앙상블 파이프라인은 동일 씬에 대해 복수의
`CoordinatorResult` 후보를 생성할 수 있다. 최종 씬을 결정하기 위해
후보 간 집계·비교·병합 로직이 필요하다.

## 결정

`AgentEnsembleEvaluator` 클래스를 신설한다.

- **임계값**:
  - `SELECT_THRESHOLD = 0.80`: 최고점 후보를 그대로 채택
  - `MERGE_THRESHOLD  = 0.55`: 상위 2개 후보 텍스트를 단락 교차 병합
  - 미달 시 `REJECT` (빈 텍스트)
- **점수 계산** (`_score_candidate`):
  - `critic_score` 기준 + 텍스트 길이 보너스 최대 +0.05 (300자 이상 시 만점)
- **MERGE** (`_merge_texts`):
  - 상위 2개 후보 단락을 교차 배치, 가중치 없이 단순 인터리빙
- **`EnsembleEvalResult`**: `to_dict` / `from_dict` 직렬화 지원

## 결과

- `NarrativeFitnessArbiter` (ADR-083)의 결정 어휘(SELECT/MERGE/REJECT)와 통일
- 단일 후보 입력도 동일 로직으로 처리 (일관성)
- 빈 후보 목록 → REJECT (safe-default)

## LLM-0 준수

외부 API 직접 호출 없음. 모든 점수는 `last_critic_score` 입력값에서 파생.
