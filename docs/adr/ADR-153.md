# ADR-153: G83 Observability Foundation Gate 신설 (V690, SP-D.1)

## Status
Accepted — V690 (2026-05-28)

## Context
SP-D.1 관측가능성 기반 (V681~V695)의 구현 완전성을 검증하는 자동화 Gate가 필요하다.
V688/V689에서 도입한 TraceContext, OtelSdkAdapter, PrometheusExporter, 
PrometheusTraceExtension의 핵심 API가 코드베이스에 실제로 동작하는지 확인한다.

## Decision

### G83 5축 체크포인트
| 축 | 검사 대상 | 합격 기준 |
|----|-----------|-----------|
| OB-1 | trace_context.py | W3C TraceContext API 완전성 + new_trace_context() 동작 |
| OB-2 | otel_adapter.py | OtelSdkAdapter span export 동작 |
| OB-3 | prometheus_exporter.py | PrometheusExporter.render() gates_total 포함 |
| OB-4 | prometheus_trace_extension.py | TraceAwareExporter + MetricsEndpoint traceparent |
| OB-5 | D-M-02 통합 | inject→extract 왕복 + /metrics child span trace_id 상속 |

### 구현 위치
```
literary_system/gates/observability_foundation_gate.py
  ├── ObsCheckpoint (dataclass)
  ├── _check_ob1_trace_context()
  ├── _check_ob2_otel_adapter()
  ├── _check_ob3_prometheus_exporter()
  ├── _check_ob4_prometheus_trace_extension()
  ├── _check_ob5_dm02_integration()
  └── run_g83_gate() → Dict[str, Any]
```

### release_gate.py 등록
```python
GATES.append((
    "observability_foundation_g83",
```

→ 총 Gates: 83 → **84** (G83 신규)

## Consequences

### 긍정적
- SP-D.1 관측가능성 기반의 회귀 방지 자동화
- OB-5 D-M-02 통합 검증으로 W3C TraceContext propagation 보증
- 독립 실행 가능 (`python3 ... observability_foundation_gate.py`)

### 부정적/트레이드오프
- 외부 HTTP 서버 없이 시뮬레이션 검증 — 실제 네트워크 경계 미커버
