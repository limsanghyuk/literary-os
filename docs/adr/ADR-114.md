# ADR-114: MAEMultiWorkGate G66 — 3작품 동시 앙상블 성능 게이트

**상태**: Accepted  
**날짜**: 2026-05-27  
**버전**: V654  
**구현 파일**: `literary_system/ensemble/mae_multiwork_gate.py`  
**게이트**: G66

---

## 배경

SP-C.2 Multi-Agent Ensemble Writing System이 단일 작품 처리(G64/G65)를 완성한 후,
실제 프로덕션 시나리오인 "3개 이상 작품 동시 처리" 성능을 검증할 게이트가 필요하다.

## 결정

**MAEMultiWorkGate (G66)** 를 구현한다.

| 항목 | 기준 |
|------|------|
| 최소 프로젝트 수 | 3개 |
| P95 지연 임계값 | ≤ 8.0초 |
| 동시 실행 | ThreadPoolExecutor (max_workers=4) |
| LLM-0 | 외부 API 호출 없음 |

## 구현 세부

- `ProjectRunSpec`: project_id + scenes + max_rounds
- `ProjectRunResult`: project_id, latency_sec, scene_count, success, ensemble_scores
- `MultiWorkGateResult`: passed, p95/p50/max latency, all_latencies, failure_reason
- `_percentile()`: 선형 보간법 백분위수
- `run_gate()`: 게이트 단일 실행
- `benchmark()`: repeat 회 반복 실행 후 합산 P95 계산
- coordinator=None이면 STUB_SCENE_LATENCY_SEC(0.05s) 기반 스텁 실행

## 결과

- `to_dict()` / `from_dict()` 직렬화 지원
- G66 PASS: P95 ≤ 8.0s AND project_count ≥ 3
- G66 FAIL: P95 > 8.0s 또는 프로젝트 수 부족 또는 타임아웃

## 대안 검토

| 방안 | 이유 |
|------|------|
| asyncio 기반 | C 확장 미지원 환경 호환성 이슈 → 기각 |
| multiprocessing | GIL 없지만 메모리 오버헤드 → 기각 |
| ThreadPool (채택) | I/O 바운드 시나리오에 적합, 단순 구현 |
