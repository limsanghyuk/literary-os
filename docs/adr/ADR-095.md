# ADR-095: Grafana + Prometheus 모니터링 스택 도입

**상태**: Accepted  
**날짜**: 2026-05-25  
**버전**: V628 (v10.33.0)  
**관련**: ADR-057 §5 (TrainPlane/ServePlane 격리), ADR-093 (TrainPlane Helm), ADR-094 (ServePlane Helm)

---

## 배경

V626/V627에서 TrainPlane + ServePlane Helm chart를 정적 검증했다.
운영 시점의 동적 상태(Gate PASS율, TC 수, GPU 비용, 서빙 지연시간)를 실시간으로 가시화하기 위해
Prometheus + Grafana 기반 모니터링 스택을 Literary OS에 통합한다.

---

## 결정

### 1. PrometheusExporter 구현

`literary_system/ops/prometheus_exporter.py`에 외부 의존성 없는 순수 Python 메트릭 익스포터를 구현한다.

**노출 메트릭 (10종):**

| 메트릭 이름 | 타입 | 설명 |
|------------|------|------|
| `literary_os_build_info` | gauge | 빌드 버전/Phase 레이블 |
| `literary_os_gates_total` | gauge | 총 Release Gate 수 |
| `literary_os_gates_passed` | gauge | PASS Gate 수 |
| `literary_os_gates_pass_ratio` | gauge | PASS 비율 (0~1) |
| `literary_os_tests_total` | gauge | 총 TC 수 |
| `literary_os_cost_slo_used_usd` | gauge | GPU 비용 SLO 사용액 ($) |
| `literary_os_cost_slo_hard_usd` | gauge | GPU 비용 SLO 하드 한도 ($) |
| `literary_os_serve_latency_ms` | gauge | ServePlane 평균 추론 지연시간 (ms) |
| `literary_os_train_jobs_active` | gauge | 활성 TrainPlane 학습 작업 수 |
| `literary_os_lora_promoted_count` | gauge | PROMOTED 상태 LoRA 모델 수 |

### 2. Grafana 대시보드 JSON

`deploy/monitoring/grafana_dashboard.json` — 6개 패널:
- Gate PASS 수 (stat)
- Gate PASS 비율 (gauge, 임계값 경고)
- 총 TC 수 (stat)
- GPU 비용 SLO 사용액 (gauge, 임계값 경고)
- ServePlane 추론 지연시간 추이 (timeseries)
- PROMOTED LoRA 모델 수 추이 (timeseries)

### 3. Prometheus 설정

`deploy/monitoring/prometheus.yml` — 3개 scrape job:
- literary-os-serve-plane (literary-serve NS, 15s)
- literary-os-train-plane (literary-train NS, 30s)
- literary-os-cost-monitor (비용 SLO, 60s)

---

## 결과

### 긍정적 결과
- Gate/TC/비용/지연시간 실시간 가시화
- 외부 의존성 없는 순수 Python 익스포터 (prometheus_client 불필요)
- TrainPlane/ServePlane 네임스페이스 분리 모니터링

### 부정적 결과
- 실제 Prometheus/Grafana 클러스터 설치는 별도 운영 작업 필요
- 메트릭 scrape endpoint는 서비스 배포 후 활성화

---

## 준수 사항

- **LLM-0**: PrometheusExporter 내부 외부 LLM 호출 없음
- **G32**: print() 없음 — logger.info/error 사용
- **G37**: PrometheusExporter, MetricSnapshot, MonitoringConfig — literary_system/ 내 유일 클래스명
- **DEV_MODE**: 항상 False 기본 (ADR-034)
