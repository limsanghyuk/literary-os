# ADR-178: ZeroTrustTokenService — HMAC-SHA256 기반 토큰 서비스 (V717)

**상태**: 채택  
**날짜**: 2026-05-28  
**버전**: V717 / v12.3.1

## 맥락

SP-D.3 G88 Zero-Trust Security 레이어 구축의 첫 번째 단계. 플러그인·에이전트·API 요청 모두에 단기 서명 토큰을 적용하여 "기본 신뢰 없음(Never Trust, Always Verify)" 원칙을 실현한다.

## 결정

1. **HMAC-SHA256** 서명 알고리즘 채택 (`alg=HS256`)
2. 토큰 형식: `header_b64url.payload_b64url.signature_b64url` (3-part)
3. 필수 클레임: `sub`, `tid`(tenant_id), `iat`, `exp`, `jti`
4. 기본 TTL = **300 초**, 서비스 생성 시 override 가능
5. `jti` 기반 **Replay-attack 방지** (메모리 nonce store)
6. `verify_tenant()` 메서드로 크로스-테넌트 접근 차단
7. `TokenExpiredError(TokenValidationError)` 예외 계층 도입

## 결과

- Zero-Trust 토큰 발급·검증·갱신·폐기 API 완성
- `literary_system.security` 패키지 신설 (G88 레이어 기반)
- 33 TC 전체 PASS (TC01~TC33)
- 다음 단계: V718 TenantAuthority (ADR-179)
