# ADR-233: 클라우드 실 배선 — dispatch_training → RealRunPodAdapter

## 상태
ACCEPTED (V773, Phase E.4 확장)

## 맥락
V772에서 RealRunPodAdapter(실 RunPod)를 만들었으나 디스패처 경로(dispatch→RLAIFTrigger→LoRAJobRunner→get_adapter)가 여전히 Mock을 호출. "CLOUD 선택 → 실 학습"이 끝까지 안 이어지는 갭. 로컬·하이브리드는 유지 요구.

## 결정
어댑터 주입 경로 3곳 개통:
- LoRAJobRunner.__init__(adapter=None): 주입 시 사용, 없으면 get_adapter(기본 Mock).
- RLAIFTrigger.__init__(adapter=None): runner로 전달.
- dispatch_training(real=False, api_key=None): real 또는 RUNPOD_API_KEY 존재 시 클라우드 단계가 RealRunPodAdapter로. CLOUD/AUTO/HYBRID(StageB) 모두 적용. LOCAL은 실 RunPod 미사용 유지.
안전: dry_run=True(기본)면 실 어댑터라도 네트워크 미호출. 실 학습=dry_run=False+키. 키는 결과 직렬화 미포함.

## 결과
13 TC PASS. **3방식 모두 사용자 선택→끝까지 실행**: LOCAL(4070 train_local) / CLOUD(키→실 RunPod) / HYBRID(로컬 선별+클라우드 강화). 키 있으면 자동 실 어댑터, 없으면 안전 dry_run. LLM-0 유지.
## 대안
registry에 실 어댑터 등록(전역 교체): dry_run 기본 안전성 훼손 위험(기각). 주입 방식으로 명시적 제어.
