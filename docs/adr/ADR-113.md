# ADR-113: AgentSafetyGuard — 앙상블 파이프라인 4축 안전 검증

**날짜**: 2026-05-26  
**상태**: 승인됨  
**버전**: V653 (SP-C.2)  
**관련 ADR**: ADR-097 (G61 Exit Gate), ADR-112 (AgentEnsembleEvaluator)

---

## 컨텍스트

멀티-에이전트 앙상블 파이프라인은 청사진 입력과 씬 출력 양방향에서
안전 문제가 발생할 수 있다. `SafetyRegressionV2` (F9 의무 조치)와
일관된 4축 검증이 필요하다.

## 결정

`AgentSafetyGuard` 클래스를 신설한다.

- **검사 축** (4개, `SAFETY_AXES`):
  1. `self_harm`   — 자해·자살 관련 표현 (고위험)
  2. `hate_speech` — 혐오·차별 발언 (고위험)
  3. `pii`         — 개인식별정보(이메일·전화·신용카드·주민번호) (중위험)
  4. `copyright`   — 저작권 표시 문구 (저위험)
- **API**:
  - `pre_check(blueprint_dict)` — 입력 청사진 검사
  - `post_check(result_dict)`   — 출력 텍스트 검사
  - `check_text(text)`          — 임의 텍스트 직접 검사
- **구현 방식**: 규칙 기반 정규식 (LLM-0 준수, 외부 API 없음)
- **심각도 계층**: `none < low < medium < high`; 위반 시 최대 심각도 반환
- **축 비활성화**: `enabled_axes` 인자로 테스트 모킹 지원

## 결과

- `SafetyCheckResult.passed=False` → 파이프라인 상위 호출자가 게이트 차단 결정
- `ENABLED_AXES` 클래스 변수로 프로덕션/테스트 전환 용이

## LLM-0 준수

정규식 패턴만 사용. 외부 서비스·API 호출 없음.
