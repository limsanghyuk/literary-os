## Decision

### §A — ConflictPolicy + CharacterConflictResolver

#### ConflictPolicy (str, Enum)
5종 정책으로 캐릭터 충돌 해결 전략을 명시적으로 선언:

| Policy    | 동작                                        |
|-----------|---------------------------------------------|
| RENAME    | 충돌 캐릭터에 `_renamed` 접미사 신규 등록   |
| MERGE     | 두 버전 traits 합집합으로 병합 (union)      |
| FORK      | 원본 유지 + `_fork` 파생 캐릭터 신규 등록  |
| BLOCK     | RuntimeError 발생 — 수동 해결 강제          |
| ESCALATE  | resolved=False 마킹 — 상위 에스컬레이션     |

#### CharacterConflictResolver
```python
class CharacterConflictResolver:
    def __init__(self, db: SharedCharacterDBV2) -> None
    def resolve(self, conflict_id: str, policy: ConflictPolicy) -> Dict[str, Any]
```

반환 dict 스키마:
```json
{
  "conflict_id": "str",
  "policy": "str",
  "resolved": "bool",
  "action": "str",
  "result_ids": ["str", ...]
}
```

충돌 생성 절차 (`detect_conflicts` 작동 조건):
1. `add_character(cid, ...)` — 초기 traits 등록
2. `register_project_state("proj_a", cid)` — 기준점 A 저장
3. `update_traits(cid, new_traits)` — 신규 key 추가로 hash 변경
4. `register_project_state("proj_b", cid)` — 기준점 B 저장
5. `update_traits(cid, more_traits)` — 현재 hash ≠ A, B
6. `detect_conflicts(cid, "proj_a", "proj_b")` → ConflictRecord

### §B — WorkloadProfile + SLO + schedule()

#### WorkloadProfile (str, Enum)
```python
class WorkloadProfile(str, Enum):
    SINGLE = "SINGLE"   # SLO: 3,000 ms
    DUAL   = "DUAL"     # SLO: 5,000 ms
    TRIPLE = "TRIPLE"   # SLO: 8,000 ms
```

SLO 상수:
```python
SLO_SINGLE_MS: int = 3_000
SLO_DUAL_MS:   int = 5_000
SLO_TRIPLE_MS: int = 8_000
```

#### classify_workload()
```python
def classify_workload(project_ids: List[str]) -> WorkloadProfile:
    n = len(project_ids)
    if n <= 1: return WorkloadProfile.SINGLE
    if n == 2: return WorkloadProfile.DUAL
    return WorkloadProfile.TRIPLE
```

#### schedule()
```python
def schedule(
    project_ids: List[str],
    scene_ms_each: int = 1000,
) -> ScheduleResult
```

스케줄링 전략:
- SINGLE: 직렬 실행, estimated = scene_ms_each × n
- DUAL: 2-way 병렬, estimated = scene_ms_each × ceil(n/2)
- TRIPLE: ceil(N/2) 슬롯 라운드로빈, estimated = scene_ms_each × ceil(n/2)

## Bug Fix

### `_resolve_merge` — `conflicting_ids` AttributeError 수정
기존 코드에서 `rec.conflicting_ids` 참조 오류 발생. `ConflictRecord`에 해당 필드 없음.
수정: 현재 캐릭터 traits 상태 기준으로 resolved 처리 (union = 현재 상태 유지).

## Consequences

**Positive:**
- SP-B.3 retrofit 3모듈 완결 — ConflictPolicy/WorkloadProfile/AdvSeed 추상화 확립
- TC 60/60 PASS (test_v622_sp_b3_retrofit.py)
- G58 (SP-B.3 LoRA Stacking) + G59 (SP-B.3 Exit Gate) 회귀 0건 유지
- `_resolve_merge` 버그 수정으로 MERGE 정책 정상 작동

**Constraints:**
- `detect_conflicts` 작동 조건: `register_project_state` 두 번 + 서로 다른 key 추가 필수
- ESCALATE 정책은 `resolved=False` — 자동 해결 없음, 수동 에스컬레이션 전용
- RewardModelV2는 LLM-0 원칙 준수 (외부 LLM 호출 없음)
