# ADR-075: PatternLibraryV2 — 압축(Compression) + 랭킹(Ranking) 설계

**Status**: Accepted  
**Date**: 2026-05-26  
**Deciders**: limsanghyuk  
**Tags**: SP-C.1, PatternLibrary, LOSDB, 압축, 랭킹

---

## Context

LOSConstitutionV2의 Bayesian Optimization이 탐색한 가중치(w1~w5)는
반복 학습 사이클마다 패턴(장면 구성 패턴, 문장 리듬 패턴 등)을 새롭게 발견한다.
패턴이 무제한으로 누적되면:
1. **중복 패턴** 증가 → 추론 비용 증가.
2. **랭킹 없음** → 고품질 패턴과 저품질 패턴 구분 불가.
3. **영속화 없음** → 세션 간 패턴 손실.

PatternLibraryV2는 이 세 가지 문제를 해결하기 위해 설계됩니다.

---

## Decision

### 1. 데이터 모델: PatternEntry

```python
@dataclass
class PatternEntry:
    pattern_id: str        # UUID4
    label: str             # 패턴 레이블
    description: str       # 사람이 읽을 수 있는 설명
    embedding: List[float] # 압축에 사용하는 벡터
    freq: int              # 관찰 빈도
    entropy_weight: float  # Constitution 엔트로피 가중치
    created_at: str        # ISO-8601 UTC
    note: str              # 자유 메모
```

**rank_score = freq × entropy_weight** — 단순하고 해석 가능.

### 2. 랭킹 전략

`rank()` 메서드는 `rank_score` 내림차순, 동점 시 `created_at` 오름차순으로 정렬.
`top_k` 파라미터로 상위 K개만 반환 가능.

### 3. 압축 전략: Greedy Cosine-Similarity Dedup

```
1. 전체 패턴을 rank_score 내림차순 정렬.
2. kept 리스트를 초기화.
3. 각 candidate에 대해:
   a. embedding이 없으면 무조건 kept에 추가.
   b. kept 내 기존 항목과 cosine_similarity > threshold 이면 중복 → 제거.
   c. 중복 아니면 kept에 추가.
4. kept로 라이브러리 교체.
```

**임계값 기본값: 0.92** — 실험적으로 선택.
- 0.95+: 실질적으로 동일한 패턴도 유지 → 과도한 중복.
- 0.90-: 약간 다른 패턴도 제거 → 과도한 압축.
- 0.92: 적절한 균형.

#### 대안 비교

| 전략 | 장점 | 단점 | 결정 |
|------|------|------|------|
| 코사인 유사도 중복 제거 | 의미적 유사성 기반, O(n²) 허용 가능 | 임베딩 필요 | **채택** |
| 클러스터링(k-means) | 더 정교한 그룹화 | 하이퍼파라미터 복잡, LLM-0 위반 가능성 | 기각 |
| 정확 레이블 중복 제거 | 단순 O(n) | 의미적 중복 미처리 | 기각 |

### 4. 영속화: JSONL append-only + 압축 시 전체 재작성

- **추가(add)**: JSONL 파일에 라인 append.
- **압축(compress)**: 중복 제거 후 파일 전체 재작성(rewrite).
- **이유**: append-only는 단순하지만 압축 후엔 일관성이 중요하여 rewrite 방식 사용.

V632 ConstitutionWeightTracker의 append-only 원칙과 일부 차이가 있으나,
compress()의 목적(크기 축소)이 rewrite를 정당화함.

### 5. 메모리 모드 (`:memory:`)

테스트 및 임시 연산용. 파일 I/O 없음.

---

## Consequences

### Positive
- 패턴 중복 제거로 추론 비용 절감.
- `rank()` API로 고품질 패턴 우선 접근.
- JSONL 영속화로 세션 간 패턴 유지.
- LLM-0 준수 — 순수 로컬 수학 연산.

### Negative
- `compress()` 실행 시 파일 전체 재작성 → 대용량 파일에서 I/O 비용.
- 코사인 유사도는 embedding 없는 패턴에는 적용 불가 (항상 유지됨).

### Neutral
- `similarity_threshold` 튜닝 필요.
