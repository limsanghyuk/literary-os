## 맥락

V574 PE(Principal Engineer) 보고서에서 다음 4개의 CRITICAL/HIGH 위반이 발견되었다:

1. **DEV_MODE 기본값 `"true"`** — `LITERARY_OS_DEV_MODE` 환경변수 미설정 시  
   모든 배포 환경에서 인증 bypass 활성화 (보안 결함)
2. **`print()` 32건** — `literary_system/` 내 직접 표준출력 사용,  
   로그 집계/레벨 제어 불가
3. **bare `except:` 4건** — 예외 타입 비명시, 의도치 않은 오류 무음 처리
4. **데드코드** — `main_v316.py` 등 미참조 파일 잔존

이를 수동 수정으로 해결하더라도, 동일 패턴이 재발할 수 있다.  
Preflight Step15를 신설하여 CI 레벨에서 위생 규칙을 자동 강제한다.

---

## 결정

`tools/preflight_step15.py`를 신설하여 다음 3개 규칙을 자동 검사한다:

| 규칙 | 수준 | 내용 |
|------|------|------|
| Rule-1 | CRITICAL | `os.environ.get("LITERARY_OS_DEV_MODE", "true")` 금지 |
| Rule-2 | HIGH | `literary_system/` 내 `print()` 사용 금지 |
| Rule-3 | MEDIUM | bare `except:` 금지 |

CI 파이프라인에 `preflight-step15` 잡을 추가:
- preflight-step14 성공 후 실행
- Step15 실패 시 test 잡 전체 블로킹

---

## 설계 결정 사항

### Rule-1: DEV_MODE 정밀 패턴 매칭

주석·독스트링 내 `"true"` 문자열과 구분하기 위해,  
`os.environ.get(...)` 호출 형태만 검사한다:

```python
re.search(r'os\.environ\.get\(["\']LITERARY_OS_DEV_MODE["\'],\s*["\']true["\']', line)
```

### Rule-2: print() — 주석 라인 제외

`#` 으로 시작하는 라인은 검사 대상에서 제외한다.

### Rule-3: AST 기반 bare except 검사

`ast.ExceptHandler.type is None` 조건으로  
정확하게 bare `except:` 만 탐지한다 (false positive 없음).

## 대안 검토

| 대안 | 거부 이유 |
|------|-----------|
| ruff lint | 별도 도구 설치 필요 + 기존 빌드 체계에 없음 |

| pre-commit hook | 로컬에서만 동작, CI 블로킹 보장 안 됨 |

- `print()` / bare `except:` 신규 도입 즉시 PR에서 탐지
