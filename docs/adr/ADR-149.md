# ADR-149: pre-commit 4종 Hook 도입 (D-M-08)

**Date:** 2026-05-28  
**Author:** limsanghyuk

**Related:** ADR-147 (CI 4단), ADR-148 (type_stubs), ADR-150 (G82 Gate)

## Context

SP-D.1 매니페스트 D-M-08 항목은 커밋 전 정적 분석을 자동화하기 위해
pre-commit 프레임워크에 4종 hook을 의무화한다.

기존 literary-os 저장소에는 pre-commit 설정이 없어:
- 개발자별 린팅 방식이 상이함
- CI에서만 정적 오류를 발견 -> 피드백 루프 느림
- mypy strict 위반이 로컬에서 무감지 상태로 누적됨

---

## Decision

`.pre-commit-config.yaml`을 신설하여 다음 4종 hook을 구성한다.

| Hook | 버전 | 역할 |
|------|------|------|
| ruff + ruff-format | v0.6.9 | 린팅 + 자동 포맷 |
| black | 24.8.0 | 코드 포맷 (line-length 120) |
| mypy | v1.11.2 | 정적 타입 검사 (strict) |
| bandit | 1.7.10 | 보안 취약점 검사 (-ll) |

부가 hook (pre-commit-hooks v4.6.0):
- trailing-whitespace, end-of-file-fixer
- check-yaml, check-toml, check-merge-conflict

mypy 대상: `literary_system/(core|enterprise|gates)/`  
bandit 대상: `literary_system/`  

---

## Rationale

1. **로컬 패스트패스**: 커밋 전 4종 검사 -> CI 실패 전에 오류 조기 발견
2. **mypy strict 강제**: ADR-147에서 설정된 `[tool.mypy] strict=true`를 로컬에서도 강제
3. **bandit -ll**: LOW/MEDIUM 이상 보안 이슈만 차단 (과도한 알림 억제)
4. **black + ruff 공존**: ruff-format은 가속 포맷터, black은 표준 레퍼런스로 사용

---

## Consequences

- 개발자는 `pre-commit install` 최초 1회 실행 필요
- CI 4단(ADR-147)과 이중 안전망 구성
- mypy ignore_missing_imports=true로 외부 패키지 스텁 누락 허용
- G82 Static Type Safety Gate(ADR-150)가 이 파일의 존재를 검증함

---

## Implementation

`.pre-commit-config.yaml` 4종 hook 신설.
ST-1(G82): .pre-commit-config.yaml 존재 + mypy/bandit/ruff/black 포함 확인.

---

## Verification

Gate G82 (ADR-150) ST-1 체크포인트:
- `.pre-commit-config.yaml` 존재 checked
- 4종 hook 키워드(mypy/bandit/ruff/black) 포함 checked
