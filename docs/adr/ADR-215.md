# ADR-215: 실 LLM 축별 Critic 5종 (LLM-1)

## 상태
ACCEPTED (V754, Phase E.2)

## 맥락
V753에서 CriticInterface 5축·G_LLM1_BOUNDARY 도입. 이제 각 축의 실 LLM critic이 필요하다.

## 결정
`critic/llm_critics.py`에 LLMCritic(base) + 5축 구상 critic(Structure/Character/Dialogue/Emotion/Genre). 주입형 llm(Callable[[str],str]), 블라인드 좌우 무작위 쌍대(절대점수 금지), RAG 필수(base). evaluate_all_axes로 5축 verdict→pairwise BT 합의.

## 근거
- llm 주입형 → gpt-5/gpt-4o-mini/fake 교체 가능, 테스트는 fake(실 API 불요), CI 안전.
- 블라인드+역변환으로 위치 편향 제거. 합의는 pairwise BT(validation 재사용).

## 결과
- 긍정: 24 TC PASS, 실 LLM 스모크 확인. critic_ensemble(V757)의 부품.
- 유의: 비용·일치율은 G_LLM1_COST/ALIGNMENT(V758~9)에서 관리.
