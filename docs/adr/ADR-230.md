# ADR-230: ParetoRouter + TrainingMode 디스패처 (GPU 라우팅 E2)

## 상태
ACCEPTED (V770, Phase E.4 확장)

## 맥락
V768 규칙 라우팅을 설계도 §4.1 E2대로 **비용 vs 예상품질 파레토 최적**으로 승급. 또한 개발자가 클라우드/로컬/하이브리드 모드를 직접 지정해도 단일 진입점에서 작동하도록 디스패처 필요.

## 결정
`learning/pareto_router.py`:
- ParetoCandidate(provider/method/cost/expected_quality) + pareto_frontier(비지배 집합).
- ParetoRouter.select: 하드 제약(R1 force/R2 민감/R4 스케줄/R6 폴백) 먼저 → 그 외(R3/R5)에서 파레토 최적. 선호 cost|balanced|quality 가중(λ 0.8/0.5/0.2). 품질=용량기반 휴리스틱(QLoRA 0.85·풀FT 1.0, **측정 아님** 명시).
- TrainingMode(CLOUD/LOCAL/HYBRID/AUTO) + dispatch_training: 모드 지정시 cloud→RunPod·local→4070·hybrid→SplitPipeline·auto→파레토. 단일 진입점.

## 결과
15 TC PASS. cost→로컬·quality→Lambda(풀FT)·민감→하드제약 LOCAL 우선. 개발자 모드 4종 모두 작동. 품질은 결정 보조 추정치이며 최종 진실은 인간 GT/보상모델. LLM-0 유지. GPU 라우팅 E2 완료.
## 대안
순수 파레토(하드제약 무시): 프라이버시 사고 위험(기각). 안전 제약 선행 후 파레토.
