# ADR-063: PPOTrainer + ConstraintGuard — SP-B.2 PPO 안정화 레이어 (V603)

## 상태
Accepted

## 날짜
2026-05-22

## 컨텍스트

SP-B.2 RLHF 루프의 세 번째 단계로, RewardModel(V601)과 RLHFDatasetBuilder(V602)가 생성한
(씬, 보상) 쌍 JSONL을 소비하여 정책 개선을 시뮬레이션한다.

### 요구사항
- **KL 안정성**: 정책 업데이트 당 KL 발산 ≤ 0.08 (Cycle 1), ≤ 0.05 (최종)
- **보상 개선**: reward_improvement ≥ 0.0 (후퇴 없음)
- **LLM-0 준수**: 외부 LLM API 직접 호출 금지
- **안전성 제어**: KL 폭발, 보상 폭발, 엔트로피 붕괴에 대한 하드 리밋

### 문제
실 LLM 가중치 없이 PPO 알고리즘을 표현하면서도 KL 발산 추적과 클립 목표
(clipped surrogate objective)를 정확히 구현해야 한다.

## 결정

### 1. PPOTrainer (ppo_trainer.py)

**알고리즘**: Proximal Policy Optimization (Clipped Surrogate Objective)

```
L_CLIP(θ) = min(r_t(θ) · A_t, clip(r_t(θ), 1-ε, 1+ε) · A_t)
```

- **r_t(θ)** = π_θ(a|s) / π_θ_old(a|s) : 정책 비율
- **A_t** = GAE(λ=0.95, γ=0.99) : 일반화 어드밴티지 추정
- **ε** = 0.20 : 클립 계수 (CLIP_EPSILON)

**KL 근사** (Gaussian 근사):
```
KL ≈ 0.5 × (Δμ)² / σ²
```

**조기 종료**: KL > kl_threshold × 2.0 시 즉시 중단

**LCG RNG**: 외부 의존성 없는 선형 합동 생성기로 재현 가능성 보장

### 2. ConstraintGuard (constraint_guard.py)

PPO 학습 루프의 안전성 가드:

| 제약 | 임계값 | 동작 |
|------|--------|------|
| KL 하드 리밋 | 0.15 | 3연속 초과 시 학습 중단 |
| 보상 클램프 | [-10, +10] | 초과값 클램프 |
| 델타 클램프 | ±0.50 | 업데이트 크기 제한 |
| 엔트로피 하한 | 0.001 | 붕괴 감지 |

### 3. Gate G55 (PPO Stability Gate)

6개 체크포인트:
- CP-1: PPOConfig 기본값 유효성
- CP-2: PPOTrainer.train() 반환 타입
- CP-3: PPOResult.passed 의미론 (kl_stable AND improvement≥0)
- CP-4: ConstraintGuard KL 하드 리밋 3연속 → should_stop
- CP-5: clamp_reward() 범위 정확성
- CP-6: PPOResult.summary() 7키 존재

## 결과

### 긍정적
- KL 발산을 단계별로 추적하여 SP-B.2 안정성 기준 충족 가능
- ConstraintGuard가 정책 폭발을 하드웨어 레벨에서 방어
- LLM-0 준수: 내부 보상 신호만 사용

### 부정적/트레이드오프
- 실 LLM 가중치가 없으므로 정책 상태는 스칼라 근사 (μ, log_std, v)
- 실제 PPO 수렴 보장이 아닌 알고리즘 구조 시뮬레이션

## 파라미터 기본값

| 파라미터 | 값 | 근거 |
|----------|-----|------|
| kl_threshold | 0.08 | TRL PPO 권장 Cycle 1 상한 |
| clip_epsilon | 0.20 | OpenAI PPO 원논문 권장값 |
| value_coef | 0.50 | 가치 함수 손실 가중치 |
| entropy_coef | 0.01 | 탐색 유지 최소값 |
| learning_rate | 1e-4 | AdamW 기본 |
| gamma | 0.99 | 할인율 |
| lam | 0.95 | GAE λ |
