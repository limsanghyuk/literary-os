# ADR-081: DataAugmentationController — 훈련 데이터 증강 컨트롤러 (V639)

## 상태
Accepted (2026-05-26)

## 컨텍스트
SP-C.1 Self-Learning Loop의 9번째 컴포넌트.
ContaminationDetector(V638)가 오염 데이터를 탐지·제거한다면,
DataAugmentationController(V639)는 정제된 데이터를 증강하여 훈련 데이터 다양성을 확보한다.

## 결정

### 증강 전략 5종

| 전략 코드 | 설명 | 적용 단위 |
|---|---|---|
| SYNONYM_SWAP | 동의어 교체 | 토큰 |
| BACK_TRANSLATE | 역번역 시뮬레이션 (어순 변형) | 문장 |
| RANDOM_DELETION | 무작위 토큰 삭제 | 토큰 |
| SENTENCE_SHUFFLE | 문장 순서 섞기 | 문장 |
| TOKEN_INSERT | 필러 토큰 삽입 | 토큰 |

### 핵심 클래스
- `AugmentedSample`: 증강 단일 샘플 (strategy, augment_ratio 포함)
- `AugmentationBatch`: 증강 배치 결과 (UUID4, JSONL 직렬화)
- `DataAugmentationController`: 증강 엔진 (augment / augment_single / history API)

### 파라미터
- `DEFAULT_AUGMENT_RATIO = 0.15`: 기본 증강 강도
- `DEFAULT_AUGMENT_COUNT = 3`: 원본 1개당 생성 수
- `MAX_AUGMENT_COUNT = 10`: 최대 증강 배수

### 영속화
- LOSDB JSONL append-only (`:memory:` 모드 지원)
- `data/losdb/data_augmentation_controller.jsonl`

### 재현성
- `random.Random(seed)` 인스턴스 사용 — 전역 상태 오염 없음

## 결과
- LLM-0 원칙 완전 준수 (외부 LLM 호출 없음)
- 순수 Python 표준 라이브러리 (random, re, json, uuid)
- TC-01~33 단위 테스트 33/33 PASS
- SP-C.1 V640 FeedbackIntegrator 기반 마련
