# ADR-212: G_PAIRWISE_REGRESSION + G_TRANSITIVITY 게이트 제정

- **Status**: Accepted
- **Date**: 2026-06-12
- **Version**: V749 / v13.3.0
- **Phase**: Phase E SP-E.0

## Context

ADR-211(V748)에서 쌍대 비교 패러다임을 도입하고 G_NO_ABSOLUTE_REWARD 게이트를 추가했다.
V749에서는 실제 판정 품질을 자동 검증하는 두 게이트를 추가한다.

## Decision

### G_PAIRWISE_REGRESSION

명작(canonical) vs 강열화(degraded) 11쌍 픽스처(`pairwise_fixtures.py`)에서
canonical이 ≥ 9/11 (81.8%) 이상 승리해야 PASS.

- 픽스처: `literary_system/validation/pairwise_fixtures.py` (`REGRESSION_PAIRS`)
- 임계값: `REGRESSION_MIN_WIN_RATE = 9/11`
- judgments 미제공 시: structural 검사(쌍 수 == 11) 만 수행 → PASS
- 실 운영: `run_pairwise_gates.py --db <path>` 로 라이브 실행

### G_TRANSITIVITY

판정 그래프에서 3-cycle 비율 < 5% 요구.

- 알고리즘: `transitivity_check()` (pairwise.py)
  - 모든 3-노드 조합 순회 → cycle 카운트 / 총 조합 수
- 임계값: 0.05 (5%)
- judgments 없으면 trivially PASS

### run_pairwise_gates.py

독립 실행 CLI:
```
python tools/run_pairwise_gates.py [--json] [--db PATH] [--cost-cap X]
```
- 두 게이트 결합 실행
- exit 0 = PASS, exit 1 = FAIL

### run_release_gate.py 통합

`_check_pairwise_regression()` + `_check_transitivity()` → final dict에 추가.
structural check 모드: live DB 없어도 CI에서 항상 PASS 가능.

## Consequences

- `literary_system/validation/pairwise_fixtures.py` 신규 (REGRESSION_PAIRS 11쌍)
- `tools/run_pairwise_gates.py` 신규 (독립 CLI)
- `tools/run_release_gate.py` — 두 함수 + final dict 키 추가
- `tests/gates/test_v749_pairwise_gates.py` — 12 TC (DoD 6/6 × 2)
- V750: WP-4b 통합 보고서 + 패키징
