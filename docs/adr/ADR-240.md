# ADR-240: gates/ 순환 의존 오탐 정리 (모듈레벨 순환 검출)

## 상태
ACCEPTED (V780)

## 맥락
Preflight 경고에 순환 3건(auto_promotion_gate 자기참조, release↔phase_exit, self_learning_gate 자기참조). 조사 결과 **전부 함수 본문의 지연(lazy) import** — 순환 회피용 표준 패턴이자 자가검증(모듈 import 확인)으로 의도된 것. `_build_dep_graph`가 ast.walk로 함수 내부 import까지 모듈 의존으로 계산해 **오탐**.

## 결정
`tools/run_preflight.py`에 `_build_module_level_deps()` 추가: 모듈 로드 시점 실행 import만 수집(FunctionDef/AsyncFunctionDef 본문 제외). `_find_cycles`만 이 그래프 사용. 영향분석·연결성용 `_build_dep_graph`(전체)는 불변 → 회귀 없음.

## 결과
순환 의존 0건(경고 8→5). G_CONNECTIVITY PASS(0고립/86) 불변, PREFLIGHT PASS 유지. 5 TC. 잔여 경고 5건: Gate 독립운영 4 + Step12 Release Gate 타임아웃(단독 실행 필요)으로 구조적·블록 아님. 13.32→13.33.
## 대안
지연 import를 모듈레벨로 이동: 실제 순환 ImportError 유발(기각). 자기 import 제거: 자가검증 의도 훼손(기각). 검출기 정밀화가 정석.
