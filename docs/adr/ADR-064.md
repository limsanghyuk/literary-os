# ADR-064: RLHFMonitor — SP-B.2 보상 추세 모니터링 + 자동 롤백 (V604)

## 상태
Accepted

## 맥락
SP-B.2 PPO 훈련(V603) 이후, 훈련이 안정적으로 수렴하는지 실시간으로 추적할 장치가 필요하다.
보상이 지속적으로 감소하는 경우 자동으로 롤백을 트리거하여 훈련 발산을 방지한다.

## 결정

### 슬라이딩 윈도우 이동 평균 (window_size=10)
- 전체 이력 대신 최근 N 스텝의 이동 평균으로 추세 판단
- cold-start 보호: min_samples(기본 5) 미달 시 trend=unknown 반환

### 추세 판정 알고리즘
윈도우를 반으로 나눠 전반 평균(avg_first) vs 후반 평균(avg_second) 비교:
- delta = avg_second - avg_first
- delta >= +0.01 → improving
- delta <= -0.01 → degrading
- 그 외 → stable

### 자동 롤백 트리거 조건 (OR)
1. 연속 감소 횟수 >= degradation_steps (기본 3)
2. moving_avg < rollback_threshold (기본 0.60) AND trend == degrading

### 파라미터 기본값 근거
| 파라미터 | 기본값 | 근거 |
|----------|--------|------|
| window_size | 10 | PPO 미니배치 epoch 단위에 대응 |
| rollback_threshold | 0.60 | G55 기준 R≥0.75 대비 안전 여유 |
| degradation_steps | 3 | 노이즈 필터링 (단발 감소 허용) |
| min_samples | 5 | cold-start 오탐 방지 |
| reward_floor | 0.0 | 음수 보상 클램프 (RewardModel 정규화 범위) |

### LLM-0 원칙 준수
외부 LLM API 직접 호출 없음. 순수 통계 연산으로 구현.
