## 배경

V586에서 LOSDBClient Facade(ADR-045)가 완성되었으나, 이는 백엔드 타입 기반의 저수준 API다.  
Literary OS 도메인(씬/캐릭터/집계)에 적합한 고수준 쿼리 레이어가 부재하여, 각 컴포넌트가 직접 LOSDBClient를 호출하는 파편화가 발생한다.

## 결정

`literary_system/db/query_interface.py` — **QueryInterface** 클래스를 신설한다.

### 핵심 API

```python
class QueryInterface:
    def find_scenes(
        self, *, character, similar_to, graph_within_hops, episode_range, limit
    ) -> List[SceneResult]: ...

    def find_characters(
        self, similar_personality, *, limit
    ) -> List[CharacterResult]: ...

    def cross_backend_aggregate(
        self, *, group_by, metric, backends
    ) -> List[AggregateResult]: ...

    def health(self) -> Dict: ...
```

### 설계 원칙

1. **LOSDBClient 위임**: QueryInterface는 직접 DB를 건드리지 않는다. 모든 I/O는 `_client`를 통한다.
2. **안전 폴백**: `_client=None`이면 빈 리스트 반환 (예외 없음).
3. **SLO 1.0초** (ADR-049 C1): 모든 쿼리 메서드에 elapsed 측정 + SLO 초과 시 경고 로그.
4. **LLM-0 원칙**: `requests`, `httpx`, `openai`, `anthropic` import 금지.
5. **중복 제거**: `find_scenes()`는 `scene_id` 기준 dedup 적용.
