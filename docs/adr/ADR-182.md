# ADR-182: PluginAuthAdapter — plugins/security 단방향 통합 (V721)

**날짜**: 2026-05-29  
**상태**: Accepted  
**구현**: V721 (v12.3.5)

## 맥락

V717~V720에서 구현된 Zero-Trust Security 패키지(`literary_system/security/`)는 완전 고립 상태(ADR-128 경고)였다. 아무 패키지도 security/를 import하지 않아 Preflight Step 13에서 `⚠️ WARN: security — 완전 고립` 감지. 2버전 연속 고립 시 ADR-128 위반으로 에스컬레이션.

## 결정

1. **PluginAuthAdapter** 신설(`literary_system/plugins/plugin_auth.py`)
   - `plugins/ → security/` 단방향 의존 (역방향 금지, 순환 차단)
   - `ZeroTrustTokenService.verify()` + `TenantAuthority.authorize()`를 플러그인 실행 전 인증·인가에 통합
   - `PluginPermission` → Zero-Trust 역할(`PERMISSION_ROLE_MAP`) 매핑
   - `strict=True`: 권한 거부 시 `PluginAccessDenied` 즉시 발생
   - `strict=False`: 부분 허용 결과(`PluginAuthResult`) 반환

2. **핵심 API**
   - `authenticate(token_str, tenant_id, required_permissions) → PluginAuthResult`
   - `verify_token_only(token_str) → TokenClaims`
   - `has_permission(claims, permission, tenant_id) → bool`

## 결과

- `security/` 패키지 고립 해소 (ADR-128 WARN 제거)
- 플러그인 실행 전 Zero-Trust 인증 강제화
- 33 TC PASS, G32 0 violations
