# ADR-059: 파인튜닝 평가 기준선 + 학습 전 안전성 + 장문 맥락 전략

**상태**: Accepted  
**날짜**: 2026-05-21  
**버전**: V599 (v10.4.0)  
**관련 ADR**: ADR-056 (LoRA Dataset Pipeline), ADR-057 (Training Config), ADR-058 (Inference Gateway)  
**구현 파일**:
- `literary_system/finetune/pre_train_safety.py`
- `literary_system/finetune/finetune_eval_pipeline.py`
- `literary_system/finetune/long_context_strategy.py`

---

## 1. 맥락

V596~V598에서 LoRA 데이터 파이프라인, 학습 설정, 추론 게이트웨이가 완성됐다.  
V599에서는 **학습 전 데이터 안전성 검사**, **평가 지표 기준선**, **장문 처리 전략**을 확립하여
SP-B.1 종료 조건(V600 Gate G54)에 진입할 수 있는 품질 토대를 마련한다.

Phase B 본안 보강 핵심 결정사항:
- **B-M-07**: BERTScore ≥ 0.85 / LLM-judge ≥ 4.0 / Style ≥ 0.80 / BLEU ≥ 0.30
- **B-M-08**: Krippendorff α ≥ 0.70 (분기별 + 인간 5명 월 1회 100 샘플 calibration)
- **B-M-09**: PII / Toxic / Copyright / Quality 4축 안전성 — 전축 PASS 필수
- **B-M-11**: 청크 100K + 오버랩 16K + NKG RAG 주입

---

## 2. 결정

### 2.1 학습 전 안전성 4축 (B-M-09)

`PreTrainSafety` 클래스 — 4가지 축 순차 검사:

| 축 | 방식 | 실패 조건 |
|----|------|-----------|
| PII | 정규식 (주민등록번호·전화·이메일·계좌·여권) | 1건이라도 탐지 |
| Toxic | 혐오·욕설·위험 키워드 패턴 | 1건이라도 매칭 |
| Copyright | verbatim 50자 이상 구절 참조 대조 | 비율 > 30% |
| Quality | 최소 50자 / 반복비율≤0.40 / 공백비율≤0.60 | 임계 미달 |

**SafetyResult.safe = True** 인 샘플만 학습 데이터로 허용.

### 2.2 파인튜닝 평가 5축 (B-M-07)

`FineTuneEvalPipeline` — LLM-0 준수 경량 근사 구현:

| 축 | 지표 | 임계값 | 구현 |
|----|------|--------|------|
| 1 | BERTScore F1 | ≥ 0.85 | n-gram 겹침 근사 |
| 2 | LLM-judge | ≥ 4.0 | 다양성+키워드 스텁 (V605+ LOSConstitution 연동) |
| 3 | Style | ≥ 0.80 | 문장 길이 분포 + TTR 유사도 |
| 4 | BLEU-4 | ≥ 0.30 | smoothing BLEU (Lin & Och 근사) |
| 5 | Equiv rate | ≥ 0.95 | EquivalenceTester 5축 pass_rate |

전축 PASS → `ArtifactStage.VALIDATED` 진급 허용.

### 2.3 Krippendorff α 계산 (B-M-08)

`compute_krippendorff_alpha(ratings)` — 간격 척도 기준:
- α ≥ 0.70 → acceptable agreement
- 분기별 1회 자동 계산 + 인간 5명 월 1회 100 샘플 calibration

### 2.4 장문 맥락 청킹 전략 (B-M-11)

`LongContextStrategy`:

```
청크 크기: 100K 토큰 (≈ 400K 문자, 1 토큰 ≈ 4 문자)
오버랩:    16K 토큰  (≈  64K 문자)
경계 정렬: \n\n → \n → 문장 부호 순서로 자연 경계 탐색
NKG RAG:  [NKG_CONTEXT]...[/NKG_CONTEXT] 블록으로 선행 주입
```

Llama-3.1-8B 128K 컨텍스트 윈도우 기준 최대 1.28개 청크/회.

## 3. 기각 대안

| 대안 | 기각 이유 |
|------|-----------|
| 실제 bert-score 라이브러리 | transformers 의존성 → LLM-0 원칙 위배; 근사치 충분 |
| PII 머신러닝 탐지기 | 외부 모델 의존성; 정규식으로 KR PII 90% 커버 가능 |
| 고정 청크 경계 | 단어/문장 중간 절단으로 맥락 손상; 자연 경계 정렬 필수 |
| 오버랩 없는 청킹 | 경계 근처 맥락 손실; 16K 오버랩이 연속성 보장 |

---

## 4. 결과

- `PreTrainSafety.check()` → 4축 즉시 검사, SafetyResult 반환
- `FineTuneEvalPipeline.evaluate()` → 5축 평가, EvalResult 반환
- `LongContextStrategy.chunk()` → ChunkingResult (청크 리스트)
- `compute_krippendorff_alpha()` → α 계산 유틸리티
