# ADR-122: ModelServingEndpoint v2.0 — Kubernetes HPA 지원

**상태**: Accepted  
**날짜**: 2026-05-27  
**버전**: V662 / v11.35.0  
**관련**: ADR-065 (ModelServingEndpoint v1.0)

---

## 컨텍스트

Phase C SP-C.3에서 프로덕션 API 안정성 확보를 위해 서빙 엔드포인트가
Kubernetes HPA(Horizontal Pod Autoscaler)와 연동되어야 한다.

v1.0은 단순 `/model_card` 엔드포인트만 제공하며 스케일링 전략이 없었다.

---

## 결정

`literary_system/serving/model_serving_endpoint_v2.py`에 **ModelServingEndpointV2** 구현.

### 핵심 컴포넌트

| 클래스 | 역할 |
|--------|------|
| `HPAConfig` | K8s HPA 파라미터 관리 (min/max replicas, CPU/memory 목표, RPS) |
| `HPAStatus` | 현재 레플리카 수 및 스케일 방향 스냅샷 |
| `MetricsSnapshot` | 단일 시점 QPS/CPU/Memory/Queue 메트릭 |
| `MetricsCollector` | 슬라이딩 윈도우 집계 (avg QPS/CPU, max queue) |
| `ModelServingEndpointV2` | 스케일 계산 + K8s 프로브 응답 + HPA 상태 리포트 |
| `generate_hpa_manifest()` | K8s HPA YAML spec 생성 (autoscaling/v2) |

### 스케일 계산 전략

```
desired = max(cpu_based_desired, qps_based_desired)
cpu_based = round(current * (avg_cpu / target_cpu))
qps_based = ceil(avg_qps / target_rps_per_replica)
result = clamp(desired, min_replicas, max_replicas)
```

### K8s 프로브

- `liveness_probe()` → `{"status": "alive"|"unhealthy", "uptime_sec": ...}`
- `readiness_probe()` → `{"status": "ready"|"not_ready", "replicas": ...}`

## 트레이드오프

| 항목 | 결정 | 이유 |
|------|------|------|
| 실제 K8s vs Python 레이어 | Python 레이어 | CI에서 K8s 없이 단위 테스트 가능 |
| autoscaling/v1 vs v2 | v2 사용 | Memory + 커스텀 메트릭 지원 |
| 쿨다운 구현 | force_scale() 으로 쿨다운 우회 제공 | 긴급/테스트 시나리오 지원 |
