**상태**: Accepted

**관련 ADR**: ADR-054 (LOSConstitution), ADR-060 (G54, SP-B.1 Exit)  
**구현 파일**:

---

## 1. 맥락

SP-B.1(V596~V600)에서 LoRA Fine-tuning Pipeline을 완성했다(Gate G54 PASS, v10.5.0).
SP-B.2(V601~V610)에서는 RLHF 루프를 구현한다.
첫 단계(V601)는 RLHF의 핵심인 **보상 함수**를 정의한다.

보상 함수는 LOSConstitution v1.0의 5축 점수를 단일 스칼라 R(scene)으로 변환해야 한다.
이 스칼라가 PPOTrainer(V603)의 학습 신호가 된다.

---

## 2. 결정

### 2.1 Constitution 5축 → 스칼라 보상

**LOSConstitution 5축 가중치 기본값**:

| 축 | 의미 | 기본 가중치 |
|----|------|------------|
| drse | 드라마 서사 엔트로피 | 0.30 |
| debt | 서사 부채 없음 | 0.20 |
| arc | 캐릭터 아크 | 0.20 |
| tension | 긴장감 | 0.15 |
| prose | 문장 품질 | 0.15 |

**보상 계산식**: R(scene) = Σ(raw_score_i × weight_i) - adv_penalty

**LLM-0 원칙**: 외부 LLM API 직접 호출 없음. LOSConstitution 규칙 기반 점수 사용.

### 2.2 마커 가중치 상한 0.20 (보상 해킹 방지, B-V2-03)

각 축의 실효 가중치를 0.20 이하로 클램프한 후 정규화.
이로써 단일 축에 과도하게 최적화하는 보상 해킹(reward hacking)을 방지한다.

**근거**: PPO 학습 중 모델이 특정 마커(예: 긴장감 관련 어휘)를 과도하게
사용해 단일 축 점수를 올리는 마커 스터핑이 발생 가능하다.
가중치 상한으로 이 효과를 제한한다.

### 2.3 적대적 시드 인터페이스 (B-V2-03 준비)

5종 적대적 시드 탐지기를 내장:

| 시드 유형 | 탐지 방법 | 패널티 |
|----------|---------|--------|
| marker_stuffing | 서사 마커 5회+ 연속 출현 | 0.15 |
| length_inflation | 어휘 다양도 < 0.30 + 단어 100+ | 0.10 |
| repetition_pattern | 동일 구절 3회+ 반복 | 0.12 |
| extreme_emotion | 감탄사 3회+ 연속 | 0.08 |
| genre_deviation | 드라마 키워드 0건 (긴 텍스트) | 0.10 |

패널티 합계 상한: 0.50. 최종 R(scene)은 [0.0, 1.0] 클램프.

### 2.4 quality_correlation() hook (D23)

보상 스칼라와 외부 품질 측정치(인간 평가 등) 간의 Pearson 상관계수를
누적 버퍼 기반으로 실시간 추적. V606~V608 RLHF cycle에서 활용.

목표 상관: ≥ 0.70 (Krippendorff α와 연동, B-M-08).

### 2.5 폴백 설계 (LLM-0)

LOSConstitution import 실패 시 규칙 기반 5축 추정으로 자동 폴백.
외부 LLM API 호출 없이 순수 텍스트 분석으로 보상 계산.

## 3. 기각 대안

| 대안 | 기각 이유 |
|------|----------|
| 단일 BERTScore 기반 보상 | 서사 품질 5축 반영 불가, LOSConstitution 자산 미활용 |
| LLM-judge 직접 호출 | LLM-0 원칙 위반, 비용/지연 문제 |
| 가중치 상한 없음 | 마커 스터핑 공격에 취약 |
