# ADR-246: 자체평가 → loop-C 통합 배선 (M1+M2+M3)

## 상태
ACCEPTED (V785) — ADR-243~245(M1/M2/M3) 통합. ADR-243 §2 배선 구현.

## 맥락
M1(자격·V782)·M2(보상닻·V783)·M3(가드·V784)가 개별 구현됨. 이를 하나의 자체평가 파이프라인으로 묶어 loop-C 입력(선호쌍+보상조정)을 생성해야 폐회로가 인간 없이 돈다.

## 결정
`critic/self_eval_pipeline.py` SelfEvalPipeline.run(items, judge, generate):
1. **M1 자격검정** → 미통과 시 전체 차단(자격 없는 심판 학습 금지).
2. **M2 NextEpisodeBench**(자격 critic만) → 생성 vs 실제 쌍대 → 선호쌍.
3. **M3 분포 가드** → 생성물 병리 감점(음성만).
4. to_preference_pairs → loop-C PreferencePair. 병리 생성이 *진*(rejected) 쌍은 유지(부정 신호), *이긴*(모순) 쌍만 제외.
SelfEvalReport.ready_for_loopc → LoopCClosure/G_LOOPC_WINRATE 입력.

## 결과
12 TC PASS. 자격 통과→loop-C 준비(선호쌍 생성)/자격 미달→차단. 인간 없이 명작 닻으로 학습 신호 생성하는 전 사슬 코드로 닫힘. 천장=모작(독창성=인간 최종시험). LLM-0.
## 대안
M1 생략하고 M2 바로: 순환(자격없는 LLM 심판) 위험(기각).
