# ADR-032: Gate Registry Single Source of Truth

| 항목 | 내용 |
|------|------|
| **상태** | Accepted |
| **버전** | V578 |
| **날짜** | 2026-05-19 |
| **결정자** | 아키텍처 위원회 (CSA · CSC · CSPE) |
| **관련 ADR** | ADR-028 (Gate Hierarchy L1-L4), ADR-035 (LLM Adapter Canonical) |

---

## 배경

V577 기준 34개 릴리즈 게이트가 `release_gate.py`의 `GATES` 튜플 목록에만 정의되어 있다. 이 구조는 다음 문제를 야기한다.

1. **메타데이터 부재**: gate_id/name/fn 3개 필드만 보유 — ADR 참조, 추가 버전, 계층 정보 없음
2. **검색 불가**: 특정 ADR에 연관된 게이트를 프로그래밍 방식으로 조회할 수 없음
3. **CI 통합 불가**: 레지스트리 구조 검증을 CI에서 자동화할 수 없음
4. **확장 저해**: 신규 게이트 추가 시 메타데이터 기준 없음

## 결정

### 1. GateRegistryEntry 데이터클래스 신설

`literary_system/gates/gate_registry.py`에 `GateRegistryEntry` 불변 데이터클래스를 정의한다.

```python
@dataclass(frozen=True)
class GateRegistryEntry:
    gate_id: str        # 고유 식별자
    name: str           # 표시 이름
    fn: Callable        # 실행 함수
    adr_ref: str = ""   # 관련 ADR (예: "ADR-035")
    version_added: str = ""  # 추가 버전 (예: "V577")
    layer: str = "L2"   # 계층 (L0/L1/L2/L3/L4)
```

### 2. GATE_REGISTRY dict 단일 소스

모듈 로드 시 `release_gate.GATES` 튜플에서 자동 구축하여 하위 호환을 유지한다.

```python
GATE_REGISTRY: Dict[str, GateRegistryEntry] = _build_registry()
```

### 3. 공개 API

```python
from literary_system.gates.gate_registry import (
    GATE_REGISTRY,   # 전체 레지스트리 dict
    get_gate,        # gate_id로 항목 조회
    list_gates,      # layer 필터 조회
    run_all_gates,   # 전체 실행
    validate_registry,  # CI 검증
)
```

### 4. CLI 진입점

```bash
# CI 레지스트리 검증
python -m literary_system.gates.gate_registry --validate

# 전체 목록 출력 (layer / gate_id / adr_ref / version_added)
python -m literary_system.gates.gate_registry --list
```

### 5. Gate G36 GateRegistry 신설

레지스트리 자체의 무결성을 검증하는 G36 게이트를 추가한다.

```python
def _gate_registry_g36() -> dict:
    from literary_system.gates.gate_registry import validate_registry
    return validate_registry()
```

## 계층 정의 (ADR-028 연동)

| 계층 | 의미 | 예시 게이트 |
|------|------|-----------|
| L0 | 핵심 불변 원칙 | llm_zero, auth_regression_g34 |
| L1 | 아키텍처 계약 | logging_discipline_g32 |
| L2 | 모듈 생존 | adapter_canonical_g35 |

| L4 | 품질 기준 | story_quality_gate28 |

## 결과

### 긍정적 결과
- 게이트 메타데이터 단일 소스 확립
- CI에서 `--validate` 플래그로 레지스트리 무결성 자동 검증
- ADR 참조 기반 게이트 추적 가능
- V579+ 중복 제거, mypy 작업 시 gate layer별 우선순위 지정 가능

### 부정적 결과
- `release_gate.py`와 `gate_registry.py` 두 파일의 동기화 필요
  → `_build_registry()`가 `GATES`를 자동 import하여 완화
