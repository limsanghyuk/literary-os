# ADR-128: 패키지 연결성 의무 (G_CONNECTIVITY)

**날짜**: 2026-05-27  
**상태**: 승인됨  
**제안자**: 3인 전문가 합의 (CA / CCE / PSE)  
**적용 버전**: v11.39.0+

---

## 배경

literary_system/ 하위 패키지들은 V665 기준 77개였으나, 11개(14%)가 
literary_system 내 어떤 패키지와도 import 연결이 없는 완전 고립 상태임이 확인되었다.
이는 설계는 됐으나 통합되지 않은 '죽어가는 로직'으로, Preflight 12단계의
Survival Matrix가 '심볼 존재'만 체크하고 '연결 여부'를 검증하지 않았기 때문이다.

---

## 결정

### 규칙 1: 패키지 생성 시 연결 계획 명시 (PR 의무)

새 literary_system/ 패키지를 추가하는 PR은 PR 설명에 다음을 포함해야 한다:

```
## 연결 계획 (ADR-128 필수)
- import_from: literary_system.X (이 패키지가 import할 대상)
- import_by: literary_system.Y (이 패키지를 import할 상위 패키지)
- 연결 예정 버전: VN+1
```

### 규칙 2: 고립 탐지 자동화 (Preflight Step 13)

`tools/run_preflight.py` Step 13은 literary_system/ 전체 import 그래프를
분석하여 완전 고립 패키지(in-degree=0, out-degree=0)를 탐지한다.

- **1버전 고립**: WARN 출력, 로그 기록 (블록 아님)  
- **2버전 연속 고립**: FAIL — 해당 Preflight FAIL, 배포 불가

### 규칙 3: G_CONNECTIVITY Release Gate

`tools/run_release_gate.py`는 G_PREFLIGHT 검사 이후 G_CONNECTIVITY를 실행한다.
2버전 연속 고립 패키지가 존재하면 Release Gate FAIL.

### 규칙 4: 고립 패키지 처분 의무

고립이 탐지된 패키지는 다음 중 하나를 선택해야 한다:
1. **통합**: 기존 모듈에서 import 추가 (1버전 내)
2. **제거**: 패키지 삭제 및 관련 테스트 정리
3. **예외 신청**: PSE 승인 + 이유 명시 (docs/adr/ADR-XXX.md)
