# ADR-101: MetaLearnerCycle + KrippendorffAlpha 평가자 간 신뢰도 측정 (V641, SP-C.1)

**상태**: Accepted
**날짜**: 2026-05-26
**버전**: V641 / v11.12.0

## 배경

blueprint v2.0 §2.2 목표 A1: Constitution v2.0 MetaLearner 4사이클, R(scene)≥0.78, Krippendorff α≥0.70.
F7 재배치에 따라 V641부터 MetaLearner 1사이클 + α 1차 측정을 수행한다.

## 결정

### MetaLearnerCycle (`literary_system/constitution/meta_learner_cycle.py`)

기존 MetaLearner (nie/) outer-loop 래퍼. 4 V버전(V641~V644)에서 각 1사이클씩 호출된다.

**역할:**
1. MetaLearner outer-loop 갱신 (advantage = L_final - L_baseline)
2. Krippendorff α 측정 (평가자 간 신뢰도)
3. DataAugmentationController augment_ratio 동적 조정 (α 기반)
4. FeedbackIntegrator avg_correction_delta → MetaLearner 이득 반영

**비침투적 통합:** 기존 MetaLearner, DataAugmentationController, FeedbackIntegrator 인터페이스를 수정하지 않음.

### KrippendorffAlpha (`literary_system/constitution/krippendorff_alpha.py`)

**알고리즘:** Krippendorff (2011) 표준 coincidence matrix 방식.

```
α = 1 - D_o / D_e

coincidence c_vw = Σ_u 2/(m_u-1)  [각 단위 u, 평가자 쌍(i,j), 값(v_i, v_j)]
D_o = Σ_{v,w} c_vw * d(v,w) / (2n)
D_e = Σ_{v≠w} n_v * n_w * d(v,w) / (n*(n-1))

n = coincidence 행렬 상삼각 합계
n_v = marginal count (행 합계)
```

**지원 척도:**
- `interval`: d(v,w) = (v-w)²
- `nominal`: d(v,w) = 0 if v==w else 1
- `ordinal`: d(v,w) = CDF 기반 g-누적 합

**임계값:** α ≥ 0.70 허용 (Krippendorff 2004), α ≥ 0.80 우수

**augment_ratio 조정 규칙:**
- α < 0.65: ratio += 0.02 (증강 강화 — 품질 개선 필요)
- α ≥ 0.80: ratio -= 0.02 (증강 완화 — 과적합 방지)
- 0.65 ≤ α < 0.80: 유지

## 결과

- V641 검증 기준: α ≥ 0.70 + R(scene) 추세 양수
- 33 TC (33/33 PASS, test_v641_meta_learner_cycle.py)
- LLM-0 준수: 외부 API 없음, 순수 Python 표준 라이브러리
- Gate G37 DuplicateZero 준수: 기존 SafetyViolation과 충돌 없음

## 대안

- Fleiss' Kappa: 명목 척도에 최적화, 연속값 지원 미흡 → 기각
- Cohen's κ: 2인 평가자 한정 → 3인 이상 시나리오 불가 → 기각
- ICC: 분산분석 기반, 음수 처리 복잡 → 기각
