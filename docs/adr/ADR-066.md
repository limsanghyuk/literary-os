# ADR-066: CanonicalBridgeV2 + Gate G56/G57 설계 결정 (V606, SP-B.2 완료)

## 상태
**Accepted** — 2026-05-22

## 맥락
V605(CanaryController) 완료 후 SP-B.2의 마지막 과제:
1. 외부 LLM + 로컬 LoRA 동시 지원 브리지 (ADR-064에서 명세)
2. G56: RLHF 보상 품질 게이트 (mean_reward ≥ 0.75, delta ≥ 0.05)
3. G57: Constitution 5축 균형성 게이트 (mean_correlation ≥ 0.80)

## 결정

### CanonicalBridgeV2 — LLM-0 원칙 준수 어댑터 브리지

| 항목 | 결정 | 이유 |
|------|------|------|
| 실 API 호출 | 없음 (LLM-0) | 테스트 격리 + 비용 통제 |
| 어댑터 주입 | `register_external_adapter()` / `register_local_adapter()` | 의존성 역전 |
| 폴백 메커니즘 | `fallback_enabled=True` 기본값 | 가용성 보장 |
| 재귀 방지 | 폴백 재호출 시 `fallback_enabled=False` 임시 비활성화 | 무한 루프 방지 |
| 상태 추적 | `call_count`, `fallback_count` | 관측성 |

### Gate G56 — RLHF 보상 게이트

| 조건 | 임계값 | 근거 |
|------|--------|------|
| mean_reward | ≥ 0.75 | Constitution 5축 종합 보상 기준 (ADR-061) |
| delta | ≥ 0.05 | 베이스라인 대비 최소 향상 5% |
| 샘플 | > 0 | 빈 보상 목록 즉시 실패 |

### Gate G57 — Constitution 5축 상관 게이트

| 조건 | 임계값 | 근거 |
|------|--------|------|
| mean_correlation | ≥ 0.80 | 5축 간 균형 보상 학습 검증 |
| 쌍 수 | C(5,2) = 10쌍 | 전체 쌍 피어슨 상관 평균 |
| 구현 | 순수 파이썬 (`math.sqrt`) | numpy 의존성 회피, 이식성 |

Constitution 5축: `safety`, `coherence`, `creativity`, `quality`, `consistency`

높은 상관 = 보상 모델이 단일 축에 편향되지 않고 5축을 균형 있게 학습 중.

## 거절된 대안

- **G56에 중앙값 사용**: 이상치 영향 감소 장점 있으나, RLHF 커뮤니티 표준은 평균 보상 → 기각
- **G57에 Spearman 상관**: 순위 기반으로 단조 비선형도 잡을 수 있으나 복잡도 ↑ → 기각
- **CanonicalBridgeV2에 asyncio**: 성능 이점 있으나 동기 PPO 루프와 혼용 어려움 → V7+ 검토

## 결과

SP-B.2 완료 조건 충족:
- G55 (PPO Stability) ✅ V603
- G56 (RLHF Reward) ✅ V606
- G57 (Constitution Axis) ✅ V606
- CanonicalBridgeV2 ✅ V606
- Total gates: 54 → 56
