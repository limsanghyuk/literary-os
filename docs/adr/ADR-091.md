# ADR-091: V624 LongRunScenario + MemoryRegressionChecker

- **날짜**: 2026-05-25
- **상태**: Accepted
- **작성자**: Literary OS 개발팀

## 컨텍스트

V623에서 60/60 Gates PASS 달성 후, 본안 v3.0 V624 항목에 따라 24h 장기 시나리오와 메모리 회귀 검증 인프라가 필요하다.

## 결정

### literary_system/testing/ 신규 패키지 (ADR-091)
- `LongRunScenario`: 24 epoch 압축 실행, epoch별 메모리·성능 스냅샷
- `MemoryRegressionChecker`: N회 연속 실행, 선형 회귀 기울기로 메모리 누수 감지

### 설계 원칙
1. **LLM-0 준수**: 외부 LLM 호출 없음. tracemalloc 기반 메모리 측정.
2. **압축 시뮬레이션**: 24h → 24 epoch (실 시간 <<1초/epoch).
3. **임계값**: 메모리 증가 허용 50MB/epoch, 회귀 기울기 허용 5 MB/run.
4. **경고 임계값**: 20MB/epoch, 2 MB/run (FAIL 아닌 WARN).

### 임계값 선택 근거
- Phase B 컴포넌트(CIMv2, SharedCharDB, RewardModelV2)가 epoch당 최대 ~15MB 일시 할당 후 GC.
- 50MB 허용은 GC 지연 + 캐시 효과를 고려한 안전 마진.
- 5 MB/run 기울기는 10 run × 5 = 50MB, 지속 누수를 의미하는 수준.

## 결과

- `test_long_run_scenario.py`: 30 TC 추가 (V624)
- 6,900 → 6,930 Tests
- 60/60 Gates 유지 (Gate 신규 추가 없음, 기존 G61 Tests≥6700 PASS 유지)
