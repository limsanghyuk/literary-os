# ADR-201: Phase E Manifest 검증 Gate — PhaseEManifestValidator (V740)

| 항목 | 내용 |
|------|------|
| 상태 | Accepted |
| 결정자 | CSC · CSPE |
| 날짜 | 2026-05-29 |
| 관련 | ADR-200, V740, deploy/phase_e/ |

## 맥락

V738~V739에서 Phase E 인프라 스텁(Helm Chart, KEDA, ArgoCD) YAML을 생성했다.
이 파일들은 Phase E 진입 시 실제 사용되므로, Phase D 종료 전(G95 통과 전)에
정확한 위치와 최소 구조가 보장되어야 한다.

수작업 확인은 파일 누락이나 키 오기입을 놓칠 위험이 있으므로
자동화된 Python 검증 모듈이 필요하다.

## 결정

`literary_system/deploy/phase_e_manifest.py`에 `PhaseEManifestValidator` 클래스를 구현한다.

### 검증 체크리스트 (ME-1 ~ ME-8)

| 체크 ID | 대상 | 검증 내용 |
|---------|------|-----------|
| ME-1 | Helm | 4개 필수 파일 존재 |
| ME-2 | Helm | Chart.yaml version == 13.0.0 |
| ME-3 | Helm | values.yaml fl.enabled + fl.minClients |
| ME-4 | KEDA | 2개 필수 파일 존재 |
| ME-5 | KEDA | ScaledObject minReplicaCount + maxReplicaCount |
| ME-6 | ArgoCD | 2개 필수 파일 존재 |
| ME-7 | ArgoCD | Application automated + selfHeal |
| ME-8 | 구조 | deploy/phase_e/{helm,keda,argocd} 디렉토리 존재 |

### 사용 예시

```python
from literary_system.deploy import PhaseEManifestValidator

validator = PhaseEManifestValidator(repo_root="/path/to/literary-os")
result = validator.run_all_checks()
assert result["all_passed"], result["summary"]
```

## 결과

- CI에서 `pytest tests/unit/test_v740_phase_e_manifest.py` 실행으로 manifest 무결성 보장
- `repo_root` 주입으로 다양한 환경(CI, 로컬, 컨테이너)에서 재사용 가능
- 8개 체크 전부 PASS여야 G95 Phase D Exit Gate 진입 가능

## 기각된 대안

| 대안 | 이유 |
|------|------|
| yamllint 단독 | 파일 존재+값 확인 불가 |
| shell script | Python 생태계 통합 어려움, 이식성 저하 |
| helm lint | 실 Helm 설치 필요, CI 의존성 증가 |
