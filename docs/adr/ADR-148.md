# ADR-148: literary_system/core/type_stubs.py — 공통 타입 패키지

**Status**: Accepted  
**Version**: V686 (v12.0.6-dev)  
**Date**: 2026-05-28  
**Deciders**: SP-D.1 팀  

---

## Context

`literary_system` 전체에서 `dict[str, Any]`, `Union[...]` 등의 타입이
파일마다 중복 정의되어 일관성이 없었다. mypy --strict 도입 시
공통 타입 허브가 없으면 오류가 산발적으로 발생한다.

---

## Decision

### `literary_system/core/type_stubs.py` 신설

| 항목 | 내용 |
|------|------|
| TypeAlias | `JSON`, `TenantId`, `GateId`, `Score` |
| TypeVar | `T`, `T_co`, `ReportT` |
| Protocol | `LiteraryCoreProtocol`, `GateProtocol`, `SerializableProtocol`, `AnalyzerProtocol` |
| 유틸 | `clamp_score()`, `is_valid_gate_id()`, `is_valid_tenant_id()` |

모든 Protocol은 `@runtime_checkable` 데코레이터 적용.

### `literary_system/core/__init__.py`

`type_stubs`의 공개 심볼을 re-export하여 `from literary_system.core import Score` 형태로 사용 가능.

---

## Consequences

- literary_system 전체에서 타입 통일 가능
- mypy --strict 점진적 도입의 기반 제공
- Protocol 기반 duck-typing 검증 가능 (`isinstance(obj, GateProtocol)`)
