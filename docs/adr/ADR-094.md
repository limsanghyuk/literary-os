# ADR-094: ServePlane Helm Chart 신설 및 정적 검증 전략

**상태**: Accepted  
**날짜**: 2026-05-25  
**버전**: V627 (v10.32.0)  
**관련**: ADR-057 §5 (TrainPlane/ServePlane 격리), ADR-058 (LLM-1 원칙), ADR-093 (TrainPlane Helm 검증)

---

## 배경

V626(ADR-093)에서 TrainPlane Helm chart 정적 검증기(HelmValidator)를 도입했다.
이와 동일한 정적 안전망을 ServePlane에도 적용하여 배포 사고를 사전에 차단한다.

ServePlane은 완료된 LoRA 파인튜닝 모델을 CPU 기반으로 서빙하는 레이어다.
ADR-057 §5에 따라 TrainPlane(literary-train 네임스페이스)과 ServePlane(literary-serve 네임스페이스)은
클러스터 내에서 완전히 분리되어야 한다.

---

## 결정

### 1. ServePlane Helm Chart 신설

`deploy/helm/serve_plane/` 경로에 Helm chart를 신설한다.

| 파일 | 역할 |
|------|------|
| `Chart.yaml` | 차트 메타데이터 (apiVersion=v2, name=literary-os-serve-plane) |
| `values.yaml` | 기본값 (namespace=literary-serve, requirePromoted=true, CPU 전용) |
| `templates/deployment.yaml` | CPU 추론 Deployment |
| `templates/service.yaml` | ClusterIP Service (port 80→8080) |
| `templates/hpa.yaml` | HorizontalPodAutoscaler (CPU/Memory 기반) |

### 2. ServePlaneValidator 구현

`literary_system/serving/serve_plane_validator.py`에 11체크포인트 정적 검증기를 구현한다.

| 체크포인트 | 검증 내용 |
|-----------|----------|
| `chart_dir_exists` | deploy/helm/serve_plane/ 디렉터리 존재 |
| `chart_yaml_valid` | apiVersion=v2, 필수 필드(name/version/appVersion/description/type) |
| `values_yaml_valid` | 필수 최상위 키(image/serving/model/namespace 등) + 하위 구조 |
| `namespace_isolation` | literary-serve 전용, literary-train/default/kube-system 금지 (ADR-057 §5) |
| `llm1_require_promoted` | model.requirePromoted=true 필수 (ADR-058 LLM-1 원칙) |
| `cpu_resources_valid` | nvidia.com/gpu 요청 금지, cpu/memory requests+limits 필수 |
| `hpa_config_valid` | minReplicas≥1, min≤max, CPU/Memory utilization 1~100% |
| `health_check_valid` | livenessPath + readinessPath 형식 검증 |
| `serving_params_valid` | port>0, workers>0, logLevel∈{DEBUG/INFO/WARNING/ERROR/CRITICAL} |
| `template_files_exist` | deployment.yaml + service.yaml + hpa.yaml 존재 확인 |
| `service_account_valid` | SA name 'default' 사용 시 경고 |

### 3. TrainPlane vs ServePlane 비교

| 항목 | TrainPlane (ADR-093) | ServePlane (ADR-094) |
|------|---------------------|---------------------|
| 네임스페이스 | literary-train | literary-serve |
| GPU 정책 | nvidia.com/gpu ≥ 1 (필수) | nvidia.com/gpu 금지 (CPU 전용) |
| 모델 정책 | LoRA 학습 작업 | PROMOTED 모델만 서빙 (LLM-1) |
| HPA | 해당 없음 | CPU/Memory 기반 오토스케일링 |
| 필수 템플릿 | lora-job.yaml + cronjob.yaml | deployment.yaml + service.yaml + hpa.yaml |
| 체크포인트 수 | 9 | 11 |

## 결과

### 긍정적 결과
- ServePlane 배포 전 정적 검증으로 GPU 리소스 혼입, 네임스페이스 오설정, LLM-1 위반을 사전 차단
- 11체크포인트 50 TC로 검증 커버리지 확보
- TrainPlane/ServePlane 완전 이중 검증 체계 완성

### 부정적 결과
- Helm chart가 실제 클러스터에서 실행되는 E2E 검증은 정적 검증으로 대체 불가 (런타임 검증은 별도 필요)

---

## 준수 사항

- **LLM-0**: ServePlaneValidator는 외부 LLM 호출 없음 (파일 파싱만)
- **LLM-1**: model.requirePromoted=true 미설정 시 배포 차단
- **DEV_MODE**: 항상 False 기본 (ADR-034)
- **G37**: ServePlaneValidator, ServePlaneValidationResult, ServePlaneChartSpec, ServePlaneValuesSpec — literary_system/ 내 유일 클래스명 확인 완료
