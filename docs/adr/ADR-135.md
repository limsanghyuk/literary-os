# ADR-135: EnterpriseSLOGate — Enterprise SLO 계약·모니터링·경보 (G73)

**날짜**: 2026-05-27  
**상태**: 수락됨  
**분류**: SP-C.4 (Competitive Absorption)

## 컨텍스트

SP-C.4 경쟁 흡수 단계에서 NovelAI·Sudowrite·Novelcrafter·NolanAI·Jenova 5개 경쟁사를 분석한 결과,
엔터프라이즈 B2B 파트너십을 위한 SLO(Service Level Objective) 계약·모니터링 체계가 부재함을 확인했다.
경쟁사 대비 신뢰성 우위를 확보하기 위해 정형화된 SLA 계약 구조와 실시간 위반 경보 시스템이 필요하다.

## 결정

`literary_system/enterprise/slo.py`에 EnterpriseSLOGate (G73)를 도입한다.

### 핵심 설계

1. **EnterpriseSLOTier** (4단계): BASIC(99.0%) / STANDARD(99.5%) / PREMIUM(99.9%) / ENTERPRISE(99.99%)
2. **EnterpriseSLOContract**: 파트너별 가용성·지연(P99)·처리량 계약 명세
3. **SLOMetricSnapshot**: 실제 측정값 스냅샷
4. **ViolationSeverity** (3단계): WARNING / CRITICAL / BREACH
5. **SLOViolationAlert**: 위반 경보 (severity + delta_pp)
6. **SLOMonitor.check()**: 계약 vs 스냅샷 비교 → 위반 목록 반환
7. **EnterpriseSLOGate.demo_run()**: PREMIUM + ENTERPRISE 2건 모두 PASS

### 네이밍 결정

`SLOTier` → `EnterpriseSLOTier`로 명명: `literary_system/tenant/production_monitor.py`의 기존 `SLOTier`(BETA/GA)와
클래스명 충돌을 방지하기 위해 명시적 접두사를 사용한다 (DuplicateZero G37 준수).

## 결과

- G73 게이트 추가 (총 74개)
