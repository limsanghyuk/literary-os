# ADR-062: RLHF 데이터셋 빌더 설계 (RLHFDatasetBuilder v1.0)

**상태**: Accepted  
**날짜**: 2026-05-22  
**버전**: V602 (v10.7.0)  
**관련 ADR**: ADR-061 (RewardModel), ADR-054 (LOSConstitution)  
**구현 파일**:
- `literary_system/rlhf/rlhf_dataset_builder.py`

---

## 1. 맥락

RewardModel(V601)이 단일 씬을 평가하는 함수를 제공한다.
V602에서는 대규모 씬 컬렉션을 평가하여 PPOTrainer(V603)가 소비할
**(씬, 보상) 쌍 JSONL** 데이터셋을 생성해야 한다.

요구사항:
1. 품질 필터링 — R(scene) ≥ REWARD_THRESHOLD 씬만 포함
2. 8B / 3B 듀얼 dataset — 동일 씬, 다른 model_target 레이블
3. 결정론적 train/val/test 분할 (기본 80/10/10)
4. LLM-0 원칙 — 외부 LLM API 호출 없음

---

## 2. 결정

### 2.1 JSONL 포맷

DatasetEntry 필드:

| 필드 | 타입 | 설명 |
|------|------|------|
| entry_id | str (UUID4) | 유일 식별자 |
| scene | str | 씬 원문 |
| reward | float [0,1] | R(scene) 스칼라 |
| passed | bool | reward ≥ threshold |
| axis_rewards | dict | 5축 raw_score |
| model_target | str | "8B" / "3B" / "both" |
| split | str | "train" / "val" / "test" |

JSONL 선택 이유: 스트리밍 처리, 행 단위 추가, HuggingFace datasets 호환.

### 2.2 품질 필터링 (filter_passed=True 기본값)

`filter_passed=True`(기본값)이면 `passed=True` 씬만 데이터셋에 포함.
이는 낮은 품질 씬이 PPO 학습 신호를 오염시키는 것을 방지한다.
`filter_passed=False`는 분석·디버깅용으로 제공.

### 2.3 8B/3B 듀얼 Dataset (`build_dual()`)

동일한 씬/보상 데이터를 `model_target="8B"`, `"3B"` 두 파일로 분리 저장.
각 파일은 `rlhf_dataset_8B.jsonl`, `rlhf_dataset_3B.jsonl`로 명명.

근거: EXAONE A/B 인터페이스(V600, ADR-060) 에서 8B/3B 모델을 병렬 평가하며,
각 모델에 맞는 별도 학습 신호가 필요하다.

### 2.4 결정론적 Split (인덱스 기반)

`floor(total * ratio)` + 인덱스 기반으로 분할하여 재실행 시에도 동일한 분할 보장.
외부 random seed 불필요 — 데이터셋 재현성 확보.

기본 비율 80/10/10:
- train: PPO 학습
- val: reward 모니터링 (V604)
- test: 최종 G56/G57 Gate 검증 (V610)

## 3. 기각 대안

| 대안 | 기각 이유 |
|------|----------|
| Parquet 포맷 | pandas 의존성 추가, LLM-0 환경에서 불필요한 무게 |
| 랜덤 Shuffle 후 Split | 비결정론적 — 재현성 위반 |
| 단일 파일 모든 target | model_target 레이블 혼재 → PPO 훈련 시 혼동 가능 |
