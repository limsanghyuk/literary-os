# ADR-078: SelfLearningMonitor — SP-C.1 파이프라인 상태 모니터

## 상태
ACCEPTED

## 맥락
V631~V635에서 MetaLearner·WeightTracker·PatternLibraryV2·RetrainingScheduler·AutoPromotionGate
5개 컴포넌트가 구현되었다. 각 컴포넌트가 독립적으로 동작하므로 전체 파이프라인의
현재 상태를 한 곳에서 파악할 방법이 없었다. 특히 롤백 급증, F1 극단 하락,
패턴 소진, Gate 연속 실패 같은 이상 징후를 조기에 감지할 수단이 필요하다.

## 결정
**SelfLearningMonitor를 도입한다.**

- `capture()`: 5개 컴포넌트 상태 + 이상 파라미터를 받아 MonitorSnapshot 생성
- `detect_anomalies()`: 4종 이상 코드 감지 (ROLLBACK_SURGE / F1_EXTREME_DROP / PATTERN_EMPTY / GATE_FAIL_STREAK)
- LOSDB JSONL append-only 영속화
- `unhealthy_snapshots()`: 비정상 스냅샷 필터링

## 이상 감지 규칙

| 코드 | 조건 | 임계값 |
|------|------|--------|
| ROLLBACK_SURGE | rollback_count ≥ threshold | 3 |
| F1_EXTREME_DROP | recent_drift ≤ -threshold | 0.10 |
| PATTERN_EMPTY | pattern_count = 0 | — |
| GATE_FAIL_STREAK | gate_fail_streak ≥ threshold | 3 |

## 결과

### 긍정적
- SP-C.1 파이프라인 전체 가시성 확보
- 이상 징후 조기 감지 → 수동 개입 시점 명확화
- JSONL 이력으로 장기 추세 분석 가능

### 부정적/제약
- 실제 컴포넌트 연동(자동 상태 수집)은 V640+ 에서 구현
- V636에서는 외부에서 상태를 주입하는 방식 (테스트 가능성 우선)
