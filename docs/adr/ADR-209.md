# ADR-209: G_INTEGRITY_MANIFEST — SHA256 자기검증 + 테스트 인벤토리 상시화 (WP-0)

**날짜**: 2026-06-12  
**상태**: Accepted  
**버전**: V746 / v13.0.1  
**관련**: ADR-128 (G_CONNECTIVITY), ADR-054 (LOSConstitution), WP-0 (Phase E SP-E.0)

---

## 배경

V745(v13.0.0) 완료 후 검증 주간(2026-06-07~11) 조사에서 다음 두 가지 위험이 식별됐다:

1. **무결성 미검증**: SHA256SUMS.txt가 존재하나 릴리즈 게이트에서 재검증하지 않아, 빌드 산출물 변조를 감지할 수 없다. (P0/P1 미결)
2. **테스트 카운트 漂流(drift)**: test_inventory.json이 수동으로만 갱신되어 실제 TC 수와 기록값이 벌어질 수 있다.

WP-0 설계(2026-06-10 Sonnet 핸드오프 문서)에서 이를 단일 게이트(`G_INTEGRITY_MANIFEST`)로 묶어 릴리즈 게이트 최종 단계에 강제화하기로 결정했다.

---

## 결정

### 1. `tools/generate_sha256sums.py` 신규 생성

| 함수 | 역할 |
|---|---|
| `generate_sums(repo_root)` | git ls-files 기반 `{상대경로: sha256_hex}` dict 반환 |
| `write_sums(repo_root)` | SHA256SUMS.txt 재생성 |
| `verify_sums(repo_root) → VerifyResult` | 기존 SHA256SUMS.txt 대비 전 항목 검증 |
| `check_minisig(repo_root)` | SHA256SUMS.txt.minisig 존재·형식 확인 |

- subprocess 실행이 아닌 **import 방식**으로 run_release_gate.py에서 호출
- SHA256SUMS.txt 자신은 재귀 해시에서 자동 제외
- `.git/`, `__pycache__/`, `*.pyc`, `out/`, `dist/`, `build/` 제외

### 2. `tools/run_release_gate.py` — `G_INTEGRITY_MANIFEST` 단계 추가

기존 흐름 `G_PREFLIGHT → G_CONNECTIVITY → run_release_gate()` 에 다음을 삽입:

```
G_PREFLIGHT → G_CONNECTIVITY → G_INTEGRITY_MANIFEST → run_release_gate()
```

`_check_integrity_manifest()` 3단계:

| 단계 | 동작 | 실패 시 |
|---|---|---|
| ① SHA256 재생성·자기검증 | write_sums() 후 verify_sums(), 불일치 0 확인 | FAIL (릴리즈 차단) |
| ② test_inventory 재생성·카운트 일치 | generate_test_inventory.py 재실행, 카운트 ≥ 이전값 or ±5% 이내 | FAIL |
| ③ minisig 존재 확인 | SHA256SUMS.txt.minisig 없으면 WARN | **차단 아님** (WARN만) |

`--verify-only` 플래그 추가: G_INTEGRITY_MANIFEST만 단독 실행 (빠른 검증용)

### 3. `tests/gates/test_v746_integrity_manifest.py` — 33 TC

WP-0 DoD 5종 포함:

| TC 그룹 | TC 수 |
|---|---|
| §1 compute_file_hash | 6 |
| §2 generate_sums / write_sums | 7 (DoD-1 포함) |
| §3 verify_sums | 8 (DoD-2, DoD-3 포함) |
| §4 check_minisig | 3 (DoD-5 포함) |
| §5 _check_integrity_manifest | 5 (DoD-4 포함) |
| §6 CLI 통합 | 4 |
| **합계** | **33** |

## 대안

- **minisig 서명 차단**: minisign 인프라 미구축 상태에서 차단은 개발 흐름을 막는다. WARN으로 완화하고 Phase E.5에서 minisign 키 발급 후 차단으로 강화(ADR-235 예약).
- **subprocess 실행**: import 방식 대비 프로세스 오버헤드가 크고 오류 추적이 어렵다. 채택 안 함.
- **릴리즈 게이트 진입 차단**: SHA256SUMS 불일치는 보안 사고 수준이므로 FAIL(차단)이 적절하다.

---

## 결과

- 릴리즈 게이트가 SHA256 무결성과 TC 카운트를 **매 릴리즈마다 자동 검증**한다.
- 빌드 산출물 변조 시 릴리즈 차단된다.
- minisig 미서명은 경고로 기록되어 이후 강화 타이밍을 추적 가능하다.
- V746 이후 모든 버전: `python tools/run_release_gate.py --verify-only` 로 빠른 무결성 확인 가능.
