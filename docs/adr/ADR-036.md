# ADR-036: AsyncDiscipline — 비동기 코드 위생 기준

**날짜**: 2026-05-19  
**상태**: Accepted  
**버전**: V580

## 맥락

Python 3.10+에서 `asyncio.get_event_loop()`는 실행 중인 이벤트 루프가 없을 때 새 루프를 자동 생성하는 동작이 deprecated 됐다.  
literary_system/ 코드베이스에서 이 패턴이 2건 발견됨 (character_intent_agent.py).

## 결정

1. `asyncio.get_event_loop()` 사용 금지 — `asyncio.get_running_loop()` 또는 `asyncio.run()` 사용
2. `async def` 함수 내부에서만 `await` 사용 (정적 검사)
3. 동기 컨텍스트에서 coroutine 직접 호출 금지
4. Gate G38 으로 CI 강제

## 결과

- V580: character_intent_agent.py `get_event_loop()` 2건 → `get_running_loop()` 교체
- G38 AsyncDiscipline 게이트: literary_system/ 내 `get_event_loop()` 0건 강제
