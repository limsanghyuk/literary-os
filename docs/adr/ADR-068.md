# ADR-068: MultiWorkOrchestratorV2 설계 결정 (V608, SP-B.3)

## 상태
**Accepted** — 2026-05-23

## 컨텍스트

V607에서 SharedCharacterDBV2 + SharedWorldDBV2 (ADR-067) 가 완성됨에 따라,
이를 활용하는 상위 오케스트레이터 업그레이드가 필요하다.

기존 `MultiWorkOrchestrator` v1 (V570)은 v1 DB를 직접 인스턴스화하므로
스냅샷, RLHF 보상 추적, 충돌 감지 기능을 활용할 수 없다.

요구사항:
1. **비파괴적 업그레이드** — v1 API(register_author, create_project, open_session,
   close_session, process_scene, apply_genre_transfer, snapshot) 완전 유지
2. **프로젝트 단위 체크포인트** — 복수 캐릭터 + 월드 전체를 한 번에 스냅샷
3. **RLHF 보상 통합** — 씬 처리 시 보상 점수를 캐릭터별로 기록
4. **다중 프로젝트 충돌 탐지** — N개 프로젝트 간 캐릭터/위치 동시 수정 감지
5. **통합 일관성 지표** — 캐릭터 일관성 + 월드 일관성을 단일 점수로 합산
6. **완전 직렬화** — export_state_v2 / import_state_v2 로 세션 간 상태 영속

## 결정

### D-1: 상속 기반 v2 (비파괴적)

```python
class MultiWorkOrchestratorV2(MultiWorkOrchestrator):
    def __init__(self, max_concurrent=10, cim_decay=0.95):
        super().__init__(...)
        self.char_db = SharedCharacterDBV2()   # v1 교체
        self.world_db = SharedWorldDBV2()       # v1 교체
```

- `super().__init__()` 호출로 v1 컴포넌트(CIM, GTL, ISO, LicenseAPI) 완전 재사용
- Python의 속성 재할당으로 v1 참조를 v2로 교체 (덕 타이핑 활용)
- 별도 잠금 `_lock_v2: threading.RLock` 추가, v1 `_lock`과 독립

### D-2: 프로젝트 캐릭터 자동 추적

`process_scene_v2(event, reward_score)` 내부에서 `_project_characters` dict에
등장 캐릭터 ID를 자동 누적. `checkpoint_project` 는 이 집합을 순회하여
개별 `char_db.checkpoint(char_id)` 호출.

```
_project_characters: Dict[str, Set[str]]   # project_id → Set[char_id]
_project_locations:  Dict[str, Set[str]]   # project_id → Set[loc_id]
```

위치 추적은 자동화하지 않고 `track_location(project_id, loc_id)` 로 명시적 등록.
(위치는 씬 이벤트에 포함되지 않는 현 API 구조 때문)

### D-3: ProjectCheckpoint 데이터클래스

```python
@dataclass
class ProjectCheckpoint:
    checkpoint_id: str               # SHA-256(project_id:label:ts)[:12]
    project_id: str
    label: str
    timestamp: float
    char_snapshot_ids: Dict[str, str]  # character_id → snapshot_id
    world_snapshot_id: str
```

`restore_project(project_id, checkpoint_id)` 는 캐릭터만 복원.
월드 복원은 전체 DB에 영향을 주므로 `world_db.restore(world_snapshot_id)` 직접 호출을 권장.

### D-4: 충돌 탐지 흐름

```
register_project_states([pa, pb, ...])
    └─ char_db.register_project_state(pid, char_id)  for each tracked char
    └─ world_db.register_project_state(pid, loc_id)  for each tracked loc

detect_inter_project_conflicts([pa, pb, pc, ...])
    └─ for each pair (pa, pb):
        └─ shared_chars = _project_characters[pa] ∩ _project_characters[pb]
        └─ char_db.detect_conflicts(cid, pa, pb) → ConflictRecord?
        └─ shared_locs = _project_locations[pa] ∩ _project_locations[pb]
        └─ world_db.detect_location_conflicts(lid, pa, pb) → LocationConflict?
    └─ → InterProjectConflictReport
```

### D-5: 이중 일관성 점수

```
dual_consistency_score(project_a, project_b):
    shared_chars = _project_characters[a] ∩ _project_characters[b]
    char_score   = mean(char_db.consistency_score(c) for c in shared_chars)
                   ← 0.5 if empty
    world_score  = world_db.consistency_score()
    return round((char_score + world_score) / 2, 4)
```

단일 프로젝트 점수는 `project_char_consistency(project_id)` 로 제공.

### D-6: 직렬화

`export_state_v2()` 는 다음 7개 키를 포함한 dict 반환:

| 키 | 내용 |
|----|------|
| version | "2.0.0" |
| exported_at | float timestamp |
| char_db | SharedCharacterDBV2.export_snapshot() |
| world_db | SharedWorldDBV2.export_snapshot() |
| project_characters | {project_id: [char_id, ...]} |
| project_locations | {project_id: [loc_id, ...]} |
| checkpoints | {checkpoint_id: ProjectCheckpoint dict} |
| checkpoint_index | {project_id: [checkpoint_id, ...]} |
| total_scenes_processed | int |

`import_state_v2(data)` 는 기존 상태에 **병합** (덮어쓰기 아님).

## 결과

- **새 파일**: `literary_system/multiwork/multi_work_orchestrator_v2.py`

## 대안 고려

| 대안 | 거부 이유 |
|------|-----------|
| v1 직접 수정 | 27개 v607 테스트 재작성 필요 |
| 컴포지션 패턴 | v1 메서드 전량 위임 코드 필요, 상속이 더 단순 |
| 자동 위치 추적 | SceneProcessEvent에 location_id 필드 없음; API 변경은 G58 이후로 연기 |
