# ADR-099 — ConstitutionWeightTracker: LOSDB JSONL 영속화 + 비파괴 롤백

**상태**: Accepted  
**날짜**: 2026-05-26  
**버전**: V632 (v11.2.0)  
**관련**: ADR-054 (LOSConstitution v1.0), ADR-098 (LOSConstitutionV2 Bayesian Opt)

---

## 컨텍스트

V631(ADR-098)에서 LOSConstitutionV2가 Bayesian Optimization으로 w1~w5를
자동 탐색하게 되었다. 최적화 결과를 다음 세션에서도 재사용하고,
잘못된 최적화 결과가 발생했을 때 이전 상태로 복원할 수 있는
**가중치 버전 관리 레이어**가 필요하다.

---

## 결정

`literary_system/constitution/constitution_weight_tracker.py`에
`ConstitutionWeightTracker` 클래스를 신설한다.

### 저장소 포맷 — JSONL (line-delimited JSON)

```
losdb/constitution_weights.jsonl
```

- 한 줄 = `WeightRecord` JSON 직렬화
- Append-only → 쓰기 잠금 불필요, 파일 손상 시 마지막 유효 라인까지 복구 가능
- 버전 순서 = 삽입 순서 (시간순)

### WeightRecord 스키마

```json
{
  "version_id": "uuid4",
  "timestamp":  "ISO-8601 UTC",
  "weights":    {"drse":0.30,"debt":0.20,"arc":0.20,"tension":0.15,"prose":0.15},
  "entropy":    2.21,
  "note":       "optimised"
}
```

### 비파괴 롤백

`rollback(version_id)` 호출 시 대상 버전의 가중치를 **새 레코드**로 재저장한다.
기존 이력은 삭제되지 않는다. `load_latest()`는 항상 마지막 레코드를 반환하므로
롤백 후에는 복원된 가중치가 최신이 된다.

### 메모리 모드

`:memory:` 경로를 지정하면 파일 없이 인메모리 리스트로 동작한다.
단위 테스트에서 파일 I/O 없이 전체 동작을 검증할 수 있다.

---

## 대안 검토

| 대안 | 기각 이유 |
|------|----------|
| SQLite 직접 사용 | 의존성 추가 없이 동일 기능 가능. JSONL로 충분. |
| 단일 JSON 파일 덮어쓰기 | 동시 쓰기 시 손상 위험. Append-only가 더 안전. |
| 파괴적 롤백 (이력 삭제) | 감사 추적 불가. 비파괴 롤백 채택. |

---

## 결과

- `ConstitutionWeightTracker` — 4 메서드: `save`, `load_latest`, `rollback`, `history`
- `WeightRecord` — 불변 dataclass (frozen=True), 직렬화/역직렬화 지원
- LLM-0 준수: 외부 LLM 호출 없음
- constitution/__init__.py 에 `ConstitutionWeightTracker`, `WeightRecord` 추가 export
