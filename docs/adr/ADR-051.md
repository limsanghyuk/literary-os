# ADR-051: GPU Adapter Contract

**Status**: Accepted  
**Date**: 2026-05-21  
**Version**: V590 (v9.5.0)  
**Deciders**: Literary OS Core Team  
**Tags**: gpu, finetune, adapter, cost-slo, sp-a3

---

## 컨텍스트

Literary OS의 파인튜닝 파이프라인(SP4, V474~)은 내부 Mock 환경에서만 동작하고 있었다.  
Phase A SP-A.3에서는 실제 외부 GPU 클라우드(RunPod / Lambda Labs / HF AutoTrain)와의  
인터페이스를 표준화하고, 비용 폭주를 방지하는 SLO 메커니즘을 구축한다.

핵심 제약:
- **LLM-0**: gpu_adapter.py는 외부 LLM API(OpenAI, Anthropic)를 호출하지 않는다.  
  GPU 훈련 API와 LLM 추론 API는 별개다.
- **비용 예측 가능성**: 월 $150 초과 지출은 즉시 중단이 필요하다.
- **드라이런 우선**: 실제 GPU 기동 없이 비용 추정이 항상 가능해야 한다.

## 결정

### 1. GPUAdapterContract ABC 도입

추상 계약 인터페이스를 통해 GPU 프로바이더 구현체를 교체 가능하게 한다.

```python
class GPUAdapterContract(abc.ABC):
    @property @abc.abstractmethod
    def provider_id(self) -> GPUProvider: ...

    @property @abc.abstractmethod
    def cost_per_hour(self) -> float: ...

    @abc.abstractmethod
    def estimate_cost(self, hours: float) -> float: ...

    @abc.abstractmethod
    def launch_job(self, request: GPUJobRequest) -> GPUJobResult: ...

    @abc.abstractmethod
    def dry_run(self, request: GPUJobRequest) -> GPUJobResult: ...
```

### 2. 3종 Adapter 구현

| Adapter            | 인스턴스          | 단가       |
|--------------------|-------------------|------------|
| RunPodAdapter      | RTX 4090          | $0.39/h    |
| LambdaLabsAdapter  | H100 SXM5 80GB    | $1.49/h    |
| HFAutoTrainAdapter | AutoTrain A100    | $4.00/h    |

모든 Adapter는 `dry_run=True`(기본값) 시 실제 API 호출 없이 비용 추정만 반환한다.

### 3. CostSLO 경계값 (ADR-051 §4)

| 수준      | 임계값  | 동작                    |
|-----------|---------|-------------------------|
| WARN      | $90/월  | 경보 발령, 작업 허용    |
| BLOCK     | $120/월 | 신규 작업 차단          |
| HALT      | $150/월 | 전체 GPU 작업 즉시 중단 |

### 4. GPUCostLedger 확장 (cost_ledger.py)

기존 CostLedger(LLM 토큰 비용)와 별도로 GPUCostLedger를 추가하여  
GPU 작업별 비용을 월 단위로 추적한다.

```python
ledger = GPUCostLedger()
result = ledger.gpu_track(provider="runpod", hours=2.0, cost_per_hour=0.39)
# → {"cost_usd": 0.78, "monthly_total": 0.78, "slo_status": "OK"}
total = ledger.monthly_total_gpu()  # → float
```

## 결과

### 긍정적 결과
- GPU 프로바이더 교체 비용 최소화 (ABC 계약 교체만으로 가능)
- 드라이런 기본값으로 비용 사고 예방
- SLO 3단계 경보로 월 비용 $150 상한 보장
- 기존 CostLedger(LLM) 완전 보존

### 부정적 결과 / 트레이드오프
- 각 프로바이더 실 API 연동은 api_key 주입 필요 (현재 Mock)
- HF AutoTrain 단가($4/h)는 가장 비쌈 — 소규모 실험에만 권장

### 중립적 결과
- Gate G49(GPUAdapterGate, GA-1~GA-10)로 계약 준수 자동 검증
- 실제 GPU CI는 월 1회 cron (SP-A.4에서 EquivalenceTester와 연동)
