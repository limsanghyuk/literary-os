# ADR-241: 생성 본체 7-pass L4 승격 (generation/)

## 상태
ACCEPTED (V781, Phase E T3 트랙)

## 맥락
우선순위맵 진단: 평가·학습 절반(critic·loop-C·GPU)은 갖췄으나 **생성 절반(7-pass)이 docs/sessions 의사코드 골격 = L4 부재**. loop-C가 개선할 *실체* 부족. "무엇을 생성하나"가 최대 빈칸.

## 결정
`literary_system/generation/` 신규 패키지:
- `schema.py`: WorkSpec/Beat/SceneBrief 정식 타입 계약 + STANDARD_ARC 7기능.
- `pass_pipeline.py` PassPipeline: Pass1 거시설계→Pass2 인과비트→Pass3 씬브리프→Pass4 RAG→Pass5 초안→Pass6 구조게이트→Pass7 패널. 주입형 훅 retrieve(corpus RAG)/generate(**loop-C 학습 생성기**)/judge(critic 쌍대).
- Pass5 generate 훅 = 생성 절반↔학습 절반 연결점. Pass7 패널 = loop-C 선호쌍 입력 생성.

## 결과
14 TC PASS. 씨드→7pass→loop-C 선호쌍 E2E 동작(7비트·게이트·패널). 생성 본체가 골격→타입 계약 파이프라인으로 승격. LLM-0(골격 무호출, LLM은 generate/judge 훅 내). 다음=각 Pass 내부 심화(NKG 배선·실 RAG·실 생성기 연결).
## 대안
docs/sessions 골격 유지: loop-C 개선 대상 빈약(기각). 정식 패키지 승격으로 연결.
