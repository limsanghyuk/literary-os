# ADR-124: B2BPartnerGate G71 — LOI 3건 이상 B2B 계약 검증

**날짜**: 2026-05-27  
**상태**: Accepted  
**버전**: V664 / v11.37.0  
**게이트**: G71 (B2BPartnerGate)

---

## 컨텍스트

SP-C.3의 마지막 비즈니스 게이트로서, Literary OS PublicSDK와 B2B Partner API의 실제 시장 수용성을 
LOI(Letter of Intent) 체결 건수로 검증한다. Phase C 본안에서 G71 통과 조건은 **유효 LOI ≥ 3건**이다.

LOI는 정식 계약 이전의 사전 의향서로, SIGNED 또는 EXECUTED 상태이어야 유효하다.
각 LOI는 파트너사명, 서명 날짜, 연간 예상 계약 금액, 담당자 이메일, API scope를 포함한다.

---

## 결정

### Gate G71 통과 기준

| 항목 | 요건 |
|------|------|
| 유효 LOI 상태 | SIGNED 또는 EXECUTED |
| 최소 건수 | ≥ 3건 |
| loi_id | 중복 불가 |
| contact_email | RFC 5322 형식 |
| signed_date | YYYY-MM-DD 형식 |
| annual_value_krw | ≥ 0 |

### 구현 구성요소

- **LOIRecord**: LOI 단건 데이터 클래스 (loi_id, partner_name, status, signed_date, contact_email, annual_value_krw, api_scope)
- **LOIStatus**: DRAFT / SIGNED / EXECUTED / EXPIRED / CANCELLED (Enum)
- **LOIRepository**: 인메모리 저장소, 중복 loi_id 방어
- **B2BPartnerGate**: 검증 실행, `run()` → `B2BPartnerReport`
- **B2BPartnerReport**: passed, valid_loi_count, total_annual_value_krw, loi_results

### 모듈 위치

```
literary_system/gates/b2b_partner_gate.py
```

---

## 결과

- **G71 PASS 조건**: 유효 LOI ≥ 3건 AND 모든 LOI 필드 검증 통과
- **누적 Gate**: 60 → 67/67 (G71 신규 추가)
- **테스트**: +33 TC (8,350 → 8,350 목표)
- **LLM-0**: 외부 LLM 호출 없음
- **DEV_MODE**: False (ADR-034)
