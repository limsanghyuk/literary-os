# ADR-229: SplitPipeline — 하이브리드 분업 파이프라인 PoC (GPU 3-모드 ③)

## 상태
ACCEPTED (V769, Phase E.4 확장)

## 맥락
V767(로컬)·V768(라우터)로 3-모드 인프라 완성. 설계도 §4.1 진화방향 E1(로컬 선별→클라우드 강화)의 실효를 처음으로 정량 측정 필요.

## 결정
`learning/split_pipeline.py` SplitPipeline:
- Stage A(candidate_selection): 생성 후보 N개를 PairwiseRewardModel(쌍대 보상)로 채점→상위 top-k 선별. 소형 3B → ProviderRouter R5 LOCAL($0).
- Stage B(final_reinforce): 선별분으로 대형 모델 최종 강화. 13B 등 로컬 천장 초과 → R3 CLOUD.
- `plan()` → SplitReport: 하이브리드 비용 vs 올클라우드 베이스라인 → 절감 정량화.
- VRAM 추정 버그 수정: "3b" in "13b" 오매칭 → 정규식 `(\d+(?:\.\d+)?)\s*b\b` + 임계 매핑(3→6/7→11/8→11.5/13→18/34→24/그외 46GB).

## 결과
13 TC PASS. 측정: StageA 3B 로컬 + StageB 13B 클라우드 = 20% 절감(올클라우드 대비); 8B 이하면 완전 로컬 100% 절감. 민감 코퍼스는 StageB도 LOCAL 강제(클라우드 금지). 하이브리드 실효 첫 수치화. LLM-0 유지(보상 judge=critic 경계 내).
## 대안
전부 클라우드: StageA 반복 비용 낭비(기각). 전부 로컬: 13B+ 불가(기각). 분업이 비용·용량 최적.
