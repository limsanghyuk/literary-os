# ADR-172: Plugin Manifest 스키마 및 PluginLoader 설계 (V711)

**날짜**: 2026-05-28
**상태**: 승인됨
**작성자**: Literary OS Engineering Team (SP-D.3)

## 맥락

SP-D.3 Plugin Registry 레이어 구축을 위한 기반 모듈이 필요하다.
Literary OS에 외부 확장 플러그인(장르 특화, 코퍼스 어댑터 등)을 안전하게
탑재하기 위해 매니페스트 스키마와 로더 정책을 먼저 확립한다.

## 결정

1. **PluginManifest**: `frozen=True` 불변 데이터클래스로 선언.
   필드: plugin_id / name / version / entry_point / permissions / dependencies / schema_version / tags.

2. **PluginPermission Enum**: READ_CORPUS / WRITE_OUTPUT / CALL_LLM / READ_NKG / WRITE_NKG / NETWORK_OUT 6종.
   WRITE_NKG, NETWORK_OUT 은 기본 차단.

3. **PluginLoader**: 화이트리스트 기반 권한 검사 + importlib 동적 임포트.
   중복 등록 방지, unload 시 sys.modules 정리.

4. **schema_version**: major 버전 일치 강제. 현재 "1.0".

## 결과

- `literary_system/plugins/plugin_manifest.py` + `plugin_loader.py` 신설
- TC: 33개 PASS
- 다음 단계: V712 PluginRegistry core (register/discover/query)
