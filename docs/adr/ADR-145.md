# ADR-145: TD-3 — is_blocking → gate_passed 연결

**Status**: Accepted  
**Version**: V683 (v12.0.3-dev)  
**Date**: 2026-05-28  
**Deciders**: SP-D.1 팀  

---

## Context

`EnterpriseCostAlert.is_blocking` 프로퍼티(기존 74행)가 정의되어 있었으나
`EnterpriseCostControlGate.demo_run()` 에서 전혀 호출되지 않았다.
결과적으로 `EnterpriseCostSuiteReport.gate_passed` 는 항상 `True` 로 하드코딩되어,
어떤 테넌트가 예산을 초과해도 Gate가 통과되는 심각한 논리 오류(TD-3)가 있었다.

---

## Decision

### 변경 사항

1. **`EnterpriseCostReport.is_blocking` 프로퍼티 추가**  
   `self.alert.is_blocking if self.alert is not None else False` 로 위임.

2. **`CostAlertSummary` 데이터클래스 신설**  
   `blocking / exceeded / critical / gate_passed` 4필드.

3. **`EnterpriseCostControlGate._evaluate_alerts()` 정적 메서드 추가**  
   `report.is_blocking` 을 순회하여 `blocking_count` 산출,  
   `gate_passed = (blocking_count == 0)` 로 결정.

4. **`demo_run()` 수정**  
   `gate_passed=True` 하드코딩 → `_evaluate_alerts(reports).gate_passed` 로 교체.  
   `tenants_exceeded` 도 `summary.exceeded` 로 일관성 확보.

---

## Consequences

- **수정 전**: 예산 초과 테넌트 존재 시에도 `gate_passed=True` → Gate 무력화
- **수정 후**: EXCEEDED 테넌트 1건이라도 있으면 `gate_passed=False` → 차단 정상 동작
- **TC**: 20건 (TC-01~TC-20) 전부 PASS
- **회귀**: 없음 (기존 테스트 영향 없음)
