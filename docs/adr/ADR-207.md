# ADR-207: G94 Observability Completeness Gate

**상태**: Accepted  
**날짜**: 2026-05-29  
**버전**: V744  
**작성자**: Literary OS Team  

## 컨텍스트

Phase D 관측성 스택(OtelSdkAdapter, TraceContext, PrometheusExporter)의 완결성 검증과 함께, ADR-128에서 식별된 `deploy/` 패키지 G_CONNECTIVITY 단절 문제를 해소해야 한다. V744 GitNexus 분석 결과 `deploy` 패키지가 0 in/0 out으로 완전 고립 상태임이 확인되었다.

## 결정

G94 Observability Completeness Gate를 5개 체크로 구성하되, OC-4에서 `literary_system.deploy.PhaseEManifestValidator` 직접 임포트로 `deploy/` 연결성 문제를 해소한다.

| 체크 | 검증 대상 | 합격 조건 |
|------|-----------|-----------|
| OC-1 | OtelSdkAdapter 스팬 | `start_span` / `end_span` 호출 성공 |
| OC-2 | TraceContext W3C 왕복 | `inject` → `extract` traceparent 키 보존 |
| OC-3 | PrometheusExporter 스냅샷 | `MetricSnapshot.counters` dict 반환 |
| OC-4 | PhaseEManifestValidator | `run_all_checks()` 8/8 check 실행 (deploy/ 연결) |
| OC-5 | DR E2E 관측성 | backup + restore 왕복 SHA 검증 |

## deploy/ G_CONNECTIVITY 해소

ADR-128 G_CONNECTIVITY 위반 해소 방안:
- `_check_oc4_phase_e_manifest_validator()`가 `from literary_system.deploy import PhaseEManifestValidator`를 직접 임포트
- 이로써 `deploy` 패키지에 in-edge가 생성되어 G_CONNECTIVITY 단절 해소
- 이후 GitNexus Step 13 재실행 시 `deploy` 패키지는 "isolated" 목록에서 제외될 것으로 예상

## 결과

G94 PASS 시 관측성 스택 완결성 + deploy/ 연결성 동시 증명. ADR-128 에스컬레이션 해소.
