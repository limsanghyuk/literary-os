# ADR-116: PublicSDK v1.0 — Literary OS 외부 공개 클라이언트

## 상태
Accepted (V656, 2026-05-27)

## 맥락
SP-C.3 첫 번째 단계로, 내부 literary_system 모듈을 외부 개발자·파트너가 직접 사용할 수 있는
공개 SDK를 제공한다. SDK는 4개 핵심 메서드를 노출하며, offline_mode(기본)에서는
실제 LLM 없이도 동작하는 rule-based stub을 제공한다.

## 결정

### 4개 코어 메서드
| 메서드 | 입력 | 출력 | 설명 |
|--------|------|------|------|
| `analyze(text)` | str, context, lang | AnalyzeResult | 씬 품질 5축 분석 |
| `repair(text, issues)` | str, List[str], target_score | RepairResult | 이슈 기반 수정 |
| `predict(context, n)` | str, int, style_hint | PredictResult | 다음 씬 예측 후보 |
| `generate(blueprint)` | title+chars+setting+conflict | GenerateResult | 씬 생성 |

### 아키텍처
```
LiteraryOSClient
  ├── SDKConfig        — 환경변수 우선, offline_mode=True 기본
  ├── _RateLimiter     — 슬라이딩 윈도우 RPM 제한 (max_rpm=1000)
  ├── sdk_models.py    — 순수 dataclass (Pydantic 미사용)
  └── sdk_exceptions.py — 계층적 예외 (LiteraryOSError 상속)
```

### offline_mode 원칙
- `True`(기본): 규칙 기반 stub 반환 — LLM 없이 테스트 가능
- `False`: InferenceGateway HTTP 호출 (온라인 배포 전용)

### LLM-1 원칙 준수
`require_promoted_model=True` 기본값 → AutoPromotionGate(G62) 통과 모델만 서빙.

### Rate Limiting
슬라이딩 윈도우(60초): max_rpm=1000 초과 시 `RateLimitError` 즉시 발생.

## 대안 검토
- Pydantic 모델 사용: 외부 의존성 증가로 불채택 → 순수 dataclass 선택
- FastAPI 라우터로 직접 노출: V658 B2B API에서 별도 처리

## 결과
- literary_system/sdk/ 패키지 신설 (5개 파일)
- LLM 없이 4개 메서드 테스트 가능
- SP-C.3 G68~G71 게이트의 SDK 기반 마련
