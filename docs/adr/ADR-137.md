# ADR-137: Enterprise 통합 테스트 강화 (SP-C.4 안정화 1)

## Status
Accepted

## Date
2026-05-27

## Context
V673(EnterpriseSLOGate)과 V674(RevenueGate) 구현 후, 두 레이어의 통합 시나리오 검증이
단위 테스트만으로는 불충분하였다. 파트너 계약 ID 일치성, 크로스 레이어 파이프라인,
Gate 레지스트리 등재 여부를 통합 관점에서 검증할 필요가 있었다.

추가로 `SLOMonitor.check()`가 인스턴스 메서드임에도 불구하고 정적 호출 방식으로
호출하는 오류가 통합 테스트 작성 중 발견되었다.

## Decision
1. `tests/integration/test_v675_enterprise_integration.py` 신설 — 20 TC
   - SLO + Revenue 파트너 일치성 (TC01~TC05)
   - SLOMonitor 동작 검증 (TC06~TC10)
   - RevenueCalculator 통합 검증 (TC11~TC15)
   - Enterprise 패키지 내보내기 검증 (TC16~TC17)
   - 전체 파이프라인 End-to-End (TC18~TC20)
2. `SLOMonitor().check()` 인스턴스화 규칙 명확화
3. 20/20 TC PASS, Release Gate 75/75 PASS 확인

## Consequences
- Enterprise 레이어 통합 신뢰도 향상
- SLOMonitor 호출 규약 문서화
- 테스트 총계: 8,678 TC
