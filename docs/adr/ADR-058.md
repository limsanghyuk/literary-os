## 1. 맥락

V597에서 LoRA 학습 파이프라인(LoRATrainingConfig + LoRAJobRunner)이 완성됐다.  
V598에서는 학습된 LoRA 아티팩트를 **서빙 레이어에서 실제 사용**하기 위한 계약이 필요하다.

Phase B 본안 보강 핵심 결정사항:
- **B-M-03**: LoRAArtifact 3-tag (seed+commit+dataset_sha) 무결성
- **B-M-16**: TrainPlane/ServePlane Helm 격리 (ServePlane은 V626 구현)
- **Gate G53**: 추론 레이턴시 ≤2초 + 응답 100자+ 합격 기준

## 2. 결정

### 2.1 LoRAArtifact — safetensors 3-tag 아티팩트 계약 (B-M-03)

| 태그 | 타입 | 설명 |
|------|------|------|
| `seed_tag` | int | 학습 재현성 시드 (default: 42) |
| `commit_tag` | str | 학습 트리거 git commit SHA 7자 |
| `dataset_sha_tag` | str | 훈련 데이터셋 sha256 32자 hex prefix |

```
tag_string: "seed=42|commit=abc1234|dataset=01234567"
```

`sha256` 필드로 safetensors 파일 무결성 검증. `load_manifest()` 호출 시 자동 검증.  
검증 실패 시 `stage = CORRUPTED` + `ValueError` 발생.

### 2.2 LoRAModelRegistry — 체크포인트 버전 관리

단계 전환 규칙:

```
register()      → CANDIDATE    (최초 등록, 중복 불가)
promote() ×1    → VALIDATED    (EquivalenceTester 5축 검증 후)
promote() ×2    → PROMOTED     (서빙 활성화, 기존 PROMOTED → RETIRED)
retire()        → RETIRED      (명시적 퇴역)
mark_corrupted()→ CORRUPTED    (sha256 불일치 등 무결성 파괴)
```

**LLM-1 원칙**: PROMOTED 아티팩트만 서빙 허용.  
CANDIDATE/VALIDATED 아티팩트 추론 요청 → `RuntimeError`.

JSON 파일 영속화 (`lora_model_registry.json`). DVC remote 연동은 V600에서 구현.

### 2.3 LoRAInferenceGateway — LLMBridgeInterface 확장

```python
class LoRAInferenceGateway(LLMBridgeInterface):
    provider_name = "lora_local"
    generate(prompt, context) -> str
    infer(prompt, context) -> InferenceResult
    generate_with_response(prompt, context) -> LLMResponse
```

`InferenceResult`:
- `text`: 생성된 텍스트
- `latency_ms`: 추론 레이턴시 (밀리초)
- `artifact_id`: 사용된 LoRA 아티팩트 ID
- `backend`: 'transformers' | 'stub'
- `passes_g53`: latency_ms ≤ 2000 AND len(text) ≥ 100

**HAS_TRANSFORMERS=False 환경**: `StubInferenceBackend` 자동 사용 (CI/테스트).  
**HAS_TRANSFORMERS=True 환경**: PEFT + HuggingFace transformers 실 추론.

### 2.4 Gate G53 — 합격 기준

| 체크포인트 | 기준 | 비고 |
|-----------|------|------|
| G53-1 | 모듈 임포트 성공 | — |
| G53-2 | PROMOTED 아티팩트 존재 + is_available() | LLM-1 원칙 |
| G53-3 | latency_ms ≤ 2000 | stub 포함 |
| G53-4 | len(text) ≥ 100 chars | — |
| G53-5 | provider_name == "lora_local" | — |
| G53-6 | 3-tag 무결성 | seed int, commit 7자, dataset hex 32자 |
| G53-7 | CORRUPTED 아티팩트 추론 차단 | RuntimeError 발생 |
| G53-8 | passes_g53 복합 조건 | latency + length |

## 3. 기각 대안

| 대안 | 기각 이유 |
|------|-----------|
| 단일 단계 등록 (CANDIDATE → PROMOTED) | 검증 단계(VALIDATED) 생략 → 품질 리스크 |
| 레지스트리 없이 직접 경로 지정 | 버전 관리 불가, 롤백 어려움 |
| 추론 게이트 레이턴시 5초 | 드라마 씬 실시간 생성 UX 저하 |
| 텍스트 길이 50자 | 씬 단위 생성에 부족 |
