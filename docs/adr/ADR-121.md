# ADR-121: FeedbackLoopGate G69 — 24h 무중단 피드백 파이프라인 안정성 게이트

**상태**: Accepted  
**날짜**: 2026-05-27  
**버전**: V661 / v11.34.0  
**관련**: ADR-119 (G68 ReaderFeedbackCollector), ADR-120 (FeedbackToRLHF)

---

## 컨텍스트

SP-C.3 Phase C 목표 중 하나는 독자 피드백 수집 → RLHF 변환 파이프라인이
프로덕션 환경에서 24시간 무중단으로 안정적으로 동작함을 검증하는 것이다.

단순 단위 테스트(G68)가 개별 컴포넌트를 검증하는 것과 달리, G69는 전체
파이프라인의 시간적 연속성과 누적 안정성을 검증한다.

---

## 결정

`literary_system/gates/feedback_loop_gate.py`에 **FeedbackLoopGate(G69)** 를 구현한다.

### 시뮬레이션 구조

```
24h = 24 tick × 1tick/h
각 tick: collect(N feedbacks) → convert(RLHFBatch) → [purge every 6h]
```

### 합격 조건 (4축)

| 축 | 조건 | 근거 |
|----|------|------|
| 오류 tick | error_ticks == 0 | 24h 중 1건의 파이프라인 오류도 허용 안 함 |
| 완주 | tick_count == 24 | 전체 시뮬레이션 미완주 시 자동 실패 |
| 데이터 손실 | total_loss == 0 | 수집된 피드백이 변환 과정에서 손실되면 안 됨 |
| purge 동작 | purge_cycles ≥ 1 | 6h 주기 purge가 최소 1회 이상 실행되어야 함 |

### 핵심 파라미터

```python
SIMULATION_HOURS = 24
TICK_COUNT = 24
MIN_FEEDBACK_PER_TICK = 5
MAX_ALLOWED_ERRORS = 0
PURGE_EVERY_N_TICKS = 6
```

### 데이터 손실 정의

`total_loss = sum(max(0, collected - converted - outliers))`

z-score로 제거된 이상치(outliers)는 손실로 계산하지 않는다.
이상치 제거는 ADR-120의 정상 동작이기 때문이다.
