# ADR-139: V677 Enterprise 테넌트 격리 강화 (TenantIsolationGate G76)

## 상태
Accepted

## 날짜
2026-05-27

## 컨텍스트
SP-C.4 Enterprise Layer 3번째 안정화 단계로, 멀티테넌트 환경에서 테넌트 간
SLO·Revenue 계약 격리를 강제하는 감사 레이어가 필요하다.
기존 `multiwork/project_isolation.py`의 `IsolationViolation` 클래스와
`tenant/tenant_manager.py`의 `TenantStatus` 클래스가 DuplicateZero(G37) 규칙에
저촉될 수 있으므로, Enterprise 전용 네임스페이스를 명확히 분리한다.

## 결정
1. `literary_system/enterprise/tenant_isolation.py` 신규 파일에 Enterprise 전용
   클래스를 정의한다.
   - `EnterpriseTenantStatus` (기존 `TenantStatus`와 충돌 방지)
   - `IsolationLevel` (SHARED / DEDICATED / STRICT)
   - `EnterpriseTenant` (SLO + Revenue 계약 보유)
   - `EnterpriseIsolationViolation` (기존 `IsolationViolation`과 충돌 방지)
   - `TenantIsolationReport` (감사 결과 집계)
   - `EnterpriseTenantRegistry` (테넌트 CRUD)
   - `TenantIsolationAuditor` (3가지 격리 규칙 적용)
   - `TenantIsolationGate` (GATE_ID="G76", demo_run 4-테넌트 시나리오)

2. 격리 규칙
   - Rule 1: STRICT 레벨 테넌트는 SLO 계약 필수 (미보유 → error)
   - Rule 2: STRICT 레벨 테넌트는 Revenue 계약 권장 (미보유 → warning)
   - Rule 3: ENTERPRISE SLO 등급 테넌트가 SHARED 격리를 사용하면 error
   - SUSPENDED 테넌트는 감사 제외

3. DuplicateZero(G37) 준수를 위해 클래스명에 `Enterprise` 접두사를 부여한다.
