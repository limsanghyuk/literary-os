# ADR-111: EnsembleMemoryCache — CoordinatorResult TTL 캐시 (V651)

## 상태
승인됨

## 컨텍스트
동일 scene_id에 대한 반복 생성 요청이 발생할 수 있다.
AgentCoordinator 결과를 메모리에 캐싱하면 재계산 비용을 줄일 수 있다.

## 결정
- `EnsembleMemoryCache` 구현 (`literary_system/ensemble/memory_cache.py`)
- `CacheEntry` dataclass: scene_id, result_dict, created_at, ttl
- `CacheStats` dataclass: size, hits, misses, evictions, expirations, hit_rate
- API: put() / get() / invalidate() / clear() / purge_expired() / stats() / contains()
- TTL 기본값: 3600초 (1시간)
- 최대 크기 기본값: 256 항목
- 퇴출 정책: LRU (OrderedDict 기반 삽입순서 + move_to_end)
- get() 시 TTL 만료 항목 즉시 삭제 후 None 반환
- 스레드 비안전 (단일 스레드 전용; 멀티 워커 시 외부 락 필요)

## 결과
- 테스트: test_v651_memory_cache.py (30 TC)
- LLM-0 준수: 외부 API 직접 호출 없음
