# ADR-146: G81 — Pre-flight Fix Gate

**Status**: Accepted  
**Version**: V684 (v12.0.4-dev)  
**Date**: 2026-05-28  
**Deciders**: SP-D.1 팀  

---

## Context

SP-D.1 에서 수정한 기술부채 3건(TD-1/TD-2/TD-3) 및 구조 변경 1건(D-M-13)이
모두 올바르게 구현되었는지를 단일 Gate로 통합 검증할 필요가 있었다.
개별 단위 테스트는 존재하지만, 릴리즈 게이트 레벨에서의 종합 확인이 없으면
미래 리팩토링 시 퇴행이 감지되지 않을 위험이 있다.

---

## Decision

`literary_system/gates/pre_flight_fix_gate.py` — Gate G81 신설.

### 5축 검증 (FX-1~FX-5)

| 축 | 대상 | 검증 내용 |
|----|------|-----------|
| FX-1 | TD-1 | `benchmark.percentile()` NIST R-7 정확성 (n=0,1,2,100) |
| FX-2 | TD-2 | `RevenueCalculator.is_contiguous()` + `calculate_tiered()` 정확성 + ValueError |
| FX-3 | TD-3 | `EnterpriseCostReport.is_blocking` 위임 + `_evaluate_alerts()` gate_passed 결정 |
| FX-4 | D-M-13 | `gates/phase_c_exit_gate.py` Wrapper 존재, MIN_GATES=80, MIN_TESTS=8845, 체크포인트 ≥8 |
| FX-5 | 통합 | FX-1~FX-4 ALL PASS → gate_passed True |

### 공개 인터페이스

```python
run_pre_flight_fix_gate() -> PreFlightFixReport
run_g81_gate() -> dict  # release_gate.py 통합용
```

---

## Consequences

- **TC**: 32건 (TC-01~TC-32) 전부 PASS
- **회귀 보호**: TD-1~TD-3 + D-M-13 미래 퇴행 자동 감지
- **release_gate 통합 가능**: `run_g81_gate()` 딕셔너리 인터페이스 제공
