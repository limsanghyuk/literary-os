# ADR-043: VectorRealAdapter — numpy-optional 벡터 스토어 (LOSDB Phase B)

**상태**: 승인됨  
**날짜**: 2026-05-20  
**버전**: V584 (8.9.0)  
**작성자**: CSA · CSC · CSPE 3인 전문가 합의  

---

## 컨텍스트

V581에서 VectorMockAdapter를, V582에서 SQLiteRealAdapter를, V583에서 MigrationEngine을 완성했다.
LOSDB 3-layer 구조에서 vector 레이어는 Mock만 존재하며, REAL 구현이 부재한 상태다.

## 결정

`VectorRealAdapter`를 `literary_system/db/vector_real_adapter.py`에 구현한다.

### 핵심 설계 결정

1. **numpy-optional**: numpy 설치 시 자동 가속, 없으면 `math` stdlib 순수 Python fallback.
2. **JSON 영속화**: pickle 금지, `json` stdlib으로 float 배열 직렬화.
3. **파일 경로 보안**: `os.path.abspath()` 정규화로 path traversal 방지.
4. **하위 호환성**: `Migration.vector_ops: object = None` 기본값으로 기존 테스트 파손 없음.
5. **rollback 전략**: `apply()` 전 스냅샷 저장 → `rollback()` 시 복원.

### 지원 메트릭

- 코사인 유사도 (기본)
- L2 유클리드 거리

### Migration vector_ops 연산

- `{"op": "upsert", "id": str, "vector": List[float], "metadata": Dict}`
- `{"op": "delete", "id": str}`
- `{"op": "save"}`

## 결과

- `literary_system/db/vector_real_adapter.py` 신규 (359 lines)
- `literary_system/db/migration_manager.py`: `Migration.vector_ops` 필드 추가
- `literary_system/db/__init__.py`: `VectorRealAdapter`, `VectorRecord` export
- Gate G43 (`vector_real_adapter_g43`): 7체크포인트 ALL PASS
- 테스트: 43개 TC ALL PASS

## 트레이드오프

| 항목 | 결정 | 대안 |
|------|------|------|
| 벡터 엔진 | numpy-optional 순수 Python | chromadb, faiss, hnswlib |
| 영속화 | JSON | SQLite, pickle |
| 인덱스 | 선형 스캔 (O(n)) | HNSW (V585+ 고려) |

선형 스캔은 수십만 벡터까지 충분히 실용적이며, CI 의존성 최소화 원칙에 부합한다.
