# ADR-129: 경쟁 흡수 아키텍처 (SP-C.4 G72, C-M-11)

**날짜**: 2026-05-27  
**상태**: Accepted  
**작성자**: Literary OS Engineering Team

## 컨텍스트

SP-C.4 진입 전, 경쟁 AI 글쓰기 도구 5종(NovelAI/Sudowrite/Novelcrafter/NolanAI/Jenova)의 기능을 체계적으로 분석하고, Literary OS에 흡수할 인사이트와 제외할 IP 위험 요소를 분리해야 한다.

C-M-11 원칙: 각 경쟁사 흡수 분석 완료 시 반드시 IP 자문 commit을 남겨야 한다. IP 자문 commit 없이는 해당 버전의 Release Gate가 차단된다.

## 결정

1. `literary_system/absorption/` 패키지 신설 (SP-C.4 전용)
2. 공통 데이터 모델: `CompetitorProfile`, `AbsorptionReport`, `FeatureGap`, `IPAdvisoryCommit`
3. Gate G72: 서브 게이트 G72-1~G72-5 (경쟁사별) → 통합 G72
4. 각 버전 commit에 IP 자문 findings를 명시적으로 포함 (C-M-11)
5. IP 위험도 HIGH 항목은 `rejected_features`로 분류, 흡수 금지

|------|--------|-------|

## 결과

- 경쟁 분석의 체계화 및 IP 리스크 관리
- C-M-11 강제 집행 (commit 없으면 Gate FAIL)
- Literary OS 고유 강점(한국 드라마 서사, RLHF, 멀티에이전트) 재확인
