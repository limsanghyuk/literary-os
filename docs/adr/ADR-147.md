# ADR-147: CI 4단 분리 + 의존성 잠금 (D-M-06/07)

**Status**: Accepted  
**Version**: V685 (v12.0.5-dev)  
**Date**: 2026-05-28  
**Deciders**: SP-D.1 팀  

---

## Context

기존 CI는 `ci.yml` 단일 파일에 모든 단계를 혼합하여 PR 피드백이 느렸고,
의존성이 비고정 상태(`>=` 범위 지정)여서 재현 가능성이 낮았다.
SP-D.1 D-M-06/07에서 CI 4단 분리와 의존성 잠금을 요구한다.

---

## Decision

### D-M-06: CI 4단 분리 — `.github/workflows/ci_4tier.yml`

| Tier | 이름 | SLA | 트리거 | 블로킹 |
|------|------|-----|--------|--------|
| 1 | Unit Tests | ≤5m | PR/push | PR 블로킹 |
| 2 | Integration Tests | ≤30m | main/develop push | 머지 블로킹 |
| 3 | Perf Gate | ≤1h | release tag | Release 블로킹 |
| 4 | Chaos Tests | nightly | cron 17:00 UTC | 비블로킹 |

### D-M-07: 의존성 잠금 — `requirements-lock.txt`

핵심 의존성 버전 고정. pip-compile 워크플로로 갱신 가능.

### `pyproject.toml` — `[tool.mypy]` 섹션 추가

`strict=true` 모드, `files=["literary_system/core", "literary_system/enterprise", "literary_system/gates"]` 대상.

---

## Consequences

- PR 피드백 시간: 전체 실행(~30m) → Unit만(≤5m) 단축
- 의존성 재현 가능성 확보 (requirements-lock.txt)
- mypy strict 설정 기반 마련 (V686에서 type_stubs.py 와 연계)
