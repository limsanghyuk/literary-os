# ADR-039: Performance Baseline (V580)

**상태**: 확정  
**날짜**: 2026-05-19  
**버전**: V580 (8.5.0)  
**관련**: ADR-036 (AsyncDiscipline), Gate G39 PerformanceBaseline

---

## 맥락

V574 Principal Engineer Report C-10 항목: "성능 베이스라인 측정이 V580에만 있으며, V575 기준선과 비교할 수 없다." 프로덕션 A 등급 진입 조건으로 핵심 연산의 성능 회귀 방지 게이트가 필요하다.

## 결정

1. **대상 연산 3종** (순수 Python, 외부 의존성 없음)
   - JSON 직렬화/역직렬화 1,000회 — 기준 500ms
   - SHA-256 해시 10,000회 — 기준 200ms
   - 정규식 컴파일+매칭 5,000회 — 기준 300ms

2. **허용 편차**: ±30% (CI 환경 속도 차이 대응)

3. **Gate G39 PerformanceBaseline**: `_gate_performance_baseline_g39()` — GATES 리스트 마지막 등록, ADR-039 참조

4. **LLM-0 원칙 준수**: 외부 LLM 호출 없음. 표준 라이브러리(`json`, `hashlib`, `re`, `time`)만 사용.

5. **레이어**: L1 (아키텍처 계약) — 성능 회귀는 사용자 경험에 직접 영향

## 결과

- `literary_system/gates/release_gate.py`: `_gate_performance_baseline_g39()` 추가
- `literary_system/gates/gate_registry.py`: `"performance_baseline_g39": ("ADR-039", "V580", "L1")` 등록
- 38 Gates 전체 PASS (G1 ~ G39, 일부 번호 병합)

## 거부된 대안

- **실제 파이프라인 E2E 타이밍**: LLM 호출 포함 — LLM-0 위반
- **pytest-benchmark 도입**: 외부 의존성 추가 — 최소 의존성 원칙 위반
- **기준값 파일 저장**: 환경 편차가 커 오탐률 높음 — 고정 상수 + 30% 여유로 대체
