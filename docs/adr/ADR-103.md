# ADR-103: FeedbackIntegrator 이력 추적 및 신호 강도 트렌드 분석 설계

- **상태**: Accepted
- **버전**: V643 (v11.14.0)
- **날짜**: 2026-05-26
- **담당**: Literary OS Core Team

---

## 컨텍스트

V641에서 FeedbackIntegrator를 MetaLearnerCycle에 연결하여 `integrate()` 결과를 MetaLearner 손실에 반영하기 시작했다. 그러나 V641/V642에서는 각 사이클의 `IntegrationResult`를 누적하지 않았기 때문에, 시간적 흐름에서 피드백 신호 강도가 개선되고 있는지·안정적인지·약화되고 있는지 파악할 수 없었다.

Self-Learning Loop (SP-C.1) 완성을 위해서는 피드백 루프가 단순히 손실을 보정하는 것을 넘어, **신호 품질 자체를 추적**하는 메커니즘이 필요하다.

---

## 결정

### 1. `_feedback_integration_history` 내부 이력 리스트 추가

`MetaLearnerCycle` 인스턴스에 `_feedback_integration_history: List[IntegrationResult]` 를 추가하고, 각 사이클에서 `integrate()`가 호출될 때마다 결과를 append한다.

### 2. `FeedbackSignalSummary` 집계 데이터클래스

이력에서 신호 강도 시계열(`signal_values`)을 추출해 평균(`mean_signal`), 유효성(`is_effective`), 트렌드(`trend`) 를 계산하는 값 객체를 도입한다.

- `trend` 판단 기준: 마지막-두 번째 신호 차이가 +0.05 초과 → `"improving"`, -0.05 미만 → `"declining"`, 그 외 → `"stable"`
- 이력이 `FEEDBACK_SIGNAL_MIN_CYCLES(2)` 미만이면 `"insufficient"` 반환

### 3. `adjusted_loss` 계산 및 `CycleReport` 노출

`avg_correction_delta != 0.0` 조건이 충족될 때만 `adjusted_loss` 를 계산한다:

```
adjusted_loss = l_final + avg_correction_delta × FEEDBACK_ADJUSTED_LOSS_SCALE(0.10)
```

스케일 계수 0.10은 피드백 신호가 직접 손실을 제어하지 않고 **약한 보조 신호**로만 기여하도록 의도적으로 낮게 설정했다. MetaLearner 손실은 실제 학습 결과(`record_work_loss`)에서 계산되며, 피드백은 그 다음 사이클의 방향성만 미세 조정한다.

### 4. `add_feedback()` 퍼블릭 API

`FeedbackIntegrator.record_feedback()` 를 MetaLearnerCycle 외부에서 직접 호출하지 않도록, `add_feedback()` 를 래퍼로 제공한다. 이는 캡슐화를 유지하면서 테스트 및 외부 모듈에서 피드백을 주입할 수 있게 한다.

### 5. `FEEDBACK_SIGNAL_MIN_STRENGTH = 0.30`

`IntegrationResult.signal_strength` 가 0.30 미만이면 `CycleReport.feedback_signal_effective = False`. 이 임계치는 `MIN_FEEDBACK_FOR_SIGNAL=3`, `CORRECTION_WEIGHT=0.8`, `REJECTION_PENALTY=0.5` 기반의 실험적 최솟값이다.

## 대안 검토

| 대안 | 거부 이유 |
|------|-----------|
| 이력을 외부 저장소(DB/파일)에 저장 | LLM-0 원칙 위반 가능성, 불필요한 의존성 |
| 신호 강도 EMA(지수 이동 평균) 적용 | 구현 복잡도 대비 현재 단계에서 불필요 |
| `feedback_signal_trend()` 를 CycleReport 필드로 포함 | 매 사이클 요약 크기 과대, 필요 시 메서드 호출로 충분 |

---

## 결과

- `FeedbackSignalSummary` 값 객체로 신호 시계열 분석 가능
- `feedback_history()` 로 원시 이력 접근 가능  
- `CycleReport.adjusted_loss` 로 손실 조정 추적 가능
- `CycleReport.feedback_signal_effective` 로 유효성 빠른 확인 가능
- V644에서 MetaLearner 가중치 수렴 분석 시 이 이력을 활용할 수 있음
