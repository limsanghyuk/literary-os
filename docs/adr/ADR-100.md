# ADR-100 — SafetyRegressionV2 4축 안전 회귀 검증기 (V640-PATCH, F9)

**날짜**: 2026-05-26
**상태**: Accepted
**구현**: V640-PATCH (Phase C 본안 v1.2 F9 의무 이행)
**기반**: Phase C 본안 v1.2 §2.3, F9 (SafetyRegressionV2 임계 V641→V640-PATCH 앞당김)
**관련**: ADR-098 (LOSConstitutionV2), ADR-099 (ConstitutionWeightTracker)

---

## 1. 맥락 (Context)

Phase C 본안 v1.1(F2)에서 `literary_system/safety/safety_regression_v2.py` 신설을
V650 이전 의무로 지정했다. 본안 v1.2(F9) 생존 분석 결과 V640 시점에도 미신설
상태임이 확인되어 임계를 **V641 이전(V640-PATCH)**으로 앞당겼다.

V639(DataAugmentationController)와 V640(FeedbackIntegrator)이 본안에 없던 신규
모듈로 편입되었고, 이 모듈들이 생성하는 학습 데이터가 안전 검증 없이 ParternLibraryV2
및 MetaLearner에 유입될 위험이 확인됐다.

---

## 2. 결정 (Decision)

`literary_system/safety/safety_regression_v2.py` 에 `SafetyRegressionV2` 구현.

### 2.1 4축 구성

| 축 ID | 축명 | 탐지 대상 |
|---|---|---|
| `self_harm` | 자해·자살 | 자해·자살 관련 한국어/영어 구문 |
| `hate_speech` | 혐오 발언 | 민족·인종·성별·장애·종교 혐오 구문 |
| `pii` | 개인정보 | 이메일·휴대폰·주민번호·SSN·카드번호 |
| `copyright` | 저작권 | © 기호, 무단복제/전재/배포 구문 |

### 2.2 합격 기준

4축 **모두 0건** → `SafetyRegressionReport.passed = True`

1건이라도 발견되면 FAIL + `logger.warning()` 발행.

### 2.3 LLM-0 준수

외부 LLM API 호출 없음. 순수 정규식(re 모듈) 기반 탐지.

---

## 3. 결과 (Consequences)

### 긍정적
- DataAugmentationController + FeedbackIntegrator 생성 데이터의 안전 검증 가능
- MetaLearner(V641~V644) 학습 전 파이프라인 의무 검증 포인트 확보
- 4축 탐지기를 V646~V680까지 재사용 가능 (SP-C.2~C.4 공용)

### 부정적·위험
- 정규식 기반 → 맥락 이해 없음 (드라마 씬에서 오탐 가능)
- 영어 패턴은 간소화됨 (한국어 드라마 특성 고려)

---

## 4. 대안 검토

| 대안 | 기각 이유 |
|---|---|
| LLM 기반 안전 분류기 | LLM-0 원칙 위반 |
| 외부 API (Azure Content Safety 등) | LLM-0 위반 + 네트워크 의존성 |
| V650까지 연기 | F9 생존 분석 결과: V640 편차 누적 위험 증가 |
