## Context

V620 종료 시점 Phase B 인터페이스(P-IF) 트레이스가 코드 표면에 부재함이 확인됨:
- **P-IF-01** AgentEnvelope — canonical_bridge_v2.py에 없음
- **P-IF-03** ReaderFeedbackIngest — multiwork/에 없음
- **P-IF-04** OpenAPI SemVer — model_serving_endpoint.py에 없음

V620_R R1/R2/R3 PATCH 시도는 V버전 통합으로 supersedes.
V621 = SP-B.2 retrofit으로 3건 동시 부착.

---

## Decision

### P-IF-01: AgentEnvelope

- `literary_system/llm_bridge/agent_envelope.py` 신규 생성
- `AgentRole` Enum: SCENE_WRITER / CRITIC / EDITOR / HISTORIAN / READER_VOICE
- `AgentEnvelope` frozen dataclass: agent_id, role, prompt, context, parent_agent_id, session_id, metadata
- `RoutingDecision` Enum: LOCAL_LORA / EXTERNAL_LLM / CASCADE
- `RoutingPolicy` 4축 (cost/latency/quality/role) 가중치 합계 = 1.0 강제
- `canonical_bridge_v2.py`에 `_bridge_generate_with_envelope()` 하위 호환 래퍼 추가
- Phase B: agent_id='default', role=SCENE_WRITER 단일 사용

### P-IF-03: ReaderFeedbackIngest

- `literary_system/multiwork/reader_feedback_ingest.py` 신규 생성
- `ReaderFeedback` frozen dataclass: rating 1~5 검증, timestamp UTC 자동
- `RewardSignalAdapter` Protocol (runtime_checkable)
- `ReaderFeedbackIngest.ingest()`: Phase B = NotImplementedError
- `is_phase_c_active()`: 어댑터 연결 여부 반환

### P-IF-04: OpenAPI SemVer

- `model_serving_endpoint.py`에 SEMVER_MAJOR/MINOR/PATCH 상수 추가
- `get_api_version_response()`: {"semver": "1.0.0"}
- `get_openapi_schema()`: OpenAPI 3.1 스키마 딕셔너리
- `build_app_with_semver()`: /openapi.yaml + /api_version FastAPI 엔드포인트
- `tools/detect_openapi_breaking.py`: 메이저 버전 변경 탐지 CLI
- `tools/export_openapi.py`: YAML/JSON 내보내기
- `.github/workflows/openapi_diff.yml`: CI 자동화 (Phase B: warn-only)

## Consequences

**Positive**:
- P-IF-01~04 Phase C/D 인터페이스 트레이스 코드 표면 확보
- V630 G61 C7(verify_interfaces_trace) PASS 기반 마련
- openapi_diff CI로 브레이킹 체인지 자동 감지 (Phase B: warn-only)
- ReaderFeedbackIngest Phase C+ 확장 진입점 확보

**Negative / Trade-offs**:
- P-IF-03 실 기능(ingest) Phase C+로 지연
- openapi_diff.yml warn-only → Phase C+ 전환 시 --warn-only 제거 필요
