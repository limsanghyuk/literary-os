# ADR-237: RunPod 학습 운영 라이프사이클 (데이터 동기화 + 어댑터 회수)

## 상태
ACCEPTED (V777, Phase E.4 확장)

## 맥락
V772 어댑터는 파드 생성만. 실 클라우드 학습은 데이터셋이 파드로 가고(업로드) 학습된 LoRA가 돌아와야(회수) 완결. 그 운영 글루 부재.

## 결정
- `finetune/runpod_lifecycle.py` RunPodJobLifecycle: ①업로드 ②파드 학습 ③폴링(완료 대기, 유한) ④어댑터 회수 ⑤등재. 업로더/다운로더 **주입**(HF/S3/B2 운영, fake 테스트). dry_run/키없음=계획만(미수행).
- RealRunPodAdapter 페이로드: DATASET_URL/OUTPUT_URL env + dockerStartCmd(다운로드→학습→업로드).
- 보안: 키·URL 토큰 직렬화 미포함(TC13).

## 결과
13 TC PASS. 전 주기 작동(주입형 fake): upload→launch→poll(EXITED)→retrieve→completed. 폴링 타임아웃·실패·업로더 미주입 등 분기 처리. **사용자가 RunPod 키+스토리지 자격증명 주면 실 데이터로 클라우드 학습 완결.** 실 스토리지 엔드투엔드는 자격증명 부재로 미검증(주입 fake로 흐름 검증). LLM-0.
## 대안
RunPod 네트워크 볼륨 전용: 스토리지 종속(기각). 업로더/다운로더 주입으로 스토리지 무관 이식성.
