# ADR-040: Multi-backend MigrationManager (LOSDB 기반 레이어 v1.0)

| 항목 | 내용 |
|------|------|
| 상태 | 수락됨 (V581) |
| 결정자 | Chief Architect / Chief Compiler / Chief System PE |
| 날짜 | 2026-05-19 |
| 관련 ADR | ADR-058(PartialAvailability), ADR-059(MOCK-REAL 등가) |

## 컨텍스트

LOSDB(Literary OS Database)는 Vector(Qdrant) + Graph(Cypher/NetworkX) + Document(SQL)
세 계층의 통합 스토리지다. Phase A(V581~V595) 첫 단계로, 세 백엔드의 스키마 버전을
단일 진실 원천에서 관리하고 마이그레이션 순서를 보장해야 한다.

## 결정

1. **SchemaRegistry**: 백엔드별 현재 버전 + 마이그레이션 히스토리 단일 보관
2. **3-어댑터 구조**: SQLMigrationAdapter / GraphMigrationAdapter / VectorMigrationAdapter
3. **MOCK 기본**: 모든 어댑터는 `mock=True`로 초기화 (ADR-059 준수)
4. **MigrationManager**: 배치 적용 + stop_on_failure + SchemaRegistry 위임

## 결과

- 긍정: 버전 일관성 보장, 백엔드 독립 교체 가능
- 부정: REAL 어댑터는 V596+ 구현 예정 — 현재 MOCK 한계
- 리스크: 세 백엔드 동시 실패 시 부분 마이그레이션 → ADR-058 PartialAvailability로 완화 예정

## Gate 연결

Gate G40 (`_gate_db_migration_g40`) — SchemaRegistry + MigrationManager 생존 검증
