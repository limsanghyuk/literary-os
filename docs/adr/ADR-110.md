# ADR-110: AgentCoordinator — SP-C.2 Multi-Agent Ensemble 오케스트레이터 (V650)

## 상태
승인됨

## 컨텍스트
V646~V649에서 Director/Script/Critic/Editor 4개 에이전트를 독립적으로 구현했다.
이들을 C-M-09 규칙에 따라 오케스트레이션하는 단일 진입점이 필요하다.

## 결정
- `AgentCoordinator` 구현 (`literary_system/ensemble/agent_coordinator.py`)
- `CoordinatorResult` dataclass: scene_id, final_text, rounds_used, success, last_critic_score, last_fitness_decision, polish_notes, error
- `coordinate()` 메서드 오케스트레이션 순서:
  1. DirectorAgent.generate_blueprint() (Round-1 전용)
  2. ScriptAgent.generate(attempt_num=attempt)
  3. CriticAgent.evaluate(round_num=attempt)
  4. request_regeneration=True 이면 2-3 반복 (최대 MAX_ROUNDS=3)
  5. EditorAgent.finalize() — 항상 실행, 거부권 없음
- 의존성 주입 + Stub 폴백: 각 에이전트 모듈 임포트 실패 시 StubAgent 사용
- Gate G64 (coordinator_gate.py): 7 체크포인트 검증

## 결과
- 테스트: test_v650_agent_coordinator.py (30 TC)
- `ensemble/__init__.py`에 AgentCoordinator / CoordinatorResult 지연 로드 등록
- LLM-0 준수: 외부 API 직접 호출 없음
