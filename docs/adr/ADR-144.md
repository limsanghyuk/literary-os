# ADR-144: TD-2 TIERED Revenue 비연속 구간 검증 (is_contiguous)

**상태**: Accepted  
**날짜**: 2026-05-28  
**버전**: V682 / v12.1.0-dev  
**관련**: ADR-136 (RevenueGate G74)

## 배경

`revenue.py` TIERED 계산 로직(L103-L117)은 티어 간 공백/겹침을 검증하지 않았다.
비연속 티어 `[0,100] → [200,∞]` 입력 시 100~200 구간이 누락된 채 계산되어 잘못된 수익 분배가 발생할 수 있었다.

## 결정

1. `RevenueCalculator.is_contiguous(tiers)` 정적 메서드 추가 — 정렬 후 `t[i].max_amount == t[i+1].min_amount` 검증
2. `RevenueCalculator.calculate_tiered(contract, gross)` 정적 메서드로 TIERED 로직 캡슐화 — is_contiguous 검증 포함
3. `calculate_partner_share()` 내 TIERED 분기를 `calculate_tiered()` 위임으로 교체

## 결과

- 비연속 티어 입력 시 명시적 `ValueError` 발생 (메시지에 contract_id + 범위 포함)
- 연속 티어 계산 로직 동일 유지 (회귀 없음)
- 20 TC ALL PASS

## 대안 검토

- 조용한 누락 허용 (No) — 잘못된 수익 계산은 재무 오류 유발
- max_amount 자동 보정 (No) — 의도치 않은 데이터 변경 위험
