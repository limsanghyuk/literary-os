# ADR-234: loop-C 폐회로 — LoopCClosure + G_LOOPC_WINRATE

## 상태
ACCEPTED (V774, Phase E.4 확장)

## 맥락
회사 데이터트랙 설계도(2026-06-16_v773_loopC_closure_design)가 V773을 "loop-C 폐회로"로 정의하고 ADR-233 제안. 그러나 개발자 V773은 클라우드 배선(ADR-233 선점). **정합화: ADR-233=클라우드 배선(유지), 폐회로 글루=ADR-234(본).** 클라우드 배선은 폐회로 ③단계 부품으로 흡수.

## 결정
- `learning/winrate_gate.py` g_loopc_winrate(G_LOOPC_WINRATE): 1차 ΔW>0 · 2차 KL≤τ · 3차 구조R 비퇴행 AND → adopt, 아니면 rollback. 표본<50 신뢰약 표기(정직).
- `learning/loopc_closure.py` LoopCClosure: 선호쌍→스펙→dispatch(dry 계획)→실측 W₁ 주입→수용판정→다음행동(adopt_continue/adopt_done/rollback_feedback). 실 학습은 GPU(4070/클라우드).
- `tools/run_loopc_round.py` CLI(계획/실측판정).

## 결과
15 TC PASS. "선호쌍→학습→명작 대비 재측정→다음 라운드" 폐회로 글루 완성 = 1단계(자율 생성 품질) 반복 엔진. 실 학습 W₁은 개발자 GPU에서 주입. LLM-0(생성기만 학습). 단 단일 라운드·소수 쌍은 ΔW 노이즈 → 라운드 누적/표본 확대 신뢰조건 동반.
## 대안
배선만으로 종료(폐회로 글루 생략): 학습→재측정→결정 자동화 불가(기각).
