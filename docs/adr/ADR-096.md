## 배경

V628까지 Literary OS의 운영 인프라(모니터링, Helm, 헬스체크)가 완성되었다.

## 결정

### §1 OpsRunbook — 운영 런북 관리자

`literary_system/ops/ops_runbook.py`에 코드-퍼스트 운영 런북을 구현한다.

**핵심 타입:**

| 타입 | 역할 |
|------|------|
| `RunbookStep` | 단일 단계 (action_fn + rollback_fn + severity) |
| `OpsRunbook` | 단계 순서 실행·롤백 관리자 |
| `RunbookResult` | 실행 결과 (success/steps_*) |
| `RunbookSeverity` | LOW / MEDIUM / HIGH / CRITICAL |
| `StepStatus` | PENDING / RUNNING / SUCCESS / FAILED / SKIPPED / ROLLED_BACK |

**실행 정책:**
- `stop_on_failure=True` (기본): 첫 실패 즉시 역순 롤백 후 중단
- `stop_on_failure=False`: 실패해도 계속 진행 (multi-failure 집계)
- `dry_run=True`: `skip_on_dry_run=True` 단계 건너뜀
- 컨텍스트 `context` dict를 통해 단계 간 결과 공유 (`__result_{step_name}` 키)

**bug fix:** `context or {}` → `context if context is not None else {}`
(빈 dict가 falsy로 평가되어 외부 컨텍스트 변경이 반영되지 않는 문제 해결)

**팩토리:** `build_health_check_runbook()` — 3단계 표준 헬스체크 런북

### §2 APIReferenceGenerator — API 레퍼런스 자동 생성기

`literary_system/docs/api_reference_generator.py`에 엔드포인트 스펙 기반
Markdown + OpenAPI 3.1 Fragment 생성기를 구현한다.

**핵심 타입:**

| 타입 | 역할 |
|------|------|
| `EndpointSpec` | 단일 엔드포인트 (path/method/params/responses) |
| `ParamSpec` | 파라미터 스펙 (query/path/header/cookie) |
| `ResponseSpec` | 응답 스펙 (status_code/description/schema_ref) |
| `HTTPMethod` | GET/POST/PUT/PATCH/DELETE/HEAD/OPTIONS |
| `APIReferenceGenerator` | 엔드포인트 등록 → Markdown/OpenAPI 생성 |
| `APIReferenceReport` | 생성 결과 (endpoint_count/markdown/openapi_fragment/tag_list) |

**출력 형식:**
- **Markdown**: 태그별 그룹핑, 파라미터 테이블, deprecated 표시
- **OpenAPI 3.1 Fragment**: paths 객체 포함 완전한 스키마 (JSON)

**operation_id 자동 생성:** 경로 슬래시·중괄호 제거 후 `{method}_{path}` 패턴

### §3 ATIAMetadataAuditor — ATIA 메타데이터 외부 감사 패키지

`literary_system/audit/atia_metadata_auditor.py`에
AI 투명성(T)·해석가능성(I)·책임성(A) 3축 감사 엔진을 구현한다.

**점수 체계:**

| 차원 | 가중치 | 측정 기준 |
|------|--------|-----------|
| Transparency (T) | 0.30 | 로그·버전·변경이력 공개 수준 |
| Interpretability (I) | 0.30 | 의사결정 설명 가능성 |
| Accountability (A) | 0.40 | 감사 추적·권한·오류 귀속 명확성 |

**종합 점수:** `overall = 0.30*T + 0.30*I + 0.40*A`

**위험 등급:**

| 등급 | 조건 |
|------|------|
| LOW | overall ≥ 0.80 |
| MEDIUM | 0.60 ≤ overall < 0.80 |
| HIGH | 0.40 ≤ overall < 0.60 |
| CRITICAL | overall < 0.40 |

**모듈 통과 조건:** overall ≥ 0.60 AND 각 축 ≥ 0.40

**시스템 통과 조건:** 시스템 평균 overall ≥ 0.60 AND critical 모듈 0건

**외부 감사 패키지:** `export_package()` → `{"audit_report.json": ..., "audit_summary.md": ...}`
