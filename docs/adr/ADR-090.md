# ADR-090: V623 SystemIntegrationTest + Helm 사전 검증

**Status:** Accepted  
**Date:** 2026-05-25  
**Version:** V623 (v10.28.0)  
**Author:** Literary OS Engineering

---

## Context

V621(SP-B.2 retrofit) + V622(SP-B.3 retrofit) 두 버전의 retrofit 작업을
통합 E2E 관점에서 교차 검증해야 한다.

G37(DuplicateZero) FAIL 원인:
- `llm_node_router.py`: RoutingPolicy(str, Enum) — 3축 라우팅 열거형
- `agent_envelope.py`: RoutingPolicy dataclass — 4축 가중치 정책 (V621 신규)

동명이형 클래스가 G37 FAIL → G61 C5-TotalGates 연쇄 FAIL.

---

## Decision

### §A — RoutingPolicy 중복 해소

agent_envelope.py의 RoutingPolicy dataclass를 AgentRoutingPolicy로 rename:
- 의미 명확화: 에이전트 기반 4축 라우팅 정책
- RoutingPolicy(str, Enum)은 llm_node_router.py 단독 소유 유지
- canonical_bridge_v2.py + 테스트 파일 모두 AgentRoutingPolicy로 업데이트

연쇄 효과: G37 PASS → passed_count(59)+1=60 >= MIN_GATES(60) → G61 C5 PASS

### §B — SystemIntegrationTest 확장

V621/V622 retrofit 4개 TC 클래스 신규 (+39 TC):

| 클래스 | TC | 검증 내용 |
|--------|----|-----------|
| TestV621AgentEnvelopeIntegration | 10 | AgentEnvelope/AgentRoutingPolicy E2E |
| TestV622SPB3RetrofitIntegration | 13 | ConflictPolicy/WorkloadProfile/AdvSeed E2E |
| TestHelmPreValidation | 7 | Chart.yaml/values.yaml + helm lint/dry-run |
| TestV623CrossComponentIntegration | 9 | V621+V622 크로스 컴포넌트 |

### §C — Helm 사전 검증 전략

- 파일 시스템 검증(Chart.yaml/values.yaml/templates/): 항상 실행
- helm lint/dry-run: helm 미설치 시 pytest.skip()

---

## Consequences

**Positive:**
- 60/60 Gates PASS 달성 (G37+G61 해소)
- V621/V622 retrofit 크로스 검증 39 TC 확보
- AgentRoutingPolicy 네이밍으로 의미 명확화

**Constraints:**
- test_v621_sp_b2_retrofit.py: AgentRoutingPolicy as RoutingPolicy alias 사용
- helm CI: self-hosted 환경 권장

**LLM-0 원칙:** 모든 테스트 외부 LLM 호출 없음 (MOCK 모드)
