# ADR-042 — LOSDB Phase B: MigrationEngine 통합 오케스트레이터

## 상태
Accepted

## 날짜
2026-05-20

## 버전
V583 (8.8.0)

## 배경
V582에서 SQLiteRealAdapter(SQL 레이어) REAL 구현이 완성됐다.
LOSDB 3-layer 아키텍처(SQL / Graph / Vector)에서 각 어댑터를 독립적으로 호출하면
오케스트레이션 로직이 호출 코드에 분산된다.

## 결정
`MigrationEngine` 통합 오케스트레이터를 도입한다.
- 복수 어댑터를 Dict로 등록하고 단일 진입점으로 조율
- `MigrationPlan`으로 실행 대상 명시
- `MigrationExecutionRecord`로 실행 결과 감사 로그 생성
- 실패 시 역순 롤백 체이닝 자동 수행

## 결과
- 장점: 어댑터 추가/교체가 엔진 코드 변경 없이 가능
- 장점: 실행 감사 레코드 JSON 직렬화로 영속성 지원
- 장점: 롤백 체이닝으로 부분 실패 상황 안전 처리
- 제약: Graph/Vector REAL 어댑터는 별도 ADR에서 구현

## Gate
G42 `migration_engine_g42` (L1)

## 관련
- ADR-040 (LOSDB Phase A Mock)
- ADR-041 (SQLiteRealAdapter REAL)
