**상태**: Accepted

**관련 ADR**: ADR-051 (GPUAdapterContract), ADR-056 (LoRA Dataset Pipeline)

## 1. 맥락

V596에서 LoRA Dataset Pipeline + Governance(ADR-056)가 완료됐다.
V597에서는 그 데이터셋을 실제 Llama-3.1-8B 모델에 적용하는 **학습 설정과 실행 파이프라인**이 필요하다.

Phase B 본안 보강 핵심 결정사항:
- **B-M-04**: 베이스 모델 Llama-3.1-8B (128K), EXAONE-3.5-7.8B A/B 후보
- **B-M-05**: rank=16, alpha=32, target_modules=[q_proj, k_proj, v_proj, o_proj]
- **B-M-06**: 격주 풀 학습($48) + 주간 미세조정($48) → 월 SLO $96
- **B-M-16**: TrainPlane/ServePlane Helm 네임스페이스 격리

## 2. 결정

### 2.1 LoRA 하이퍼파라미터 (B-M-05)

| 파라미터 | 값 | 근거 |
|----------|-----|------|
| `lora_rank` | 16 | 8K~10K 샘플 미만 데이터셋에서 rank-16이 rank-32 대비 성능 동등, 메모리 2배 절감 |
| `lora_alpha` | 32 | 관례: alpha = 2 × rank. scaling_factor = 2.0 |
| `lora_dropout` | 0.05 | 과적합 방지 최소 드롭아웃 |
| `target_modules` | q/k/v/o_proj | Attention 4-head 전체 적용 (MLP 제외로 파라미터 최소화) |
| `learning_rate` | 2e-4 | LoRA 표준 초기 LR |
| `batch_size` | 4 | A100-40G 기준 OOM 없는 최대 배치 |
| `grad_accum` | 4 | effective batch = 16 |
| `max_seq_len` | 2048 | 드라마 씬 단위 기준 |
| `bf16` | True | Llama-3.1 공식 권장 |

### 2.2 베이스 모델 전략 (B-M-04)

**1차 후보**: `meta-llama/Llama-3.1-8B`
- 128K 컨텍스트, 한국어 지원 OK (multilingual)
- HuggingFace Hub 공개 접근 가능

**A/B 후보**: `LGAI-EXAONE/EXAONE-3.5-7.8B-Instruct`
- LGAI Research 한국어 특화 모델
- V597에서 `LoRATrainingConfig.exaone_candidate()` 인터페이스 확보
- 실제 A/B 비교는 V600 FinetuningGate에서 수행

### 2.3 학습 스케줄 정책 (B-M-06)

```
격주 학습 (FULL_BIWEEKLY):
  - 주기: 14일마다 (KST 월요일 02:00)
  - 예상 시간: 6h (A100 기준)
  - 예상 비용: $48/회 → 월 $48

주간 미세조정 (FINE_WEEKLY):
  - 주기: 7일마다 (KST 수요일 02:00)
  - 예상 시간: 2h
  - 예상 비용: $48/월 (4회)

월 합계 목표: $96 (SLO soft=$90, hard=$120, emergency=$150)
```

`BiweeklyScheduler`가 마지막 실행 타임스탬프를 기반으로 자동 판단.

### 2.4 TrainPlane / ServePlane 격리 (B-M-16)

```
literary-train 네임스페이스:
  - LoRA 학습 Job (GPU exclusive)
  - CronJob: full-train-cron (격주), fine-tune-cron (주간)
  - StorageClass: dataset PVC (10Gi) + output PVC (50Gi)

literary-serve 네임스페이스 (V598에서 구현):
  - LoRAInferenceGateway
  - 추론 전용 GPU
```

네임스페이스 격리로 학습 중 추론 성능 저하 방지.

### 2.5 비용 SLO 집행

`LoRAJobRunner._check_slo_before_run()`:
1. 예상 비용 계산 (adapter.estimate_cost)
2. 월 누적 + 예상 합산
3. HALT → RuntimeError (즉시 중단)
4. BLOCK → RuntimeError (신규 차단)
5. WARN → UserWarning (계속 허용)

## 3. 기각 대안

| 대안 | 기각 이유 |
|------|-----------|
| rank=32 | 파라미터 2배 증가, 동등 성능, 불필요한 비용 증가 |
| MLP 레이어 포함 | 추가 성능 개선 미미, 학습 시간 증가 |
| 매주 풀 학습 | 월 비용 $96→$192, SLO 초과 |
| 단일 네임스페이스 | 학습/추론 GPU 경쟁으로 P95 latency 악화 |

---

## 4. 결과

- `LoRATrainingConfig`: 설정 직렬화/역직렬화, default_full() / default_fine() 팩토리
- `LoRAJobRunner`: GPUAdapter 연동, SLO 집행, 이력 JSONL 영속화
- `BiweeklyScheduler`: 격주/주간 due 판단 로직
- `deploy/helm/train_plane/`: Kubernetes 배포 스텁
