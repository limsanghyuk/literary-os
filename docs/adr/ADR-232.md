# ADR-232: RealRunPodAdapter — RunPod 실 REST API 연동

## 상태
ACCEPTED (V772, Phase E.4 확장)

## 맥락
기존 RunPodAdapter(V590)는 Mock이라 "클라우드 학습"이 시뮬레이션 숫자만 반환. 실 클라우드 학습을 위해 RunPod REST v1 실연동 필요. (사용자 지적: 클라우드로 학습하면 결과가 나오지 않나)

## 결정
`finetune/runpod_real_adapter.py` RealRunPodAdapter(GPUAdapterContract):
- RunPod REST v1(https://rest.runpod.io/v1), Bearer 인증. verify_key(GET /pods), launch_job(POST /pods 실 파드 생성, env로 학습 파라미터 주입), poll(GET /pods/{id}).
- 주입형 transport로 네트워크 없이 단위 테스트. **dry_run·키없음=네트워크 미호출(안전)**.
- 보안: api_key는 인자/RUNPOD_API_KEY env만, 결과 직렬화에 키 미포함(TC13 검증).

## 결과
13 TC PASS. 키 주입 시 POST /pods 실 호출로 파드 생성·RUNNING 반환, 키 없으면 dry_run. **사용자가 RunPod 키+자금을 넣으면 진짜 클라우드 학습 가동.** 단 실 RunPod 계정 부재로 엔드투엔드(실 과금 파드)는 미검증 — 주입 transport로 계약·페이로드·인증·에러 경로 검증. LLM-0(GPU 인프라 호출).
## 대안
GraphQL(구 API): REST v1이 신규 권장. 데이터 동기화 자동화는 후속(현재 env+dockerStartCmd 골격).
