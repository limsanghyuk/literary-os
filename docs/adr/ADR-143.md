## 배경

`literary_system/enterprise/benchmark.py:130`에서 P99 계산:

```python
# 구버전 (최댓값 편향)
p99_ms = float(sorted(elapsed_list)[int(len(elapsed_list) * 0.99)])
```

n=50일 때 `int(50 * 0.99) = 49` → `sorted[49]` = 최댓값 반환.  
n=100일 때 `int(100 * 0.99) = 99` → `sorted[99]` = 최댓값 반환.  
이는 P99가 아닌 P100(최댓값)을 반환하는 **최댓값 편향** 오류.

## 결정

NIST recommended method (R-7) 기반 선형 보간 `percentile()` 함수를 도입.

```python
def percentile(data: List[float], p: float) -> float:
    rank = p * (n - 1)
    lo, hi = int(rank), min(int(rank) + 1, n - 1)
    frac = rank - lo
    return sorted_data[lo] + frac * (sorted_data[hi] - sorted_data[lo])
```

n=50, P99: rank = 0.99 × 49 = 48.51 → 49 + 0.51×(50-49) = **49.51** (정확)  
n=100, P99: rank = 0.99 × 99 = 98.01 → 99 + 0.01×(100-99) = **99.01** (정확)

## 결과

| 구분 | 구버전 | 신버전 (R-7) |
|------|-------|------------|
| n=50 P99 | 50.0 (최댓값) | 49.51 (정확) |
| n=100 P99 | 100.0 (최댓값) | 99.01 (정확) |
| n=1 P99 | data[0] | data[0] |
| n=0 P99 | IndexError | 0.0 |

## 영향 범위

- `benchmark.py` 수정 1줄 + `percentile()` 함수 추가
- BenchmarkGate G75-BM 결과값 소폭 변경 (P99 낮아짐 → 더 엄격한 측정)
- 기존 SLO 임계값(max_p99_ms) 변경 불필요

## 대안 검토

- `statistics.quantiles(data, n=100)[98]`: Python 3.8+ exclusive method — 결과 유사하나 API 불일치 위험
- numpy.percentile: 의존성 추가 불필요 (내부 구현으로 충분)
