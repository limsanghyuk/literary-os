# ADR-080: ContaminationDetector — 훈련 데이터 오염 탐지기 (V638)

## 상태
Accepted (2026-05-26)

## 컨텍스트
SP-C.1 Self-Learning Loop의 8번째 컴포넌트.  
훈련 데이터 품질 보증을 위해 오염 유형별 자동 탐지가 필요하다.

V637(ConstitutionEvalV2)이 평가 단계의 품질을 보장한다면,
V638(ContaminationDetector)은 훈련 데이터 입력 단계의 품질을 보장한다.

## 결정

### 탐지 유형 4종

| 플래그 코드 | 임계값 | 탐지 기준 |
|---|---|---|
| LABEL_NOISE | 5% | 레이블 불일치 수 / 전체 샘플 수 |
| NEAR_DUPLICATE | 10% | MinHash 중복 샘플 수 / 전체 샘플 수 |
| DISTRIBUTION_SHIFT | 2.0σ | 축별 점수 z-score 이탈 |
| POISON_PATTERN | 1% | 블랙리스트 패턴 매칭 비율 |

### 핵심 클래스
- `ContaminationFlag`: 개별 오염 플래그 (flag_id, severity, detail)
- `ContaminationReport`: 오염 탐지 보고서 (UUID4, JSONL 직렬화)
- `ContaminationDetector`: 스캔 엔진 (scan / batch_scan / history API)

### 영속화
- LOSDB JSONL append-only (`:memory:` 모드 지원)
- `data/losdb/contamination_detector.jsonl`

### 블랙리스트 패턴
- `ignore ... previous instruction`
- `system prompt`
- `jailbreak`
- `<|...|>` 특수 토큰

## 결과
- LLM-0 원칙 완전 준수 (외부 LLM 호출 없음)
- 순수 Python 표준 라이브러리 (math, re, json, uuid)
- TC-01~33 단위 테스트 33/33 PASS
- SP-C.1 V639 DataAugmentationController 기반 마련
