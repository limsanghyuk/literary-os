# ADR-194: FLCoordinator — 연합 학습 조율자 기초 (V732)

**날짜**: 2026-05-29
**버전**: v12.5.1 (V732)
**상태**: ACCEPTED
**결정자**: SP-D.4 FL 팀

## Context

SP-D.4의 D-M-10 목표는 Federated Learning PoC 구현이다. 연합 학습에서는
중앙 서버(Coordinator)가 클라이언트의 로컬 가중치를 집계하여 글로벌 모델을
갱신한다. 먼저 타입 스키마(FLClientState, FLGlobalModel, FLRound)와
조율자 골격을 구현한다.

## Decision

`literary_system/federation/` 신규 패키지:
- `fl_types.py`: FLClientState, FLGlobalModel, FLRound 데이터클래스
- `fl_coordinator.py`: FLCoordinator
  - `register_client(client_id)` / `unregister_client(client_id)`
  - `start_round() → FLRound` — min_clients 미충족 시 InsufficientClientsError
  - `submit_client_state(state)` — 클라이언트 로컬 결과 수집
  - `finalize_round(global_model) → FLRound` — 수렴 판정 포함
  - `should_continue() → bool` — max_rounds / convergence_threshold 기준

## Rationale

- 최소 클라이언트 수(min_clients) 강제 → 단일 클라이언트 글로벌 업데이트 차단
- 수렴 판정 δ < 1e-4 → 조기 종료 지원
- FLRound 로그로 재현 가능한 실험 이력 확보

## Consequences

- V733: ADR-195 (FedAvg 알고리즘 스텁)
- V734~V735: FedAvg 완전 구현 + FL 클라이언트
- G90 FL Gate 진입 기반 마련
