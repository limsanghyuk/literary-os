# ADR-228: ProviderRouter — GPU 3-모드 자동 라우팅 (GPU 3-모드 ②)

## 상태
ACCEPTED (V768, Phase E.4 확장)

## 맥락
LocalGPUAdapter(V767, ADR-227)로 로컬 경로가 생겼으나, 작업마다 사람이 provider를 고르는 건 비효율. 설계도 §3.3의 5규칙으로 자동 분배 필요.

## 결정
`learning/provider_router.py` ProviderRouter.select(spec, signals)→RoutingDecision:
- R1 force_provider 명시 / R2 민감 코퍼스→LOCAL 강제(클라우드 금지, 프라이버시 1순위) / R3 8B 초과→CLOUD / R4 biweekly 무인→CLOUD / R5 기본 개발 반복+Preflight PASS→LOCAL / R6 Preflight 실패→CLOUD 폴백.
- `validate_routing()` = G_GPU_ROUTING: V1 민감 데이터 CLOUD 금지, V2 민감 폴백 금지, V3 provider 유효.
- `route_trigger()`: 결정 후 RLAIFTrigger(V765) 자동 실행.
- `tools/run_gpu_routing_gate.py` 6시나리오 CLI.

## 결과
28 TC PASS(V767 포함). 3-모드(로컬/클라우드/하이브리드)가 한 코드베이스에서 작업별 자동 선택. 이 샌드박스(GPU 없음)는 기본 R6→cloud 폴백, 4070에선 R5→local. 프라이버시·용량·스케줄 신호가 안전하게 라우팅. LLM-0 유지. GPU 3-모드 완성.
## 대안
수동 provider 지정: 비효율·실수 위험(기각). 규칙 기반 자동화 채택, 파레토 최적화는 차기(E2).
